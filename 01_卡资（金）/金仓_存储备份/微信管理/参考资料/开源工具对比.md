## macOS 推荐方案

### 方案一：chatlog（推荐）

**优点**：
- 支持 macOS 和 Windows
- 支持微信 3.x 和 4.x
- 自动解密数据库
- 提供命令行 UI 和 HTTP API
- 活跃维护中

**安装**：

```bash
# macOS
brew install chatlog

# 或下载二进制
# https://github.com/sjzar/chatlog/releases
```

**使用**：

```bash
# 启动 HTTP API 服务
chatlog server

# 命令行模式
chatlog cli
```

### 方案二：wechat-decipher-macos

**优点**：
- 专为 macOS 设计
- 使用 DTrace 抓取密钥
- 开源脚本可定制

**缺点**：
- 需要关闭 SIP
- 需要手动操作

**使用**：

```bash
# 1. 关闭 SIP（需重启到恢复模式）
csrutil disable

# 2. 运行 DTrace 脚本
sudo dtrace -s dbcracker.d

# 3. 登录微信，等待抓取密钥

# 4. 使用密钥解密数据库
```

## Windows 工具详情

### WeChatMsg

**功能亮点**：
- 支持所有消息类型导出
- 界面还原度高（类似微信聊天界面）
- 生成年度聊天报告
- 支持 Word/HTML/CSV/TXT 格式
- MIT 开源协议

**系统要求**：
- Windows 64位
- 微信 4.0+

**使用限制**：
- 不支持 macOS
- 无法恢复已删除消息

### PyWxDump

**功能亮点**：
- Python 编写，可移植性好
- 支持多账户
- 适合二次开发
- 可用于 AI 训练数据

**适合人群**：
- 开发者
- 需要自定义分析的用户

## 解密技术对比

### 密钥获取方法

| 方法 | 工具 | 难度 | 说明 |
|:-----|:-----|:-----|:-----|
| **LLDB 断点** | 系统自带 | ⭐⭐⭐ | 在 sqlite3_key 设断点 |
| **DTrace 脚本** | wechat-decipher-macos | ⭐⭐ | 监控文件操作抓取 |
| **内存扫描** | chatlog | ⭐ | 自动化处理 |

### SQLCipher 版本

| 微信版本 | SQLCipher | Page Size | KDF 迭代 |
|:---------|:----------|:----------|:---------|
| 3.x | v3 | 1024 | 64000 |
| 4.x | v4 | 4096 | 256000 |

## 选择建议

### macOS 用户

```
推荐路径：chatlog → 自动解密 → API 对接
备选路径：wechat-decipher-macos → 手动解密 → 自定义脚本
```

### Windows 用户

```
推荐路径：WeChatMsg → 一键导出 → 年报生成
开发路径：PyWxDump → 数据处理 → 自定义分析
```

### 跨平台需求

```
推荐工具：chatlog
原因：同时支持 macOS/Windows，API 接口统一
```

## 注意事项

### 法律合规

1. **仅限个人数据**：只能导出自己的微信数据
2. **隐私保护**：导出数据需妥善保管
3. **合规使用**：遵守微信服务协议

### 技术风险

1. **版本兼容**：微信更新可能导致工具失效
2. **SIP 关闭**：降低系统安全性
3. **密钥变化**：重新登录后密钥可能变化

### 数据安全

1. 导出的数据库包含敏感信息
2. 建议加密存储
3. 避免上传到公共云端
