# Time Machine 备份到家里 DiskStation 报错排查

**现象**：系统设置 → 时间机器 → 「共享 - DiskStation.local」显示**红点**，提示「正在等待完成第一次备份」。

**家里 NAS**：DiskStation（192.168.110.29 / DiskStation.local）

---

## 一、NAS 端（DSM）必查项

在浏览器打开 **http://192.168.110.29:5000** 登录 DSM，按下面逐项检查。

### 1. 启用 Time Machine 备份

- **控制面板** → **文件服务** → **高级** 或 **Time Machine**
- 勾选 **「启用 Time Machine 备份」**
- 选择用于 Time Machine 的**共享文件夹**（建议单独建一个，如 `TimeMachine`）
- 确认该文件夹对备份用的 DSM 用户有**读写**权限

### 2. SMB 高级设置（重要）

- **控制面板** → **文件服务** → **SMB/AFP/NFS** → **高级**
- 建议设置：
  - **最大 SMB 协议**：SMB3
  - 勾选：**启用 Opportunistic Locking**
  - 勾选：**启用 SMB2 租约**（或 SMB2 lease）
  - 勾选：**启用 SMB durable handle**（持久句柄）
- 保存后如有需要可**重启 SMB 服务**

### 3. Bonjour 服务发现

- **控制面板** → **文件服务** → **高级** → **Bonjour**
- 启用 **Bonjour 打印机/文件共享发现**，这样 Mac 才能通过 `DiskStation.local` 发现 NAS

### 4. Time Machine 共享文件夹

- **禁用该共享文件夹的回收站**（避免与 Time Machine 的稀疏包冲突）
- 路径示例：`/volume1/TimeMachine` 或你当前使用的共享名

### 5. 空间与配额

- 可用空间建议 ≥ Mac 已用容量的 **1.5 倍**
- 若对用户做了**配额**，给该备份用户足够配额（建议 2–3 倍 Mac 已用容量）

---

## 二、Mac 端操作

### 1. 网络

- 确保 Mac 与 NAS 在**同一局域网**（家里 Wi-Fi/有线与 192.168.110.29 互通）
- 若 NAS 刚开机，等 2–3 分钟再试

### 2. 重新选择备份磁盘（推荐）

1. **系统设置** → **通用** → **时间机器**
2. 点击「共享 - DiskStation.local」右侧的 **✕** 移除该备份目标
3. 点击 **「+」** 重新添加备份磁盘
4. 选择 **DiskStation**（或 192.168.110.29）上用于 Time Machine 的共享文件夹
5. 输入 **DSM 用户名和密码**（与 DSM 登录一致）
6. 等待「正在连接备份磁盘…」完成后再看是否仍显示红点

### 3. 若仍报错：用 IP 代替 .local

- 在 Finder **前往** → **连接服务器**（⌘K）
- 输入：`smb://192.168.110.29/TimeMachine`（把 `TimeMachine` 换成你的共享名）
- 连接并勾选「在钥匙串中记住密码」
- 再在时间机器里添加备份磁盘，选择该已挂载的卷或通过「浏览」选同一共享

### 4. M1/M2/M3 Mac 首次备份慢

- 首次备份可能持续数小时，红点可能是「进行中」而非最终失败
- 可接**电源**、保持 Mac 与 NAS **不睡眠**，让备份跑完
- 若长时间卡住：移除该目标 → 在 NAS 上清空该 Time Machine 共享文件夹内容 → 再按上面「重新选择备份磁盘」添加一次

---

## 三、快速检查清单

| 检查项           | 位置           | 预期 |
|------------------|----------------|------|
| Time Machine 已启用 | DSM 文件服务   | 已勾选，并选对共享文件夹 |
| SMB 高级选项     | DSM SMB 高级   | SMB3、租约、持久句柄已开 |
| Bonjour          | DSM 文件服务高级 | 已启用 |
| 共享文件夹权限   | DSM 共享文件夹 | 备份用户可读写 |
| 回收站           | 该共享文件夹   | 已关闭 |
| Mac 与 NAS 同网 | —              | 能 ping 通 192.168.110.29 |
| 凭证             | Mac 钥匙串/重新输入 | 与 DSM 一致 |

---

## 四、仍失败时

- 在 **控制台.app** 中筛选「Time Machine」或「backupd」看具体错误
- DSM **日志中心** 查看文件服务 / SMB 相关日志
- 若 DSM 版本较老，可考虑升级到最新 DSM 7.x 再试

完成 NAS 端 1～4 项并做一次「重新选择备份磁盘」后，多数「等待完成第一次备份」+ 红点问题会消失或变为正常进行中状态。
