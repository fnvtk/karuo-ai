## 三、消息传输到「指定哪个号」的详细拆解

### 3.1 协议与入口（来自《存客宝技术架构》）

- **通道**：WebSocket  
- **地址**：`wss://s2.siyuguanli.com:9993`  
- **认证**：先通过 HTTP 获取 Token（`https://s2.siyuguanli.com:9991/token`），再在 WS 里发 `CmdSignIn`（带 `accessToken`、`accountId`、`client`）。

### 3.2 发送消息的命令：CmdSendMsg

| 参数 | 含义 | 说明 |
|:---|:---|:---|
| `wechatAccountId` | **用哪个微信号发** | 指定「发信号」；对应存客宝体系内的微信账号 ID |
| `toWxid` | **发给谁** | 目标微信 ID（好友或群） |
| `msgType` | 消息类型 | 文本/图片等类型标识 |
| `content` | 消息内容 | 与 msgType 对应的内容 |

即：**消息传输到指定号**在协议上已经支持两层「指定」：

1. **指定发信号**：`wechatAccountId` → 用哪个存客宝绑定的微信号发；  
2. **指定接收方**：`toWxid` → 发到哪个微信号/群。

### 3.3 消息流简图

```
外部系统 / 业务后台
        │
        ▼
  存客宝后端 (ThinkPHP)
        │
        │ WebSocket (CmdSendMsg)
        ▼
  代理/私域 SDK (wechatAccountId → 对应设备上的微信)
        │
        ▼
  目标微信 (toWxid)：好友或群
```

### 3.4 风控与限制（技术架构中已写）

- 发送间隔：如 3–8 秒随机间隔。  
- 单号每日上限：如 200 条。  
- 同一目标间隔：如 60 秒内不重复发同一对象。  

这些会约束「消息传输到指定号」的实际可用频率，对接时需遵守。
