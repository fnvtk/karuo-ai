### 三、内容结构
- **时间线/事件线推进**：按时间或事件发展顺序推进，逻辑清晰。

- **情感递进**：情绪自然流动，时有自省、感慨、幽默或犀利。

---

### 四、写作技巧
- **短句+空行**：每一句话后面空一行，段落之间也空行，增强可读性和节奏感。
- **口语化表达**：用词接地气，贴近生活。
- **对比反转**：善用对比、反转，突出矛盾和冲突。
- **穿插议论**：故事与议论交替，观点自然流露。
- **生活化细节**：如吃饭、出行、聊天、买卖、日常琐事等，增强真实感。
- **自嘲/幽默**：适当自嘲，增加亲和力。

---

- **真诚自然**：情感表达真实，不做作。
- **共鸣点**：挖掘普遍情感或经历，引发共鸣。
- **情绪递进**：情感有递进和高潮，结尾有余韵。

---

### 六、结尾处理

## 五、写作技巧库

### 5.1 开篇钩子（选一个）

| 类型 | 模板 | 示例 |
|:---|:---|:---|
| **冲突型** | {年份}，{重大事件}，{后果} | "2018年，电商公司倒闭，大量劳务纠纷。" |
| **自问型** | 你有没有发现，{普遍现象}？ | "你有没有发现，很多私域运营者每天都在追求新客、新流量，但回头一看……" |
| **场景型** | {时间}，{具体场景描述} | "昨晚，又是一场从下午三点五十五分开始的会议。" |
| **颠覆型** | 如果你以为{常见认知}，那你就错了。 | "如果你以为卖猫只是挑选品种、找到合适的买家，那你就错了。" |
| **观点型** | {激进观点，制造冲突} | "首先，得承认，社会上的规则真的是两套。" |

### 5.2 对话技巧

```markdown
正确示范：

"就是让客户知道我们的价值很高，也是这个意思。"

我当时就直接说了，这是价值定位的问题。

"你这个OKR，那我就，嗯，对，那我这个得回去再弄一下。"

---

错误示范：

我说："这是价值定位的问题，我们要让客户清晰地感受到产品价值。"他回答说："好的，我知道了。"

（太书面化，缺乏真实感）
```

### 5.3 数据植入

```markdown
必须包含的数据类型：
- 转化率：如"转化率只有可怜的百分之五左右"
- 客户数：如"聊了大概两百九十几个客户"
- 金额：如"带来20万的产值"
- 比例：如"删除和无回复的比例太大了，大概两百八十个"
- 时间：如"三个月的目标"

数据来源：
- 用户提供的聊天记录（优先）
- 会议记录
- 后台数据
- 行业报告（如艾瑞咨询）

⚠️ 不要自己编造数据！
```

### 5.4 隐性植入（会员/合作伙伴）

```markdown
正确示范：

"有一个老板，做高端护肤品的。"

"他开始重新梳理客户路径。"

"有一家门店，在引流阶段吸引了上千名客户。"

---

错误示范：

"我的会员小王最近遇到一个问题。"

"作为我的合作伙伴，他……"

（太直白，失去真实感）
```

### 5.5 粗体使用规则

```markdown
粗体用于：
1. 核心观点总结
2. 转折性语句
3. 行动号召

示例：
**价值定位是关键，几十块和几万块，客户感知天壤之别。**

**复制和放大，是实现规模化的核心。**

**AI不会取代所有程序员，但一定会取代那些只会"搬砖"的程序员。**
```

## 十二、常见问题

**Q1: 多场景怎么切换？**

用分割线 `---` 分隔不同场景，每个场景保持独立的时间线或逻辑线。

**Q2: 数据不够怎么办？**

优先使用模糊表达："大概""将近""超过"，但不要凭空编造具体数字。

**Q3: 如何体现会员关系？**

用"有一个老板""我认识的一个朋友""之前合作的一家企业"等隐性表达。

**Q4: 文章太短怎么办？**

增加：对话细节、心理描写、案例补充、方法论拆解。

## 一、核心风格特征

### 1.1 句式特点

| 特点 | 说明 | 示例 |
|:---|:---|:---|
| **短句主导** | 每句不超过20字 | "一路走来，踩了无数的坑。" |
| **单句成段** | 一句话就是一段 | "尤其是在私域这块，流量起来了，团队架构反倒成了新的难题。" |
| **空行分隔** | 每句话后空一行 | （全部文章遵循） |
| **口语化** | 贴近日常说话 | "搞个阿米巴出来。""降成本呗。" |

### 1.2 语言风格

| 风格 | 频率 | 示例 |
|:---|:---|:---|
| **直接犀利** | 高频 | "这场面试像是一场荒诞的闹剧。" |
| **自问自答** | 高频 | "为什么？因为你没有做好复购这件事。" |
| **反问加强** | 中频 | "你有没有发现……？" |
| **对比反转** | 中频 | "不是人多力量大，而是人对力量大。" |
| **自嘲幽默** | 低频 | "你太固执……" |

### 1.3 结构特点

```
开篇：冲突/悬念/场景 → 快速切入

主体：
├── 小主题1（隐藏标题）
│   ├── 事件/对话
│   ├── 数据支撑
│   └── 观点总结（粗体）
├── 小主题2
│   ├── ...
│   └── ...
└── 小主题N

结尾：升华/感恩/行动号召
```

## 三、对话使用分析

### 3.1 对话特点

- **口语化**：保留"嗯""呃""那个"等语气词
- **不完整句**：还原真实对话的断续感
- **带情绪**：通过用词体现说话人状态

### 3.2 对话示例库

```markdown
1. 汇报型对话：
"就是让客户知道我们的价值很高，也是这个意思。"

2. 思考型对话：
"你这个OKR，那我就，嗯，对，那我这个得回去再弄一下。"

3. 建议型对话：
"核心是什么？就是矩阵，把那些IP游戏的IP签下来。"

4. 总结型对话：
"只要有单，比如说有单，你比如说给小北他这种，那他反正就很乐意去配合。"
```

| **金额/产值** | 中频 | "20万的产值""客单价1000元" |
| **时间目标** | 中频 | "三个月的目标""两个月内" |
| **比例对比** | 低频 | "删除和无回复的比例太大了，大概两百八十个" |

### 4.2 数据表达方式

```markdown
精确表达：
"转化率从5%提升到了35%"

模糊表达：
"大概两百九十几个客户"

对比表达：
"10万个用户中，只有10个真正下单"
"同样的10万块，吸引来了1000个真正有消费意愿的用户"
```

## 五、观点呈现分析

### 5.1 粗体使用规律

粗体用于：
1. **核心观点总结**
2. **转折性判断**
3. **行动号召**

### 5.2 观点表达模板

```markdown
类型1 - 定义式：
**{概念}，不是{表面理解}，而是{深层理解}。**
示例：**复购，不是简单的老客户回头，而是信任的延续。**

类型2 - 判断式：
**{主语}是{核心判断}。**
示例：**价值定位是关键，几十块和几万块，客户感知天壤之别。**

类型3 - 否定式：
**{错误做法}不会{好结果}，{正确做法}才会{好结果}。**
示例：**AI不会取代所有程序员，但一定会取代那些只会"搬砖"的程序员。**

类型4 - 强调式：
**{动作}，是{意义}的核心。**
示例：**复制和放大，是实现规模化的核心。**
```

def markdown_to_blocks(content):
    """将Markdown内容转换为飞书文档块（简化版）"""
    # 简化处理：直接作为文本块插入
    # 实际使用时可以解析Markdown格式
    blocks = []
    lines = content.split('\n')
    
    for line in lines:
        if line.strip():
            # 处理标题
            if line.startswith('# '):
                blocks.append({
                    'block_type': 1,  # heading1
                    'heading1': {
                        'elements': [{'text_run': {'content': line[2:]}}]
                    }
                })
            elif line.startswith('## '):
                blocks.append({
                    'block_type': 2,  # heading2
                    'heading2': {
                        'elements': [{'text_run': {'content': line[3:]}}]
                    }
                })
            elif line.startswith('### '):
                blocks.append({
                    'block_type': 3,  # heading3
                    'heading3': {
                        'elements': [{'text_run': {'content': line[4:]}}]
                    }
                })
            elif line.startswith('---'):
                blocks.append({
                    'block_type': 22  # divider
                })
            elif line.startswith('**') and line.endswith('**'):
                # 粗体文本
                blocks.append({
                    'block_type': 2,  # text
                    'text': {
                        'elements': [{
                            'text_run': {
                                'content': line.strip('*'),
                                'text_element_style': {'bold': True}
                            }
                        }]
                    }
                })
            else:
                # 普通段落
                blocks.append({
                    'block_type': 2,  # text
                    'text': {
                        'elements': [{'text_run': {'content': line}}]
                    }
                })
    
    return blocks

        if line.startswith('# '):
            line = line[2:]  # 去掉标题标记
        elif line.startswith('## '):
            line = line[3:]
        elif line.startswith('### '):
            line = line[4:]
        
        # 创建文本块
        blocks.append({
            "block_type": 2,
            "text": {
                "style": {},
                "elements": [{
                    "text_run": {
                        "content": line,
                        "text_element_style": {"bold": bold}
                    }
                }]
            }
        })
    
    return blocks

def send_blocks(doc_token, blocks):
    """发送块到文档"""
    result = api_request('POST', f'/docx/v1/documents/{doc_token}/blocks/{doc_token}/children', {
        'children': blocks
    })
    return result and result.get('code') == 0

# ============== 主函数 ==============

def upload_article(content, title, category='金融'):
    """
    上传文章到飞书知识库
    
    参数:
        content: 文章内容（Markdown格式）
        title: 文章标题
        category: 分类（金融/私域干货等）
    
    返回:
        {'success': bool, 'url': str, 'message': str}
    """
    # 获取知识库配置
    wiki_config = CONFIG['WIKI_SPACES'].get(category, CONFIG['WIKI_SPACES']['金融'])
    space_id = wiki_config['space_id']
    parent_node = wiki_config['parent_node']
    
    if not space_id or not parent_node:
        return {'success': False, 'url': '', 'message': f'分类 {category} 未配置'}
    
    # 检查是否有同名文章
    print(f"🔍 检查是否有同名文章: {title}")
    existing = find_existing_article(space_id, parent_node, title)
    
    if existing:
        print(f"⚠️ 发现同名文章，将创建新版本")
        title = f"{title}（更新版）"
    
    # 创建新文档
    print(f"📝 创建文档: {title}")
    doc_info = create_wiki_node(space_id, parent_node, title)
    
    if not doc_info:
        return {'success': False, 'url': '', 'message': '创建文档失败'}
    
    doc_token = doc_info['obj_token']
    node_token = doc_info['node_token']
    
    # 添加固定结尾链接
    if CONFIG['FOOTER_LINK'] not in content:
        content = content.rstrip() + f"\n\n---\n\n{CONFIG['FOOTER_LINK']}"
    
    # 转换为飞书块格式
    blocks = markdown_to_blocks(content)
    
    # 分批上传
    print(f"📤 上传内容（共 {len(blocks)} 块）...")
    batch_size = 10
    success_count = 0
    
    for i in range(0, len(blocks), batch_size):
        batch = blocks[i:i+batch_size]
        if send_blocks(doc_token, batch):
            success_count += len(batch)
            print(f"  ✅ 已上传 {success_count}/{len(blocks)} 块")
        else:
            print(f"  ❌ 上传失败")
        time.sleep(0.3)
    
    url = f"https://cunkebao.feishu.cn/wiki/{node_token}"
    print(f"\n✅ 上传完成!")
    print(f"📄 飞书链接: {url}")
    
    return {'success': True, 'url': url, 'message': '上传成功'}

def api_request(method, endpoint, data=None):
    """API请求"""
    url = f"https://open.feishu.cn/open-apis{endpoint}"
    if data:
        data = json.dumps(data).encode('utf-8')
    req = urllib.request.Request(url, data=data, method=method)
    req.add_header('Authorization', f'Bearer {USER_TOKEN}')
    req.add_header('Content-Type', 'application/json')
    try:
        resp = urllib.request.urlopen(req, timeout=30)
        return json.loads(resp.read().decode('utf-8'))
    except Exception as e:
        return {'code': -1, 'msg': str(e)}

def get_child_nodes(parent_node_token):
    """获取子节点列表"""
    result = api_request('GET', f'/wiki/v2/spaces/{SPACE_ID}/nodes?parent_node_token={parent_node_token}&page_size=50')

def get_doc_content(obj_token, obj_type='docx'):
    """获取文档内容"""
    if obj_type == 'docx':
        result = api_request('GET', f'/docx/v1/documents/{obj_token}/raw_content')
    elif obj_type == 'doc':
        result = api_request('GET', f'/doc/v2/{obj_token}/raw_content')
    else:
        result = api_request('GET', f'/docx/v1/documents/{obj_token}/raw_content')
    
    if result and result.get('code') == 0:
        return result.get('data', {}).get('content', '')
    return None

def safe_filename(name):
    """生成安全的文件名"""
    # 移除或替换不安全字符
    name = re.sub(r'[<>:"/\\|?*]', '', name)
    name = name.strip()
    if not name:
        name = 'untitled'
    return name[:100]  # 限制长度

def download_node(node, parent_dir, depth=0):

    title = node.get('title', 'untitled')
    node_token = node.get('node_token')
    obj_token = node.get('obj_token')
    obj_type = node.get('obj_type', 'docx')
    has_child = node.get('has_child', False)
    
    indent = '  ' * depth
    safe_title = safe_filename(title)
    
    if has_child:

        print(f"{indent}📁 {title}/")
        
        # 获取并下载子节点
