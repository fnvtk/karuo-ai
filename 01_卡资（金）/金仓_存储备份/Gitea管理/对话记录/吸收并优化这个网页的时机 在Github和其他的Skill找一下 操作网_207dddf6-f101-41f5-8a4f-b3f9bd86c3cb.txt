## 二、Playwright MCP（Cursor 内操作网页）

### 2.1 来源与特点

- **仓库**：https://github.com/microsoft/playwright-mcp （Microsoft 官方，2.6万+ star）
- **特点**：用**无障碍树（accessibility snapshot）**，不依赖截图/视觉模型；确定性操作（按 role/name 点）；Token 友好。
- **对比**：写代码/高吞吐用 [Playwright CLI + SKILLS](https://github.com/microsoft/playwright-cli) 更省 Token；MCP 适合「对话里持续操作浏览器、保持上下文」的场景。

### 2.2 Cursor 配置（核心）

在 Cursor 中：**Settings → MCP → Add new MCP Server**。

- **Name**：`playwright`（或自定）
- **Type**：command
- **Command**：`npx @playwright/mcp@latest`

或直接改 MCP 配置文件，加入：

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
```

可选参数（按需加在 `args` 里）：

| 参数 | 说明 | 示例 |
|:---|:---|:---|
| `--headless` | 无头模式（默认 headed） | `"--headless"` |
| `--browser` | 浏览器：chrome/firefox/webkit/msedge | `"--browser", "chromium"` |
| `--viewport-size` | 视口 | `"--viewport-size", "1280x720"` |
| `--timeout-navigation` | 导航超时(ms) | `"--timeout-navigation", "60000"` |
| `--storage-state` | 登录态文件 | `"--storage-state", "/path/to/storage.json"` |
| `--isolated` | 不持久化 profile，每次干净会话 | `"--isolated"` |

### 2.3 常用 MCP 工具（对话时可用）

| 工具 | 作用 |
|:---|:---|
| `browser_navigate` | 打开 URL |
| `browser_snapshot` | 获取当前页无障碍快照（优先于截图） |
| `browser_click` | 点击（需 ref，来自 snapshot） |
| `browser_fill_form` | 填表 |
| `browser_type` | 输入文字 |
| `browser_take_screenshot` | 截图 |
| `browser_handle_dialog` | 处理 alert/confirm/prompt |

**重要**：先 `browser_navigate` 再 `browser_snapshot`，再根据 snapshot 里的 ref 做 `browser_click` 等；lock/unlock 顺序见 Cursor MCP 说明。

### 2.4 核心调用顺序（防坑）

1. **先 navigate，再 lock**：有 tab 后再 lock，不能先 lock 再 navigate。
2. **操作前先 snapshot**：拿到当前结构再点击，避免 stale ref。
3. **等待策略**：用 2～3 秒短等 + 多次 snapshot 判断是否加载完，避免单次长等。

## 五、常见坑与对策

| 坑 | 对策 |
|:---|:---|
| 验证码用了旧短信 | 所有取码处传 `--since` 命令发出时间，只用之后收到的 |
| 页面未加载就点击 | 先 `wait_until="networkidle"` 或短等 + 判元素可见再点 |
| 选择器失效（高德改版） | 用多种 fallback：`nth(1)`、`placeholder*="验证码"`、`name*="code"` |
| 滑块无法自动过 | 用 `--headed` 人工过一次，或接 2captcha；严站可能需人工 |
| MCP 先 lock 再 navigate | 顺序改为：navigate → lock → 操作 → unlock |
| 无 API 只能看屏操作 | 走金链「截屏操作」：截屏→分析→再操作 |

**操作步骤**：

## 5. 页面改版导致选择器找不到（如高德控制台）

**操作步骤**：

1. 用 `--headed` 运行脚本，让浏览器停在你希望自动点击的页面；用开发者工具（F12）查看按钮的 `id`、`class`、`data-*`、文案。
2. 打开 `run_amap_key.py`（或对应平台脚本），找到「获取验证码」「登录」「添加 Key」等 locator，按新页面结构改：
   - 优先用 `has-text("按钮文案")`、`placeholder`、`name` 等稳定属性；
   - 用 `nth(0)`、`nth(1)` 区分同类型多个输入框；
   - 多个备选选择器用 `for sel in [...]` 依次 try。
3. Key 提取逻辑在 `_create_key_and_exit`：若 Key 不在 `document.body.innerText` 的 32 位十六进制里，需根据截图和 DOM 改 `page.evaluate` 里的正则或选择器。

## 用网页操作插件在美团订「附近最热门」的步骤

### 1. 确认插件可用

- **若已装 Playwright MCP**：在 Cursor 里新开一个对话，说：「打开 https://waimai.meituan.com/ 美团外卖」
- **若用 Cursor 自带的 Browser MCP**：在同一对话里说：「用浏览器打开美团外卖」或「navigate to https://waimai.meituan.com/」

先确认能打开美团外卖首页，再进行下面步骤。

### 2. 打开美团外卖首页

对 AI 说（任选一种说法即可）：

- 「打开美团外卖首页」
- 「Navigate to https://waimai.meituan.com/」

### 3. 搜「附近最热门 / 最多人想吃」

页面打开后，对 AI 说（任选一种）：

- 「在搜索框里输入“附近热门”或“人气美食”，然后点搜索」
- 「搜索：附近 热门 最多人想吃」
- 「点页面上的搜索框，输入“热门”或“人气”，然后点搜索/回车」

美团网页版通常会有「搜索框」或「热门/人气」入口，用上面任一句即可让插件帮你完成「搜索附近最热门、最多人想吃」这类内容。

### 4. 选一个「最热门」的店/商品

搜出列表后，对 AI 说：

- 「选第一个/第二个商家点进去」或  
- 「选销量最高或评分最高的那家店点进去」

如需更具体，可以说：

- 「点进列表里第一个商家」
- 「点「人气榜」或「销量高」的那一家」

### 5. 选菜并加购

进入店铺后，对 AI 说：

- 「选一道热门/推荐菜加入购物车」
- 「把第一个推荐菜或销量最高的菜加入购物车」

### 6. 下单（需你本人确认）

- 对 AI 说：「去结算」或「打开购物车，去结算」  
- **收货地址、支付方式**需要你本人在页面上确认或登录美团账号后再付；若未登录，按页面提示用手机号/验证码登录后再继续。
