## 一、Auto 模式到底用哪个模型？

### 官方定义

**Auto** 是 Cursor 的**自动模型选择**功能：
- Cursor 根据**当前任务**和**服务负载**，自动选择最适合、最稳定的模型
- 官方**不公开**具体会选哪个模型
- 会检测输出质量下降时**自动切换**到其他模型

### 实际行为（社区反馈）

- Auto **始终使用高端模型**（Claude Sonnet、Gemini Pro 等）
- **不会**使用小模型（如 GPT-4o-mini、cursor-small）
- 具体可能是：Claude 4.5 Sonnet、Gemini 3 Flash/Pro、GPT-5.2 等，由 Cursor 内部调度

### 规则总结

| 维度 | 说明 |
|:---|:---|
| **选择逻辑** | 任务难度 + 当前负载 + 可靠性 |
| **模型池** | 高端模型（Sonnet、Pro、Flash 等） |
| **切换** | 检测到质量下降时自动换模型 |
| **可预测性** | 低——用户无法指定具体模型 |

---

## 二、Auto 的付费形式

### 计费方式（按 Token）

Auto 使用**固定费率**，不按具体模型计费：

| 类型 | 价格（美元/百万 Token） |
|:---|:---|
| **Input + Cache Write** | $1.25 |
| **Output** | $6.00 |
| **Cache Read** | $0.25 |

### 与其他模型对比（参考）

| 模型 | Input | Output | 说明 |
|:---|:---|:---|:---|
| **Auto** | $1.25 | $6.00 | 固定价，Cursor 内部调度 |
| Claude 4.5 Sonnet | $3.00 | $15.00 | 比 Auto 贵 |
| Claude 4.5 Opus | $5.00 | $25.00 | 最贵 |
| Gemini 3 Flash | $0.50 | $3.00 | 比 Auto 便宜 |
| Composer 1 | $1.25 | $10.00 | 接近 Auto |

Auto 的 Input 便宜，Output 中等，整体比直接选 Opus 省不少。

---

## 三、$20 Pro 套餐能用多久？

### 套餐内容

- **Pro**：$20/月
- **包含**：$20 的 API 用量 + **额外 bonus 用量**（不固定）
- **Auto**：用量计入这 $20，但**用完后仍可继续用 Auto**（见下文）

### 按官方估算（基于中位用量）

| 若选固定模型 | 约等于请求数 |
|:---|:---|
| Claude Sonnet 4 | ~225 次 |
| Gemini | ~550 次 |
| GPT-4.1 | ~650 次 |

### 按 Auto 费率粗算

假设典型对话：**5K input + 2K output**（约 7K tokens）

- Input：5K × $1.25/1M ≈ $0.00625  
- Output：2K × $6/1M ≈ $0.012  
- **单次约 $0.018**（约 ¥0.13）

**$20 ÷ $0.018 ≈ 1,100 次** 典型对话（仅作参考，实际看上下文长短）

### 用户类型参考（官方）

| 类型 | 月用量 |
|:---|:---|
| 每日 Tab 补全为主 | 通常不超 $20 |
| 轻度 Agent | 多数不超 $20 |
| 每日 Agent | 约 $60–100 |
| 重度 / 多 Agent | 常 $200+ |

## 四、用尽 $20 之后会怎样？

### 官方说明

1. **Auto 仍可用**：用尽 $20 后，**Auto 模式不会被停用**
2. **其他模型**：选 Claude Opus、Sonnet 等固定模型时，会提示超限
3. **继续使用的方式**：
   - 继续用 **Auto**（不额外扣费，或按官方后续政策）
   - 开通 **按需付费**：按 API 价继续用任意模型
   - **升级套餐**：Pro Plus $60（约 3× 用量）、Ultra $200（约 20× 用量）

### 注意

- 2025 年 6 月后，Auto 的「无限」政策有过调整
- 建议在 [Cursor Dashboard](https://cursor.com/dashboard?tab=usage) 查看当前用量和限制

## 五、Opus Max 为什么「不能用了」却「还能用」？

### 可能指什么

1. **Claude Opus 4 / 4.5**：高端模型，仍在 Cursor 中提供  
2. **Max Mode**：扩展上下文（如 1M tokens），部分模型支持  
3. **旧名称**：如「Opus Max」可能是旧 UI 里的名字，现已改为「Claude 4.5 Opus」等

### 为什么感觉「不能用了」

- **UI 调整**：模型列表改名或重组，找不到原来的「Opus Max」
- **用量限制**：$20 用完后，直接选 Opus 会被限流或提示超限
- **Max Mode**：需要单独开启，或只在部分模型上可用

### 为什么「还能用」

1. **Auto 模式**：Auto 可能仍会调用 Opus 级模型，你看到的是 Auto 的输出  
2. **历史会话**：之前开的对话还在用旧模型，未刷新  
3. **Bonus 用量**：Pro 的 bonus 可能让你还能偶尔用到高端模型  
4. **缓存/降级**：系统可能用缓存或降级策略，体验上仍可用

### Claude 4.5 Opus 当前状态（文档）

- **仍可用**，未下线  
- 可能需要：Pro 订阅 + 在设置中开启 usage-based pricing  
- 价格：Input $5/1M，Output $25/1M，比 Auto 贵不少
