> 参考来源：GitHub [yusufkaraaslan/Skill_Seekers](https://github.com/yusufkaraaslan/Skill_Seekers)。Skill Seekers 将文档站、GitHub 仓库、PDF 转为 Claude AI skills，带冲突检测与多平台导出。此处仅抽象与「规范生成」相关的通用原则，供卡若AI 生成/优化 Skill 时复用。

## 1. 来源类型（输入）

| 类型 | 说明 | 在卡若AI 中的对应 |
|:---|:---|:---|
| 文档站 | 官方文档、教程站 | 可抓取或手动整理为 references |
| 代码仓库 | GitHub 等，含 README/代码/Issues | 可引用仓库链接或摘录到 references |
| PDF | 手册、白皮书 | 摘录要点放入 references 或 SKILL 正文 |
| 多源合并 | 文档 + 代码同时用，并做冲突检测 | 可选：文档与实现不一致时做对比与标注 |

卡若AI 内不强制实现「自动抓取」，但生成 Skill 时可明确**输入来源**（文档/仓库/PDF/人工），并在「能力说明」中写清。

## 2. 输出结构（统一）

Skill Seekers 产出通常包含：

- **SKILL.md**：主入口，含 YAML frontmatter + 说明与指令。
- **references/**：分主题的参考文档（按类别组织）。
- **scripts/**：可选，可执行脚本或工具。
- **Router/Hub**：多子 Skill 时有一个主 Skill 做路由到子 Skill。

卡若AI 已采用相同思路：

- 每个 Skill 一个 `SKILL.md`，必含 frontmatter（name, description, triggers, owner, group）。
- 正文结构固定为五段：能力说明、执行流程、常用命令、相关文档、常见问题。
- references 可选；子 Skill 通过 SKILL_INDEX 的关键词路由到对应路径。

## 3. Frontmatter 与元数据

Skill Seekers 与 Claude/GitHub Agent Skills 常用：

- `name`：Skill 名称。
- `description`：一句话描述 + 触发/使用场景。

卡若AI 扩展为：

- `name`、`description`、`triggers`（关键词列表）、`owner`（执行人）、`group`（金/水/木/火/土），便于**路由与归属**。

生成规范 Skill 时：**必填上述五项**，无 owner 时填「待分配」。

## 4. 质量与冲突思路（可选）

Skill Seekers 的特点：

- **冲突检测**：文档描述与代码实现不一致时，并排展示并标注（缺失于代码/缺失于文档/签名不一致等）。
- **质量提升**：通过 AI 增强步骤，把简要说明扩展为更完整的教程（步骤、故障排查、前置条件、后续步骤）。

在卡若AI 中可选用：

- **一致性检查**：若 Skill 既引用文档又引用脚本/代码，可提醒「文档与脚本是否一致」或列出差异。
- **增强**：对「执行流程」「常见问题」做补充，使每条可操作、可验证。

不强制每个 Skill 都做冲突检测，但「规范 Skill 生成器」在优化已有 Skill 时，可把「文档与实现一致性」作为可选检查项。
