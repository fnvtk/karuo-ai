function generateInviteCode(): string {
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789'  // 去掉容易混淆的 I/O/0/1
  let code = ''
  for (let i = 0; i < 6; i++) {
    code += chars.charAt(Math.floor(Math.random() * chars.length))
  }
  return code
}

      }

  async getAllCommissions(page = 1, limit = 20, filters: any = {}) {

**1. `lib/models/Distribution.ts`（新建）** - 分销核心模型
- **Commission（佣金记录）**：记录每笔订单产生的佣金，支持一级/二级分销，状态流转（待结算→已结算→已取消）
- **Withdrawal（提现记录）**：记录用户提现申请，支持微信/支付宝/银行卡，审核流程（待审核→通过/拒绝→完成）
- **DistributionConfig（分销配置）**：全局分销开关、佣金比例、产品专属比例、提现门槛、自动结算天数、分销等级门槛

**2. `lib/models/User.ts`（增强）** - 用户模型新增分销字段
- `inviteCode`：6位唯一邀请码（自动生成）
- `inviterId` / `inviterCode`：邀请人关系链
- `distributionLevel`：分销等级（normal/silver/gold/diamond）
- `totalEarnings` / `withdrawableBalance` / `frozenBalance` / `totalWithdrawn`：收益四维度
- `teamSize` / `totalTeamSize` / `totalSales`：团队数据
- `isDistributor` / `distributorSince`：分销商标识

**3. `lib/models/Order.ts`（增强）** - 订单模型新增分销追踪
- `referrerId` / `referrerCode`：一级推荐人追踪
- `level2ReferrerId`：二级推荐人追踪
- `commissionAmount` / `level2CommissionAmount`：佣金金额
- `commissionStatus`：佣金状态
- `source`：订单来源（direct/invite/scan/share）

### 二、服务层（1个文件）

**`lib/database.ts`（增强）** - 新增 `distributionService`
- `getConfig()` / `updateConfig()` - 配置管理
- `bindInviter()` - 绑定邀请关系（防循环、防自绑、防重绑）
- `calculateCommission()` - **核心：支付成功后自动计算一级+二级佣金**
- `settleCommission()` / `autoSettleCommissions()` - 佣金结算
- `getUserCommissions()` / `getUserDistributionStats()` / `getTeamMembers()` - 用户端查询
- `createWithdrawal()` / `reviewWithdrawal()` / `getWithdrawals()` - 提现全流程
- `getOverviewStats()` / `getDistributors()` / `getAllCommissions()` - 管理后台查询

### 三、支付分佣打通（2个文件）

**`app/api/payment/notify/route.ts`** - 支付成功后自动触发：
```
用户扫脸/做测试 → 支付 → 支付回调 → 自动计算佣金 → 佣金入账
```

**`app/api/payment/create/route.ts`** - 创建订单时携带推荐人信息

### 完整分销流程
```
用户A 分享邀请码 → 用户B 绑定 → B 扫脸/做测试 → B 支付 
→ 支付回调触发 → 自动计算佣金（A获得20%一级佣金）
→ 如果A也有上级C → C获得10%二级佣金
→ 7天后自动结算 → 余额可提现 → 管理后台审核 → 到账
```
