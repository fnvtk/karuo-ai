## 三、本地模型是什么？怎么用？

### 3.1 本地模型 = Ollama + qwen2.5

- **Ollama**：一个软件，让你在 Mac 上跑 AI 模型（类似「本机 ChatGPT」）
- **qwen2.5**：阿里开源的轻量模型，你本机已安装两个版本：
  - `qwen2.5:0.5b`（397MB）— 超轻量，快速
  - `qwen2.5:1.5b`（986MB）— 稍强，代码辅助

### 3.2 本机现状（已确认）

```bash
# 你电脑上已安装
ollama list
# 输出：
# qwen2.5:0.5b      397 MB
# qwen2.5:1.5b      986 MB
# nomic-embed-text  274 MB（嵌入用）
```

### 3.3 本地模型什么时候被调用？

| 场景 | 谁调用 | 怎么调用 |
|:---|:---|:---|
| **任务拆解** | `task_decomposer` 脚本 | `from _共享模块.local_llm import analyze_task` |
| **文本摘要/分类** | 卡若AI 的 Skill | `summarize()`, `classify()` |
| **神射手热点查询** | `hotspot_query.py` | 断网时切换本地模型 |
| **你手动测试** | 命令行 | `echo "问题" \| ollama run qwen2.5:1.5b` |

**重要**：Cursor 主对话（你现在这个聊天）**不会**自动调用本地模型。Cursor 只能用云端 Claude。

### 3.4 本地模型成本

- **API 费**：0 元
- **电费**：MacBook 跑 qwen2.5 约 15–25W，1 分钟 ≈ 0.3 Wh ≈ ¥0.0002（按 0.6 元/度）
- **结论**：可以忽略不计

## 五、卡若AI 规则里的「智能模型分配」是什么意思？

规则里写的是：**简单任务用本地模型，复杂任务用高级模型**。

这里的「简单任务」指的是：

- 当**脚本/Skill 执行**时（如任务拆解、摘要、分类）
- 那些脚本会去调用 `ollama run qwen2.5`
- 而不是让 Cursor 主对话去调用

**所以**：你在 Cursor 里跟卡若AI 聊天，**始终用的是 Claude**。  
「智能模型分配」只影响**脚本内部**是否用本地模型，不影响 Cursor 主对话。
