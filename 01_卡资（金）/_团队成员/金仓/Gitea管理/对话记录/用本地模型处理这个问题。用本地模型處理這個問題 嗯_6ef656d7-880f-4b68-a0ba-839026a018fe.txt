  contents: # Cursor "Unable to resolve your shell environment in a reasonable time" 处理

## 现象
弹窗：Unable to resolve your shell environment in a reasonable time. Please review your shell configuration and restart.  
按钮：Learn More / Configure / Restart

## 原因
Cursor 启动集成终端时会 source `~/.zshrc`，若 zsh 交互式启动时间过长（本机实测约 6.4s），超过 Cursor 的等待时间就会报错。常见慢项：
- `pyenv init`
- `compinit`（尤其配合 Docker 等大量补全）
- `source <(openclaw completion --shell zsh)` 等子进程

## 处理（已落地）
在 `~/.zshrc` 最开头增加「Cursor/VS Code 快速路径」：当 `TERM_PROGRAM=Cursor` 或 `vscode` 时只做必要 PATH + 单次 pyenv init + 常用 alias，然后 `return`，不执行 compinit、openclaw completion 等。  
效果：Cursor 下 zsh 启动从约 6.4s 降到约 0.1s。

## 验证
```bash
# 慢路径（普通终端）
time (zsh -i -c 'echo ok')   # 约 6s+

# 快路径（模拟 Cursor）
time (TERM_PROGRAM=Cursor zsh -i -c 'echo ok')   # 约 0.1s
```

## 后续
若仍超时，可检查是否有其他 profile（如 `.zprofile`、oh-my-zsh）在 Cursor 下被加载；或临时在 Cursor 设置中指定使用「Login Shell」/ 自定义 shell 参数（一般不必要）。
