## Hooks 是什么？

**简单说：Hooks 就是“自动触发器”**，在 Cursor 的特定操作时自动执行你的自定义脚本或命令。

### 具体能干什么？

1. **保存文件后自动格式化代码**
   - 保存时自动运行 `prettier` 或 `eslint`

2. **提交代码前自动检查**
   - Git commit 前自动跑测试，有问题就阻止提交

3. **项目打开时自动执行**
   - 打开项目时自动安装依赖、启动服务等

4. **AI 代理运行前后执行**
   - 代理运行前备份数据，运行后清理临时文件

### 为什么有用？

- **自动化重复操作**：不用每次都手动执行
- **减少错误**：提交前自动检查，避免把错误代码推上去
- **统一流程**：团队用同一套 Hooks，保证流程一致

### 实际例子

比如你想在每次保存 `.md` 文件时自动格式化，可以这样配置：

```bash
# Hook: 文件保存后
触发条件: *.md 文件保存
执行命令: prettier --write {file_path}
```

这样每次保存 Markdown 文件，代码会自动格式化。

## 📝 详细说明

### format-markdown.sh
**功能**：
- 使用 `prettier` 格式化 Markdown
- 检查章节结构是否符合「写书规则」
- 检查是否有「智能追问五问」标记

**检查项**：
- ✅ 标题层级是否正确（# → ## → ###）
- ✅ 是否有「一句话定位」
- ✅ 是否有「情绪张力」标记
- ✅ 是否有「招商价值」说明
- ✅ 是否有「主线逻辑」
- ✅ 是否有「故事素材」引用

### check-code-comments.sh
**功能**：
- 检查代码文件是否有中文注释
- 检查函数/类是否有注释说明

**规则**：
- Python: 函数必须有 `"""中文注释"""` 或 `# 中文注释`
- TypeScript/JavaScript: 函数必须有 `/** 中文注释 */` 或 `// 中文注释`
- Java: 类和方法必须有 `/** 中文注释 */`

### check-sensitive-info.sh
**功能**：
- Git commit 前检查是否有敏感信息泄露
- 检查 API 密钥、密码、Token 等

**检查项**：
- ❌ API Key / Secret
- ❌ 密码（password, pwd）
- ❌ Token（ghp_, sk-, v1:）
- ❌ 数据库连接字符串
- ❌ 支付密钥

### format-memory.py
**功能**：
- 自动去重（同一天相同条目）
- 格式化时间戳
- 检查文档大小（超过 1M 自动清理旧条目）
- 保持文档结构

**规则**：
- 按日期分类：`### YYYY-MM-DD`
- 时间戳格式：`- [HH:MM] 内容`
- 自动去重：同一天相同内容只保留一条
- 大小限制：超过 1M 删除最旧条目

### auto-read-memory.sh
**功能**：
- 项目打开时自动读取记忆文档
- 输出关键信息摘要

# 代码注释检查 Hook
# 触发时机：保存代码文件时
# 功能：检查是否有中文注释

FILE_PATH="$1"

if [ -z "$FILE_PATH" ]; then

echo "🔍 检查代码注释: $FILE_PATH"

# 获取文件扩展名
EXTENSION="${FILE_PATH##*.}"

# 根据文件类型检查注释
case "$EXTENSION" in
    py)
        # Python: 检查函数是否有中文注释
        FUNCTIONS=$(grep -n "^def " "$FILE_PATH" | head -10)
        if [ ! -z "$FUNCTIONS" ]; then
            echo "$FUNCTIONS" | while IFS= read -r line; do
                LINE_NUM=$(echo "$line" | cut -d: -f1)
                FUNC_NAME=$(echo "$line" | grep -oE "def [a-zA-Z_][a-zA-Z0-9_]*" | cut -d' ' -f2)
                
                # 检查函数下方3行内是否有中文注释
                NEXT_LINES=$(sed -n "${LINE_NUM},$((LINE_NUM+3))p" "$FILE_PATH")
                if ! echo "$NEXT_LINES" | grep -qE "(#|""\"\"\"|''').*[\u4e00-\u9fa5]"; then
                    echo "⚠️  第 ${LINE_NUM} 行：函数 $FUNC_NAME 缺少中文注释"
                fi
            done
        fi
        ;;
    ts|tsx|js|jsx)
        # TypeScript/JavaScript: 检查函数是否有中文注释
        FUNCTIONS=$(grep -nE "^(export )?(function |const |async function |class )" "$FILE_PATH" | head -10)
        if [ ! -z "$FUNCTIONS" ]; then
            echo "$FUNCTIONS" | while IFS= read -r line; do
                LINE_NUM=$(echo "$line" | cut -d: -f1)
                
                # 检查函数上方3行内是否有中文注释
                PREV_LINES=$(sed -n "$((LINE_NUM-3)),${LINE_NUM}p" "$FILE_PATH")
                if ! echo "$PREV_LINES" | grep -qE "(//|/\*).*[\u4e00-\u9fa5]"; then
                    echo "⚠️  第 ${LINE_NUM} 行：函数/类缺少中文注释"
                fi
            done
        fi
        ;;
    java)
        # Java: 检查类和方法是否有中文注释
        CLASSES=$(grep -nE "^(public |private |protected )?class " "$FILE_PATH")
        METHODS=$(grep -nE "^(public |private |protected ).*\(.*\)" "$FILE_PATH" | head -10)
        
        if [ ! -z "$CLASSES" ]; then
            echo "$CLASSES" | while IFS= read -r line; do
                LINE_NUM=$(echo "$line" | cut -d: -f1)
