# 家里 Station NAS · 资源优化与降负载指南

> **目标**：针对家里 NAS（192.168.110.29）的 CPU 99%、磁盘空间告警，降低资源占用、保障稳定运行。  
> **维护**：金仓 | 2026-02-06

---

## 一、当前问题

| 资源 | 状态 | 影响 |
|:-----|:-----|:-----|
| **CPU** | 99% | 系统卡顿、任务响应慢，可能影响其他服务 |
| **RAM** | 61% | 暂可接受 |
| **磁盘** | 空间即将不足 | 存储空间 1 告警，需清理或扩容 |

---

## 二、降 CPU 负载（优先排查）

### 2.1 DSM 界面操作（推荐，先做）

1. **资源监控 → 进程**  
   DSM → 控制面板 → 资源监控 → 进程  
   查看 CPU 占用最高的进程，记下名称。

2. **常见高 CPU 来源及处理**

   | 来源 | 处理方式 |
   |:-----|:---------|
   | **套件索引 / 媒体索引** | 控制面板 → 索引服务 → 暂停或关闭不需索引的文件夹 |
   | **Docker 容器** | 套件中心 → Container Manager → 停止不用的容器；或限制 CPU 使用率 |
   | **Cloud Sync / 同步** | 控制面板 → 云同步 → 暂停或调整同步频率 |
   | **安全顾问 / 防病毒** | 控制面板 → 安全顾问 → 关闭或延后扫描 |
   | **下载任务** | Download Station → 暂停或限制并发数 |
   | **Photo / Moments 索引** | 停用或限制索引范围 |
   | **备份任务** | 控制面板 → 备份 → 调整执行时间（避开白天） |

3. **停用不常用套件**  
   套件中心 → 找到不用的套件 → 停用或卸载。

### 2.2 SSH 排查（需先开启 SSH）

在**本机**（家里局域网）执行：

```bash
# 连接家里 NAS（用户按实际，如 admin）
ssh admin@192.168.110.29

# 查看 CPU 占用最高的进程（前 10）
top -b -n 1 | head -20

# 或
ps aux --sort=-%cpu | head -15
```

根据进程名判断：是套件、Docker、系统进程，再回 DSM 对应停用或限流。

### 2.3 快速临时降负载

- 重启 DSM：控制面板 → 任务计划 → 新增 → 用户定义的脚本 → 填 `reboot`，执行一次后删除或禁用。
- 或：控制面板 → 系统 → 关机/重启 → 重新启动（会短暂停机）。

## 三、释放磁盘空间

1. **存储空间**  
    DSM → 存储空间管理员 → 查看各卷使用率，找出占用最高的卷。

2. **清理建议**

   | 位置 | 操作 |
   |:-----|:-----|
   | **回收站** | File Station 各共享文件夹 → 回收站 → 清空 |
   | **@Recently 等系统缓存** | 控制面板 → 索引服务 → 确认不需要的索引目标可删除 |
   | **下载任务临时文件** | Download Station → 检查未完成/临时文件 |
   | **Docker 镜像/容器** | Container Manager → 删除未使用的镜像、停止的容器 |
   | **日志** | 控制面板 → 日志中心 → 清理旧日志（若可保留时间短） |
   | **大文件** | File Station 搜索大文件（如 >1GB），按需迁移或删除 |

3. **扩容**  
   若磁盘已满且无法清理，需考虑加硬盘或更换更大容量盘（按机型支持）。

---

## 四、长期优化建议

- **索引**：只为必须搜索的文件夹启用索引，其余关闭。  
- **套件**：只保留常用套件，定期检查并停用不用的。  
- **定时任务**：备份、快照等放在凌晨执行，避开白天。  
- **Docker**：限制容器 CPU/内存；不用的容器及时停止。  
- **云同步**：减少同步频率或只同步必要目录。

---

## 五、执行顺序（建议）

1. **先查 CPU 来源**：资源监控 → 进程，看谁占 CPU。  
2. **优先停可疑进程**：索引、Cloud Sync、Docker 等。  
3. **磁盘清理**：回收站、Docker 未用镜像、大文件。  
4. **若仍高**：重启 DSM 观察；必要时 SSH 查 `top` 找具体进程。
