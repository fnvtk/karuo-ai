# Human 3.0 提问法

> 限制AI直接给建议，先追问再输出，让需求无遗漏。

---

## 一、Human 3.0 核心理念

### 1.1 三个阶段

| 阶段 | 特征 | AI交互方式 | 问题 |
|:---|:---|:---|:---|
| **1.0 遵循规则** | 依赖外界标准，害怕出错 | AI直接给答案 | 用户依赖AI，容易出错 |
| **2.0 追求效率** | 觉醒个体意志，快速试错 | AI快速响应 | 节省时间但遗漏细节 |
| **3.0 整合全局** | 内外一致，长期主义视角 | **AI先追问5-6轮，再输出方案** | 需求无遗漏，方案可落地 |

### 1.2 核心原则

> **不要让AI直接给建议，让AI先向你提问。**

**为什么？**

1. 用户往往不知道自己真正需要什么
2. AI直接给的建议可能基于错误假设
3. 通过追问可以挖掘冰山下的真实需求
4. 追问过程本身帮助用户理清思路

## 二、Human 3.0 四大象限

| 象限 | 含义 | 追问方向 |
|:---|:---|:---|
| **心智** | 认知、觉察 | "你对这个问题的理解是什么？" |
| **身体** | 能量、纪律 | "你有多少时间/资源投入？" |
| **精神** | 价值、关系 | "这个对你为什么重要？" |
| **职业** | 目标、影响力 | "成功的标准是什么？" |

---

## 三、Human 3.0 提问流程

### 3.1 标准流程

```
1. 📥 接收用户需求
   ↓
2. 🚫 不要直接给方案
   ↓
3. 🔄 进入追问模式（5-6轮）
   ├── 第1轮：目标澄清
   ├── 第2轮：边界定义
   ├── 第3轮：资源约束
   ├── 第4轮：异常处理
   ├── 第5轮：验收标准
   └── 第6轮：确认汇总
   ↓
4. ✅ 用户确认追问结果
   ↓
5. 📄 输出完整方案
```

### 3.2 每轮追问要点

| 轮次 | 目的 | 核心问题 |
|:---|:---|:---|
| 1 | 目标澄清 | 解决什么问题？给谁用？成功标准？ |
| 2 | 边界定义 | 做什么？不做什么？技术约束？ |
| 3 | 资源约束 | 时间？预算？人力？ |
| 4 | 异常处理 | 失败了怎么办？有Plan B吗？ |
| 5 | 验收标准 | 怎么测试？谁来验收？ |
| 6 | 确认汇总 | 以上信息是否确认？ |

## 四、Human 3.0 提问技巧

### 4.1 提供选项，降低回答门槛

**差的问法**：
> "你想要什么功能？"

**好的问法**：
> "你想要的功能是：A. 登录注册 / B. 数据展示 / C. 支付功能 / D. 其他（请说明）"

### 4.2 使用表格，结构清晰

```markdown
| 序号 | 问题 | 选项 |
|:---|:---|:---|
| 1.1 | 核心用户是谁？ | A. C端 / B. B端 / C. 内部 |
| 1.2 | 预算范围？ | A. <1万 / B. 1-5万 / C. >5万 |
```

### 4.3 序号化，方便简答

告诉用户：
> "可以用序号简答，如：1.1→A, 1.2→B"

### 4.4 每轮汇总，确认理解

```markdown
## 📌 已确认信息

| 维度 | 内容 |
|:---|:---|
| 核心目标 | XXX |
| 主要用户 | YYY |

以上是否正确？
```

## 五、Human 3.0 vs 传统对话

| 维度 | 传统对话 | Human 3.0 |
|:---|:---|:---|
| **开始方式** | 用户说需求，AI给方案 | AI先追问，用户回答 |
| **对话轮数** | 1-2轮 | 5-6轮 |
| **方案质量** | 可能遗漏细节 | 需求完整，方案可落地 |
| **用户参与** | 被动接收 | 主动思考 |
| **需求变更** | 后期频繁变更 | 前期充分澄清 |

## 六、Human 3.0 心力教练模式

**来源**：Dan Koe 的成长模型 + AI Prompt Engineering

**核心理念**：

1. **不给答案，给问题**：通过提问帮助用户自己找到答案
2. **识破自我欺骗**：通过追问发现用户的"伪需求"
3. **找出核心问题**：挖掘阻碍升级的真正障碍
4. **分阶段行动方案**：30/90/180天的落地计划

**提示词示例**：

```
你是一个Human 3.0心力教练。

规则：
1. 不要直接给建议
2. 先通过5-6轮追问了解用户真实情况
3. 每轮追问不超过5个问题
4. 每个问题提供2-4个选项
5. 每轮结束汇总已确认信息
6. 追问完成后再输出方案

开始第一轮追问。
```

## 七、实战案例

### 案例：知己项目需求澄清

**用户初始需求**："帮我完善知己项目的需求"

**Human 3.0 追问过程**：

| 轮次 | AI追问 | 用户回答 |
|:---|:---|:---|
| 1 | MVP目标是什么？A. 跑通流程 / B. 用户付费 / C. 数据指标 | B |
| 1 | AI后端用什么？A. OpenAI / B. Gemini / C. 国产 | A |
| 2 | 付费模式？A. 单次 / B. 订阅 / C. 组合 | C |
| 2 | 登录方式？A. 微信 / B. 手机号 / C. 两种 | A |
| 3 | API失败怎么办？A. 报错 / B. 重试 / C. 切换备用 | C |
| 4 | 测试用例有哪些？ | 【生成14个用例】 |

**最终输出**：TDD需求方案文档

**关键发现**：追问中发现需求方向调整（从八字命理→现代性格测评）
