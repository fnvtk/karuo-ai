# 个人档案生成示例

从对话记录中提取信息的参考示例。

## 信息提取模式

### 基础信息提取

**模式**：寻找关于年龄、出生年份、教育、所在地的明确表述

**对话示例**：
```
说话人 2: 我85年生，08年大学毕业，遇到金融危机
```

**提取结果**：
- 出生年份：1985年
- 毕业年份：2008年
- 背景：金融危机

### 性格特质提取

**模式**：观察决策模式、沟通风格、风险承受能力

**对话示例**：
```
说话人 1: 他很谨慎，考察半年才打定金
说话人 2: 我们不懂的就是不懂，没必要装，虚心学习就好
```

**提取结果**：
- 决策风格：谨慎，需要6个月考察期
- 沟通方式：诚实面对不足，愿意学习
- 风险承受：中低风险

### 投资策略提取

**模式**：寻找系统性方法、规则、框架

**对话示例**：
```
说话人 1: 每年就3个时间点选股：国庆、春节前，选强板块+强个股
```

**提取结果**：
- 时机：每年3个关键时间点（国庆、春节前）
- 方法：强板块+强个股
- 频率：年度，不频繁交易

### 资源提取

**模式**：识别提及的资产、关系、能力

**对话示例**：
```
说话人 2: 我在金蝶做销售，各行业都有认识的人
说话人 1: 他家族在五缘湾有房产
```

**提取结果**：
- 职业：金蝶销售，跨行业人脉
- 资产：家族在五缘湾有房产
- 网络：广泛的商业关系

### 项目参与提取

**模式**：识别角色、投资金额、参与程度

**对话示例**：
```
说话人 1: 玩值电竞项目，20万-200万投资区间，49%股份出让
说话人 2: 前期主要是资金支持，后期市场拓展时发挥作用
```

**提取结果**：
- 项目：玩值电竞
- 投资：20万-200万区间
- 角色：投资人，后期市场拓展
- 股份：49%可出让

## 常见信息类别

### 必须提取

1. **姓名变体**：目标人物可能以不同名称/头衔出现
2. **年龄/出生年份**：通常在上下文中提及
3. **所在地**：城市、地区
4. **职业**：当前工作、行业
5. **教育**：学校、毕业年份
6. **家庭**：如提及（配偶、子女、父母）

### 经常提取

1. **性格指标**：决策模式、沟通风格
2. **投资理念**：风险承受、策略
3. **专业技能**：技术、销售、管理
4. **资源**：资金能力、人脉、资产
5. **项目**：当前参与、角色
6. **关系**：关键联系人、合作伙伴

### 有时提取

1. **健康信息**：如与工作/合作相关
2. **个人兴趣**：爱好、热情
3. **人生事件**：重要里程碑
4. **挑战**：面临的困难
5. **目标**：短期和长期目标

## 说话人识别

### 模式识别

**对话中的多个说话人**：
- 寻找说话人标签："说话人 1"、"说话人 2"、姓名
- 识别目标人物 vs. 其他人
- 注意说话人之间的关系

**示例**：
```
说话人 1: [目标人物说话]
说话人 2: [合作伙伴/同事说话]
说话人 3: [其他人]
```

**提取策略**：
- 关注关于目标人物的陈述（来自他人）
- 包含目标人物自己的陈述（第一人称）
- 区分目标人物自己的话 vs. 他人的描述

## 常见挑战

### 挑战1：信息不完整

**解决方案**：标记为"未明确"或"待确认"，在99_原始文件清单.md中注明

### 挑战2：信息冲突

**解决方案**：包含两个版本，注明冲突，优先使用更近期或更具体的信息

### 挑战3：推断信息

**解决方案**：明确标记为"推测"或"基于对话推断"，在99_原始文件清单.md中注明

### 挑战4：多个名称

**解决方案**：在01_自我画像中列出所有名称变体，使用最常见的名称作为主要名称

### 挑战5：大文件

**解决方案**：使用`limit`参数分块读取，系统化提取，合并洞察
