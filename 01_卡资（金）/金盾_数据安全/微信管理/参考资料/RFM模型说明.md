# RFM 价值评估模型

> 基于聊天记录的好友价值评估体系

---

## 模型概述

RFM 模型源自营销领域，用于评估客户价值。我们将其适配到微信好友管理场景：

| 维度 | 原义 | 微信场景含义 |
|:-----|:-----|:-------------|
| **R** (Recency) | 最近购买时间 | 最近联系时间 |
| **F** (Frequency) | 购买频率 | 联系频率 |
| **M** (Monetary) | 消费金额 | 潜在价值 |

---

## 评分规则

### R - Recency（最近联系）

衡量好友与你最近一次互动的时间距离。

| 分值 | 天数 | 说明 |
|:-----|:-----|:-----|
| 5 | ≤7天 | 本周活跃 |
| 4 | 8-30天 | 本月活跃 |
| 3 | 31-90天 | 近期有联系 |
| 2 | 91-180天 | 半年内联系 |
| 1 | >180天 | 超过半年未联系 |

### F - Frequency（联系频率）

统计最近30天内的消息互动数量。

| 分值 | 消息数 | 说明 |
|:-----|:-------|:-----|
| 5 | ≥50条 | 高频互动 |
| 4 | 20-49条 | 经常联系 |
| 3 | 10-19条 | 偶尔联系 |
| 2 | 5-9条 | 很少联系 |
| 1 | <5条 | 几乎不联系 |

### M - Monetary（潜在价值）

基于聊天内容中的关键词计算价值分。

**高价值关键词 (+3分)**

```
合作、投资、采购、签约、付款、项目、需求、预算、订单
```

**中价值关键词 (+2分)**

```
咨询、了解、推荐、介绍、报价、方案、沟通、交流
```

**低价值关键词 (+1分)**

```
谢谢、感谢、帮忙、请教、学习
```

**负面关键词 (-1分)**

```
广告、推销、砍价、免费
```

**M值归一化**

| 分值 | 关键词累计分 | 说明 |
|:-----|:-------------|:-----|
| 5 | ≥50分 | 极高价值 |
| 4 | 30-49分 | 高价值 |
| 3 | 15-29分 | 中价值 |
| 2 | 5-14分 | 低价值 |
| 1 | <5分 | 无明显价值 |

---

## 价值等级

根据 R+F+M 总分（3-15分）划分等级：

### ⭐⭐⭐⭐⭐ 钻石好友（13-15分）

**特征**：
- 近期频繁联系
- 有明确合作意向或已合作
- 高价值对话内容多

**建议动作**：
- 优先响应消息
- 定期维护关系
- 主动提供价值
- 节日特别关怀

### ⭐⭐⭐⭐ 黄金好友（10-12分）

**特征**：
- 定期保持联系
- 有潜在合作可能
- 互动质量较高

**建议动作**：
- 及时响应
- 适时深化关系
- 挖掘合作机会

### ⭐⭐⭐ 普通好友（7-9分）

**特征**：
- 偶尔联系
- 关系一般
- 互动较少

**建议动作**：
- 正常响应
- 节日问候
- 保持基本联系

### ⭐⭐ 沉默好友（4-6分）

**特征**：
- 长期不联系
- 曾有互动但已减少
- 可能有激活价值

**建议动作**：
- 定期激活尝试
- 分析沉默原因
- 考虑分组管理

### ⭐ 僵尸好友（1-3分）

**特征**：
- 超长时间未联系
- 几乎无互动
- 价值关键词为零

**建议动作**：
- 评估是否保留
- 考虑清理
- 或归入备用池

## 进阶应用

### 1. 动态权重

根据业务场景调整 R/F/M 权重：

| 场景 | R权重 | F权重 | M权重 |
|:-----|:------|:------|:------|
| 销售型 | 30% | 30% | 40% |
| 社交型 | 40% | 40% | 20% |
| 合作型 | 20% | 30% | 50% |

### 2. 分群策略

```
           高 M
            │
    沉睡大客户 ← → 重点维护
            │
低 R ───────┼─────── 高 R
            │
    考虑清理  ← → 潜在培养
            │
           低 M
```

### 3. 预警机制

- **流失预警**：R值从5降到3，提醒主动联系
- **降级预警**：总分下降超过3分，分析原因
- **机会预警**：M值突然上升，可能有合作机会

```yaml
  - 投资
  - 签约
  - 自定义关键词...

medium_value:  # +2分
  - 咨询
  - 方案
  - 自定义关键词...

low_value:  # +1分
  - 感谢
  - 学习
  - 自定义关键词...

negative:  # -1分
  - 广告
  - 推销
  - 自定义关键词...
```

```yaml
# rfm_config.yaml
recency:
  - {days: 7, score: 5}
  - {days: 30, score: 4}
  - {days: 90, score: 3}
  - {days: 180, score: 2}
  - {days: 365, score: 1}

  - {count: 50, score: 5}
  - {count: 20, score: 4}
  - {count: 10, score: 3}
  - {count: 5, score: 2}
  - {count: 0, score: 1}
```
