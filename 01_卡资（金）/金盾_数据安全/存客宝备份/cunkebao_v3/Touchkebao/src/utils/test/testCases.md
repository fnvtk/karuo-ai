# 存客宝新架构测试用例

## 测试用例说明

本文档包含新架构的详细测试用例，用于功能测试和回归测试。

## 1. 会话列表功能测试

### 1.1 切换账号测试

**测试用例ID**: TC-SESSION-001
**测试目标**: 验证切换账号功能正常工作
**前置条件**:
- 已登录系统
- 有多个微信账号数据
- 会话列表已加载

**测试步骤**:
1. 打开会话列表
2. 切换到"全部"账号（accountId=0）
3. 切换到账号1
4. 切换到账号2
5. 切换回"全部"

**预期结果**:
- 每次切换耗时 < 100ms
- 会话列表正确显示对应账号的数据
- 切换回之前账号时使用缓存（更快）
- 无错误提示

**性能指标**:
- 切换账号响应时间: < 100ms
- 缓存命中时响应时间: < 50ms

**性能指标**:
- 缓存读取时间: < 50ms
- 无缓存时API调用时间: 正常（取决于网络）
