- **地点**：你在北京，客户在上海
- **问题**：客户的会议平板无法使用
- **目标**：远程诊断和修复

### 沟通过程

#### 客户: "我们的会议平板打不开应用了"

```bash
# 你对AI说：
"引导客户告诉我设备IP"
```

AI会生成引导话术：

```
请告诉客户：

"您好，我需要远程帮您诊断问题。请在平板上按以下步骤操作：

1. 打开'设置'
2. 找到'关于平板'或'关于设备'
3. 点击'状态'或'网络'
4. 找到'IP地址'
5. 将看到的IP地址告诉我

例如：192.168.1.100"
```

#### 客户: "IP是 192.168.50.88"

```bash
# 你对AI说：
"尝试连接 192.168.50.88"
```

```
[ERROR] 连接失败: 网络不可达

可能原因:
  1. 我们不在同一网络
  2. 客户网络有防火墙
  3. 设备未开启ADB调试

建议方案:
  方案1: 使用VPN连接到客户网络
  方案2: 使用远程桌面工具 (TeamViewer/AnyDesk)
  方案3: 引导客户进行本地操作
```

#### 采用方案2：使用远程桌面

```bash
# 你对AI说：
"引导客户安装AnyDesk并提供ID"
```

AI生成引导话术：

```
请告诉客户：

"我需要通过远程桌面帮您操作，请按以下步骤：

1. 在电脑上打开浏览器
2. 访问: anydesk.com/download
3. 下载并安装 AnyDesk
4. 打开后会显示一个9位数字ID
5. 将这个ID告诉我

或者，如果您有TeamViewer，也可以使用"
```

#### 客户提供AnyDesk ID后

通过AnyDesk远程连接到客户电脑，然后：

```bash
# 在客户电脑的终端执行（通过远程桌面）
adb devices

# 输出显示设备离线
192.168.50.88:5555    offline
```

```bash
# 对AI说（在你的电脑）：
"设备显示offline，如何解决？"
```

AI提供解决方案：

```
[INFO] ADB设备离线解决方案

方案1: 重启ADB服务（推荐）
  在客户电脑执行:
  adb kill-server
  adb start-server
  adb connect 192.168.50.88:5555

方案2: 重启设备无线调试
  在平板上:
  设置 → 开发者选项 → 无线调试 → 关闭 → 再开启

方案3: USB连接
  如果客户有USB线，可以直接USB连接
```

#### 执行方案1

```bash
# 在客户电脑执行（你远程操作）
adb kill-server
adb start-server
adb connect 192.168.50.88:5555

# 输出
* daemon not running; starting now at tcp:5037
* daemon started successfully
connected to 192.168.50.88:5555
```

成功连接后：

```bash
# 对AI说：
"诊断 192.168.50.88 的应用问题"
```

```
[INFO] 开始远程诊断...

设备信息:
  型号: MAXHUB V6
  系统: Android 9
  IP: 192.168.50.88

[SCAN] 收集应用日志...
[SCAN] 分析崩溃记录...

问题诊断:
  应用: 腾讯会议 (com.tencent.wemeet)
  状态: disabled (被禁用)
  原因: 系统更新后应用被自动禁用

修复方案:
  1. 启用应用
  2. 清理缓存
  3. 重启应用

是否执行自动修复？(y/N): y
```

执行修复后：

```
[✓] 应用已启用
[✓] 缓存已清理
[✓] 应用启动成功
```

#### 客户: "好了！可以用了！谢谢！"

```bash

```

## 💡 使用技巧总结

### 1. 优先使用AI对话方式
```
直接说："扫描局域网设备"
直接说："连接 192.168.2.15"
直接说："安装微信到所有设备"
```
**优势**: AI会自动处理所有细节，无需记忆命令

### 2. 善用批量操作
```
"连接所有 Android 设备"
"给所有设备安装应用"
"检查所有设备状态"
```
**优势**: 一次性处理多台设备，提高效率

### 3. 问题诊断找AI
```
"192.168.2.15 的应用闪退，帮我诊断"
"为什么ADB连接失败？"
"如何解决XXX问题？"
```
**优势**: AI会自动收集日志、分析问题、提供方案

### 4. 保存常用配置
```
"把 192.168.2.15 保存为'会议室TV'"
"记住这些设备的IP"
```
**优势**: 下次直接使用别名，无需记忆IP

### 5. 生成专业报告
```
"生成设备列表报告"
"生成技术支持报告"
"导出诊断结果"
```
**优势**: 专业文档，便于存档和汇报
