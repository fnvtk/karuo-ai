## 一、止损规则

### 1. 单股止损

| 策略 | 止损线 | 执行方式 |
|:---|:---|:---|
| 7根K线 | -5% | 无条件平仓 |
| 节假日 | -3% | 无条件平仓 |
| 高手跟踪 | -5% | 无条件平仓 |

**执行原则**：触发止损线，立即平仓，不等反弹。

### 2. 单日止损

```yaml
单日最大亏损: -5%
触发后: 暂停交易1个交易日
恢复后: 降低仓位50%
```

### 3. 最大回撤

```yaml
最大回撤: -15%
触发后: 
  - 暂停所有自动交易
  - 人工介入评估
  - 可能需要调整策略参数
恢复条件: 人工确认后重新启动
```

## 二、仓位控制

### 1. 持仓数量限制

| 限制项 | 数值 | 说明 |
|:---|:---|:---|
| 最大持仓数 | 5只 | 分散风险 |
| 最小持仓数 | 0只 | 可空仓 |
| 单股仓位 | 最多25% | 不重仓单只 |

### 2. 板块集中度

```yaml
同板块最大仓位: 40%
原因: 避免板块系统性风险
例外: 节假日策略可放宽至50%
```

### 3. 现金保留

```yaml
最低现金: 10%
原因: 应对突发情况
用途: 止损补仓、打新
```

### 4. 仓位计算

```python
def calculate_position_size(total_assets, risk_per_trade=0.02):
    """
    计算单笔仓位
    
    风险管理公式:
    仓位 = (总资产 × 单笔风险比例) / 止损幅度
    
    例: 10万资产，单笔风险2%，止损5%
    仓位 = (100000 × 0.02) / 0.05 = 40000元
    """
    stop_loss_pct = 0.05
    position_size = (total_assets * risk_per_trade) / stop_loss_pct
    return min(position_size, total_assets * 0.25)  # 不超过25%
```

## 三、交易频率控制

### 1. 每日交易限制

```yaml
每日买入次数: 最多5次
每日卖出次数: 不限（止损优先）
每日交易金额: 5万元（T+1循环）
```

### 2. 单股交易间隔

```yaml
同一股票:
  买入间隔: 至少1小时
  加仓条件: 盈利>2%后可加仓
  加仓次数: 最多1次
```

### 3. 异常交易监控

```yaml
异常情况:
  - 1分钟内超过3笔交易 → 暂停10分钟
  - 1小时内超过10笔交易 → 暂停1小时
  - 连续3笔亏损 → 人工确认后继续
```

## 五、市场状态适应

### 1. 大盘监控

```python
def get_market_state(index_change_pct):
    """
    判断大盘状态
    
    index_change_pct: 上证指数涨跌幅
    """
    if index_change_pct > 0.02:
        return '大涨'
    elif index_change_pct < -0.02:
        return '大跌'
    else:
        return '平衡'
```

### 2. 状态对应策略

| 大盘状态 | 买入策略 | 卖出策略 | 仓位建议 |
|:---|:---|:---|:---|
| 大涨 | 谨慎追高 | 高点止盈 | 降低20% |
| 平衡 | 正常执行 | 正常执行 | 标准仓位 |
| 大跌 | 观望/抄底 | 止损优先 | 提高现金 |

### 3. 极端行情处理

```yaml
熔断触发:
  - 暂停所有交易
  - 等待恢复后评估

连续跌停:
  - 不抄底
  - 等待企稳信号

连续涨停:
  - 不追高
  - 持仓不动
```

## 六、异常情况处理

### 1. 网络异常

```yaml
检测: 每30秒ping服务器
处理:
  - 断网超过1分钟 → 暂停交易
  - 断网超过5分钟 → 发送告警
  - 恢复后 → 同步账户状态
```

### 2. 客户端异常

```yaml
同花顺崩溃:
  - 自动重启客户端
  - 重连券商
  - 同步订单状态

订单超时:
  - 超过30秒未确认 → 撤单重试
  - 超过3次失败 → 放弃该笔交易
```

### 3. 策略异常

```yaml
信号冲突:
  - 同一股票买卖冲突 → 取消买入
  - 多策略信号冲突 → 取置信度最高

数据异常:
  - 价格异常 → 跳过该信号
  - 数据缺失 → 使用上一有效值
```

## 八、紧急情况预案

### 1. 熔断预案

```
触发条件: 大盘跌幅>5%
动作:
1. 暂停所有自动交易
2. 发送告警通知
3. 等待人工确认
4. 评估持仓风险
5. 决定是否止损
```

### 2. 系统故障预案

```
触发条件: 交易系统无法连接
动作:
1. 尝试重连(最多3次)
2. 切换备用通道
3. 发送告警通知
4. 记录未执行订单
5. 人工介入处理
```

### 3. 资金异常预案

```
触发条件: 账户资金异常变动
动作:
1. 暂停交易
2. 核对交易记录
3. 联系券商确认
4. 恢复正常后继续
```
