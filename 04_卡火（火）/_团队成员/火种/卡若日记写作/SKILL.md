## 三、核心格式规范

### 3.1 排版铁律

```
【规则1】每一句话后面必须空一行

【规则2】每个自然段之间空一行

【规则3】对话、心理、细节、观点都要分行

【规则4】避免大段文字堆积（每段不超过3句话）

【规则5】用粗体突出核心观点

【规则6】短句为主，长句拆分
```

### 3.2 字数要求

- **最低字数**：1500字
- **推荐字数**：1500-1900字
- **多场景时**：需要切换场景，每个场景至少300字

### 3.3 名字规则

- "卡若"全文最多提1次，出现在开篇自我介绍或文末
- 不重复提到"我是卡若"
- 用"我"代替大部分人称

## 五、写作技巧库

### 5.1 开篇钩子（选一个）

| 类型 | 模板 | 示例 |
|:---|:---|:---|
| **冲突型** | {年份}，{重大事件}，{后果} | "2018年，电商公司倒闭，大量劳务纠纷。" |
| **自问型** | 你有没有发现，{普遍现象}？ | "你有没有发现，很多私域运营者每天都在追求新客、新流量，但回头一看……" |
| **场景型** | {时间}，{具体场景描述} | "昨晚，又是一场从下午三点五十五分开始的会议。" |
| **颠覆型** | 如果你以为{常见认知}，那你就错了。 | "如果你以为卖猫只是挑选品种、找到合适的买家，那你就错了。" |
| **观点型** | {激进观点，制造冲突} | "首先，得承认，社会上的规则真的是两套。" |

### 5.2 对话技巧

```markdown
正确示范：

"就是让客户知道我们的价值很高，也是这个意思。"

我当时就直接说了，这是价值定位的问题。

"你这个OKR，那我就，嗯，对，那我这个得回去再弄一下。"

---

错误示范：

我说："这是价值定位的问题，我们要让客户清晰地感受到产品价值。"他回答说："好的，我知道了。"

（太书面化，缺乏真实感）
```

### 5.3 数据植入

```markdown
必须包含的数据类型：
- 转化率：如"转化率只有可怜的百分之五左右"
- 客户数：如"聊了大概两百九十几个客户"
- 金额：如"带来20万的产值"
- 比例：如"删除和无回复的比例太大了，大概两百八十个"
- 时间：如"三个月的目标"

数据来源：
- 用户提供的聊天记录（优先）
- 会议记录
- 后台数据
- 行业报告（如艾瑞咨询）

⚠️ 不要自己编造数据！
```

### 5.4 隐性植入（会员/合作伙伴）

```markdown
正确示范：

"有一个老板，做高端护肤品的。"

"他开始重新梳理客户路径。"

"有一家门店，在引流阶段吸引了上千名客户。"

---

错误示范：

"我的会员小王最近遇到一个问题。"

"作为我的合作伙伴，他……"

（太直白，失去真实感）
```

### 5.5 粗体使用规则

```markdown
粗体用于：
1. 核心观点总结
2. 转折性语句
3. 行动号召

示例：
**价值定位是关键，几十块和几万块，客户感知天壤之别。**

**复制和放大，是实现规模化的核心。**

**AI不会取代所有程序员，但一定会取代那些只会"搬砖"的程序员。**
```

## 十三、常见问题

**Q1: 多场景怎么切换？**

用分割线 `---` 分隔不同场景，每个场景保持独立的时间线或逻辑线。

**Q2: 数据不够怎么办？**

优先使用模糊表达："大概""将近""超过"，但不要凭空编造具体数字。

**Q3: 如何体现会员关系？**

用"有一个老板""我认识的一个朋友""之前合作的一家企业"等隐性表达。

**Q4: 文章太短怎么办？**

增加：对话细节、心理描写、案例补充、方法论拆解。

**Q5: 飞书上传报Bad Request？**

原因：JSON中有特殊字符（中文引号""等）。解决：用 `upload_feishu.py` 脚本自动处理。

**Q6: 飞书已有同名文章怎么办？**

脚本自动检测，创建新版本（标题加"更新版"后缀），不会覆盖原文。

**Q7: Token过期怎么办？**

脚本自动用refresh_token刷新。如果刷新也失败，重新运行飞书工具箱授权。

**Q8: 如何把观点文章改成故事形式？**

模板：
```
1. 开头：一个具体的人找我（喝酒/聊天/咨询）
2. 他的问题（代表读者痛点）
3. 我给他讲故事/算账/分析
4. 他的反应（共鸣点）
5. 我的建议
6. 他的行动和结果
7. 结尾升华
```
