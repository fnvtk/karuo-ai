## 🎯 v0提示词最佳实践（官方指南）

### 核心框架：三要素法

```
1️⃣ 产品表面（What）
   → 明确说明：组件类型、功能点、数据展示
   例："Dashboard显示Top 5业绩表"

2️⃣ 使用上下文（Who/When）
   → 说明：谁用、什么场景、什么设备
   例："销售经理在晨会时用桌面查看"

3️⃣ 约束与风格（How）
   → 指定：技术栈、设计风格、布局、颜色
   例："卡片布局、蓝色主题、移动优先"
```

### 高质量提示词模板

```
【组件名称】：用户登录页面
【核心功能】：手机号+验证码登录、记住密码
【设计风格】：iOS风格、圆角卡片、蓝色主题
【技术要求】：React + TypeScript + Tailwind + shadcn/ui
【约束条件】：移动端优先、有loading状态、有表单验证
```

### 避免的错误

| ❌ 错误 | ✅ 正确 |
|:---|:---|
| "做一个好看的登录页" | "做一个iOS风格的手机号登录页，蓝色主题，有验证码倒计时" |
| 一次写10个需求 | 先做核心功能，再迭代添加 |
| "简洁但功能全面" | 明确优先级，先简洁再扩展 |
| 不说技术栈 | 明确指定React+Tailwind+shadcn |

### 迭代技巧

```
第1轮：先生成基础结构
"生成一个登录表单组件，手机号+验证码"

第2轮：调整样式
"把按钮改成圆角，加渐变背景"

第3轮：添加功能
"加上loading状态和错误提示"

第4轮：优化细节
"加上表单验证，手机号必须11位"
```

### .cursorrules（社区最佳实践）

```markdown
# Next.js + React + TypeScript + shadcn/ui 开发规则

## 项目结构
- 使用 Next.js 14+ App Router
- 页面放 src/app/
- 组件放 src/components/
- UI组件放 src/components/ui/（shadcn）
- 目录名用小写+连字符（如 auth-wizard）

## TypeScript规范
- 使用函数式和声明式编程模式
- 优先使用 interface 而非 type
- 避免使用 enum，用 map 替代
- 变量名用描述性+辅助动词（isLoading, hasError）

## React组件规范
- 最小化 'use client'，优先使用 Server Components
- 客户端组件用 Suspense 包裹，提供 fallback
- 非关键组件使用 dynamic loading
- 图片用 WebP 格式 + lazy loading

## UI/样式规范
- 使用 shadcn/ui 组件库
- 使用 cn() 工具函数处理条件类名
- Tailwind CSS 移动端优先
- 必须中文注释

## 表单/数据规范
- 表单用 react-hook-form
- 数据验证用 Zod
- 状态管理用 Zustand
```
