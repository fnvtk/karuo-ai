## 📱 常用命令速查

### 设备发现
```bash
# 扫描局域网所有设备
./scripts/局域网设备管理.sh

# 快速ping扫描
for i in {1..254}; do ping -c 1 -W 500 192.168.2.$i >/dev/null && echo "192.168.2.$i"; done &

# 检查ADB设备
adb devices -l

# 查看本机IP
ifconfig | grep "inet " | grep -v 127.0.0.1
```

### 设备连接
```bash
# 连接Android设备
adb connect <IP>:5555

# 尝试多个端口
for port in 5555 5037 4444 5556; do
  adb connect <IP>:$port
done

# 断开所有设备
adb disconnect

# 断开指定设备
adb disconnect <IP>:5555

# 重启ADB服务
adb kill-server && adb start-server
```

### 设备信息
```bash
# 快速查看设备信息
adb -s <IP>:5555 shell "getprop ro.product.brand; getprop ro.product.model; getprop ro.build.version.release"

# 详细设备信息
adb -s <IP>:5555 shell getprop | grep -E "model|brand|version.release|cpu.abi"

# 电池电量
adb shell dumpsys battery | grep level

# 存储空间
adb shell df -h /data

# 内存状态
adb shell cat /proc/meminfo | grep MemTotal
```

### 屏幕控制
```bash
# 启动屏幕镜像（单设备）
scrcpy

# 指定设备镜像
scrcpy -s <IP>:5555

# 降低延迟配置
scrcpy -s <IP>:5555 -m 1024 -b 2M --max-fps 30

# 录屏
scrcpy --record screen.mp4

# 只录屏不显示
scrcpy --record screen.mp4 --no-display
```

### 应用管理
```bash
# 列出所有应用
adb shell pm list packages

# 查找特定应用
adb shell pm list packages | grep <keyword>

# 安装应用（覆盖+自动授权）
adb install -r -g <app.apk>

# 卸载应用
adb uninstall <package_name>

# 启动应用
adb shell monkey -p <package_name> 1

# 停止应用
adb shell am force-stop <package_name>

# 清理应用数据
adb shell pm clear <package_name>

# 启用/禁用应用
adb shell pm enable <package_name>
adb shell pm disable <package_name>

# 查看应用详细信息
adb shell dumpsys package <package_name>
```

### 文件传输
```bash
# 推送文件到设备
adb push <本地文件> /sdcard/

# 拉取文件到本地
adb pull /sdcard/<文件> <本地路径>

# 浏览设备文件
adb shell ls /sdcard/

# 删除设备文件
adb shell rm /sdcard/<文件>
```

### 系统诊断
```bash
# 查看实时日志
adb logcat

# 查看崩溃日志
adb logcat -d | grep -E "FATAL|CRASH"

# 清空日志
adb logcat -c

# 查看系统服务
adb shell dumpsys

# 查看运行进程
adb shell ps

# 查看网络连接
adb shell netstat -an
```

## 🛠️ 故障排除

### ADB连接失败
```bash
# 1. 重启ADB
adb kill-server && adb start-server

# 2. 尝试不同端口
adb connect <IP>:5555
adb connect <IP>:5037

# 3. 检查网络
ping <IP>

# 4. 对AI说：
"检查 <IP> 的ADB状态"
```

### 屏幕镜像卡顿
```bash
# 降低分辨率
scrcpy -m 1024

# 降低比特率
scrcpy -b 2M

# 降低帧率
scrcpy --max-fps 30

# 组合使用
scrcpy -m 1024 -b 2M --max-fps 30
```

### 应用安装失败
```bash
# 1. 检查系统版本
adb shell getprop ro.build.version.release

# 2. 检查架构
adb shell getprop ro.product.cpu.abi

# 3. 检查存储空间
adb shell df -h /data

# 4. 强制安装
adb install -r -d -g <app.apk>

# 5. 对AI说：
"检查设备兼容性并安装应用"
```

### 设备扫描不到
```bash
# 1. 检查网络
ping 192.168.2.15

# 2. 检查ADB服务
ps aux | grep adb

# 3. 检查端口
nc -zv 192.168.2.15 5555

# 4. 对AI说：
"为什么扫描不到设备？"
```

## 📊 网络环境适配

### 家庭网络
```
网段: 192.168.1.x / 192.168.0.x
设备: 手机、平板、电视
操作: 直接扫描和连接
```

### 办公室网络
```
网段: 192.168.x.x / 10.x.x.x
设备: 会议室TV、演示设备
操作: 扫描后保存配置
```

### 公共WiFi
```
限制: 可能禁止设备间通信
方案: 使用USB连接或个人热点
```

### 跨网段
```
场景: Mac在192.168.110.x，设备在192.168.2.x
方案: 
  1. 切换到相同WiFi
  2. 配置路由器
  3. 使用VPN
  4. 对AI说："如何跨网段连接设备？"
```

## 💡 最佳实践

### ✅ DO 推荐
- 先扫描网络，了解设备分布
- 使用 -r -g 参数安装（覆盖+授权）
- 定期重启 ADB 服务
- 保存常用设备IP
- 使用AI自动化重复操作

### ❌ DON'T 避免
- 不要在不同网段直接连接
- 不要忽略设备架构兼容性
- 不要删除系统应用
- 不要在公共WiFi操作敏感数据
- 不要手动执行复杂批量操作（让AI帮你）
