## 核心架构

存客宝采用分层分布式微服务架构，核心能力是通过**设备代理 + WebSocket 长连接**实现对第三方系统的远程控制和数据同步。

### 整体数据流

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   手机设备    │ ←→  │  代理服务器   │ ←→  │  存客宝后端   │
│  (微信客户端) │     │ (私域管理SDK) │     │  (ThinkPHP)  │
└──────────────┘     └──────────────┘     └──────────────┘
```

## 连接管理

### 心跳检测

```php
protected $heartbeatInterval = 30; // 心跳间隔，单位秒

protected function sendHeartbeat()
{
    $heartbeat = [
        "cmdType" => "CmdHeartbeat",
        "seq" => time()
    ];
    $this->client->send(json_encode($heartbeat));
    $this->lastHeartbeatTime = time();
}
```

### 自动重连

```php
protected function checkConnection()
{
    if (!$this->isConnected || 
        (time() - $this->lastHeartbeatTime) > $this->heartbeatInterval * 2) {
        $this->reconnect();
    }
}

protected function reconnect()
{
    if ($this->client) {
        $this->client->close();
    }
    $this->isConnected = false;
    $this->connect();
}
```

## 核心算法

### 健康分计算

用于控制自动化操作频率，保护账号安全。

```
健康分 = 基础分(60-100) + 动态分
每日最大操作次数 = 健康分 × 0.2

基础分组成：
- 默认基础分: 60分
- 已修改微信号: +10分
- 好友数量加分: 3-12分

动态分组成：
- 首次频繁: -15分，暂停24小时
- 再次频繁: -25分，暂停24小时
- 封号: -60分，暂停72小时
- 连续3天不频繁: +5分/天
```

### 流量分发算法

轮询分配 + 配额限制

```php
$i = 0;
$accountNum = count($accounts);

foreach ($friends as $friend) {
    if ($accountNum == 0) break;
    if ($i >= $accountNum) $i = 0;  // 轮询重置
    
    $account = $accounts[$i];
    
    // 配额检查
    if ($account['todayCount'] >= $config['maxPerDay']) {
        unset($accounts[$i]);
        $accounts = array_values($accounts);
        $accountNum = count($accounts);
        continue;
    }
    
    // 执行分配
    $this->allotWechatFriend([...]);
    $i++;
}
```

## 应用到网站逆向分析

### 1. 模拟登录

参考存客宝的Token获取方式：
- 分析登录接口的请求格式
- 提取必要的请求头（如 client, verifysessionid）
- 保存Token用于后续请求

### 2. 建立长连接

如果目标网站使用WebSocket：
- 参考存客宝的WebSocket连接代码
- 实现心跳保活机制
- 处理自动重连

### 3. 接口调用

使用适配器模式封装API：
- 每个接口封装为独立方法
- 统一处理认证、错误、重试
- 提供一致的调用接口

### 4. 风控考虑

- 控制请求频率
- 实现任务队列
- 添加随机延迟
- 记录操作日志
