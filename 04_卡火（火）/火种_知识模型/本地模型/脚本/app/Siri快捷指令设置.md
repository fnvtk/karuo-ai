# 🔥 卡若AI Siri快捷指令设置

## 一、前置条件

1. 确保「卡若AI本地助手」服务正在运行
2. 双击桌面的「卡若AI本地助手.app」启动服务

## 二、创建Siri快捷指令

### 方法1：手动创建（推荐）

1. 打开「快捷指令」App（Spotlight搜索 "快捷指令"）
2. 点击 `+` 创建新快捷指令
3. 添加以下操作：

```
操作1：要求输入
    提示：问卡若AI什么？
    输入类型：文本
    
操作2：获取URL内容
    URL：http://localhost:5888/api/siri?q=[要求输入的结果]
    方法：GET
    
操作3：从输入获取字典值
    键：response
    
操作4：朗读文本（可选）
    文本：[字典值]
```

4. 命名为「问卡若」或「卡若AI」
5. 点击右上角设置 → 添加到Siri
6. 录制唤醒词，如「问卡若」

### 方法2：使用AppleScript

在终端运行：

```bash
osascript <<EOF
tell application "Shortcuts Events"
    -- 创建快捷指令的脚本
end tell
EOF
```

## 三、使用方式

### 语音唤醒
- 「Hey Siri，问卡若」
- 「Hey Siri，卡若AI」

### 快捷方式
- 在快捷指令App中直接运行
- 添加到菜单栏/Dock

## 四、API接口

服务运行后，以下接口可用：

| 接口 | 方法 | 说明 |
|-----|-----|-----|
| `/api/siri?q=问题` | GET | Siri快捷指令专用，自动语音播报 |
| `/api/chat` | POST | 对话接口 |
| `/api/memory` | GET/POST | 记忆读写 |
| `/api/skills` | GET | 获取技能列表 |
| `/api/status` | GET | 服务状态 |

## 五、示例调用

```bash
# 命令行测试
curl "http://localhost:5888/api/siri?q=今天天气怎么样"

# 返回格式
{"response": "我是本地运行的AI，无法获取实时天气..."}
```

## 六、故障排除

1. **Siri说"无法连接"**：检查服务是否启动，端口5888是否被占用
2. **没有语音播报**：确认macOS系统偏好设置中语音功能已开启
3. **响应慢**：首次启动模型加载需要几秒，之后会更快

## 七、高级：直接语音对话

创建一个持续对话的快捷指令：

```
重复
    ├─ 听写文本
    ├─ 获取URL：http://localhost:5888/api/siri?q=[听写文本]
    ├─ 获取字典值：response
    └─ 朗读文本
结束重复（当听到"退出"时）
```

这样可以实现连续语音对话，类似ChatGPT语音模式。
