# 卡若AI 项目开发文档（总入口）

更新时间：2026-03-23

这套文档是给「能看懂业务、也准备开始开发」的人用的。
不讲花里胡哨的术语，不贴代码，用大白话把项目讲清楚。

你可以把它理解成一本“开店手册”：
- 先看客户是谁、要卖什么（需求）
- 再看店怎么搭（架构）
- 再看店面怎么摆（原型、前端）
- 再看后厨怎么出餐（后端、接口、数据库）
- 最后看怎么开门营业（部署、手册、项目管理）

## 建议阅读顺序（第一次看）

1. 先看 `1、需求/项目需求文档.md`，搞清楚这个项目到底解决什么问题。
2. 再看 `2、架构/系统架构说明.md`，知道系统由哪些模块组成。
3. 再看 `10、项目管理/开发进度.md`，知道现在做到哪一步。

---

## 本文档怎么维护

- 每做完一个功能，更新一次：
  - `1、需求/功能迭代记录.md`
  - `10、项目管理/开发进度.md`
- 每次更新时间都写当天日期，方便回溯。
- 发现“看不懂”的句子，就立刻改成更口语的话。
