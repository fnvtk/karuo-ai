# Soul 成片流程 · 追问与自检

> **用途**：在跑命令前后用**可回答的问题**卡住丢项；适合 **Agent 自检** 与 **人工复核**。  
> **原则**：答不上来或答「不确定」→ **先补文件/先跑 analyze / 先对表**，再往下阶段。

---

## 〇、编排层（任何场次开工前必问）

| # | 追问 | 合格答案要点 |
|---|------|----------------|
| O1 | 本场输出目录绝对路径是？ | 已存在或即将创建；命名含 `第N场_YYYYMMDD` |
| O2 | 原片是否已落在 `原视频/` 且与场次对应？ | 文件名与妙记/表一致 |
| O3 | 竖屏还是横屏全幅？ | 竖屏：`Soul竖屏切片_SKILL` + 裁剪检查；横屏：`Soul横屏全幅高光_SKILL` 或取向分析先判 |
| O4 | 是否允许覆盖已有 `highlights.json`？ | 精修过则 **禁止** 盲目 `--force-transcribe`；用 skip 参数见工作流 |
| O5 | 阶段Ⅲ 的 `CROP_VF` 来源是本场 `裁剪检查/*.txt` 吗？ | **禁止**照抄旧场数字 |

---

## 一、阶段Ⅰ（转录 + 清单 + JSON）

| # | 追问 | 不合格时 |
|---|------|----------|
| I1 | `transcript.srt` 是否已做 Ⅰ-4（通顺、指代、术语统一）？ | 先修 SRT 再热点分段 |
| I2 | 附录 A 清单是否已按时间排序、去重、6～8 条量级？ | 重跑识别或手改 |
| I3 | 每条 `start_time`/`end_time` 是否落在**同一段连续论述**？ | Ⅰ-6 对表 |
| I4 | 四短字段是否 **2/4 字、通俗、同条互异、全场不撞**？ | 见工作流 §三 表 |
| I5 | 高危词是否在 JSON / 字幕定稿前已处理？ | 人工删改后再切片 |

---

## 二、阶段Ⅱ（仅切片）

| # | 追问 | 不合格时 |
|---|------|----------|
| II1 | `highlights.json` 是否已盖「Ⅰ-7 定稿」章（心理上）？ | **禁止** `batch_clip` |
| II2 | `切片/` 内每条是否粗听头尾完整、无半句？ | 只改 JSON + 重跑 clip |
| II3 | 文件名 stem 是否仍符合字数与去重？ | 回改 JSON |

---

## 三、阶段Ⅲ（soul_enhance）

| # | 追问 | 不合格时 |
|---|------|----------|
| III1 | 命令里是否有 **`--force-burn-subs`**？要逐字是否加了 **`--typewriter-subs`**？ | 补参重跑 |
| III2 | `--crop-vf` / `--overlay-x` 是否与**本场**塑形 txt **逐字符一致**？ | 重跑 analyze |
| III3 | 封面首屏优先字来自 `viral_hook` → `hook_3sec` → `title` 是否知情？ | 读 `Soul成片封面_SKILL` |
| III4 | 贴片默认开；若挡脸挡字幕，是否已知可用 `--no-stickers` 或调策略？ | 读 `Soul成片特效_SKILL` |
| III5 | 语速与静音：当前用的是默认加速还是 `--silence-gentle` / `--no-speedup`？ | 写进场次笔记避免下一场混 |

---

## 四、阶段Ⅳ（SEO 尾帧）

| # | 追问 | 不合格时 |
|---|------|----------|
| IV1 | 成片是否已**备份无尾帧母片**？ | 先备份再跑 |
| IV2 | 该目录是否**从未跑过**或确定要重跑（避免双尾帧）？ | 读 `append_seo` 说明 |
| IV3 | 正片最后一句人声是否在尾帧**之前**收完？ | 调切片或 cta |

---

## 五、三件套视角（M01l / m / n 快速追问）

**字幕（M01l）**：烧录开关开了吗？SRT 与切片时间轴同一偏移吗？  
**封面（M01m）**：四字段表自检做了吗？与竖屏 Skill §三 冲突吗？  
**特效（M01n）**：裁剪参数、贴片、语速、尾帧 SEO 四项里**哪几项本场启用**？写清了吗？

---

## 六、与剪映手剪并行时

| 追问 | 说明 |
|------|------|
| 程序化成片是否已定稿？ | 定稿后再进剪映做**非破坏**微调 |
| 剪映里改字幕是否回灌 SRT？ | 若回灌，下一场应用脚本应以**新 SRT** 为准，避免双源 |
