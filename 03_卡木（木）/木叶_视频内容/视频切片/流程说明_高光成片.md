# Soul 视频高光成片流程（与 `soul_enhance.py` 对齐）

## 1. 数据准备

| 文件 | 作用 |
|------|------|
| `切片/*.mp4` | 已按 `highlights.json` 的 `start_time`/`end_time` 切好的竖条或横版素材 |
| `highlights.json` | 每条含 `hook_3sec`（封面主文案）、`question`、`title`、`cta_ending`、`start_time` |
| `transcript.srt` | **整场**绝对时间轴字幕；若某时段重复乱码，需对该场 `audio.wav` 重跑 Whisper 后再成片 |

## 2. 高光规则（脚本内已落地）

- **封面大字**：优先 `hook_3sec` → 其次 `question` → 最后 `title`（吸睛、观点感强）。
- **封面视觉**：竖屏为「视频帧强模糊 + 渐变叠层」；横版为双阶高斯模糊。
- **去停顿**：`silencedetect` 检出的静音段用 `trim+concat` 切除，并**整体平移**字幕时间轴。
- **字幕**：默认逐字；相对已导出切片时间轴，音画 PTS 明显错位时才加小幅延迟。
- **片尾**：`cta_ending` 在成片末尾约 2～3.8 秒以字幕条形式固定出现。

## 3. 仅处理与 highlights 对齐的切片

- 文件名需能解析出切片序号（如 `soul127_01_xxx.mp4` → 1）。
- **序号大于 highlights 条数的 mp4 会自动跳过**，避免误跑、误占 CPU。

## 4. 一键命令（第 127 场示例）

见同目录 `脚本/batch_127_vertical.sh`。若 2～8 条仍提示「转录稿异常」，请先对 `audio.wav` 完整转录并覆盖 `transcript.srt`。

## 5. 可选参数

- `--no-trim-silence`：保留长停顿。
- `--title-only`：输出文件名为 Hook 截断后的中文名（便于分发）。
- `--crop-vf`：与 `analyze_feishu_ui_crop.py` 输出一致时粘贴，避免裁偏。
