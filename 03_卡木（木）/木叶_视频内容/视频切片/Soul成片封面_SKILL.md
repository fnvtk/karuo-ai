# Soul 成片 · 封面 Skill

## 追问（读本 Skill 前先答）

- 四条短字段是否已按工作流 §三 表自检（2/4 字、同条异字、全场不撞）？  
- 本场命名规则是「短字」还是竖屏 Skill 注里的「长 title 文件名」模式？**勿混用不自知**。  
- 更全题：`参考资料/成片流程_追问与自检.md` §一-I4、§五。

## 能做什么（Capabilities）

- 把**剪映里「封面」+ 大字标题**对应到 `highlights.json` 字段与 `soul_enhance` 片头卡片。
- 强制 **2 字或 4 字通俗名词、同条/全场不撞字**（与 `Soul派对成片工作流_从零到片尾.md` §三、`Soul竖屏切片_SKILL.md` 一致）。
- 区分：**平台列表封面**（多取成片首帧或平台单独上传） vs **视频内首屏大字**（脚本烧录）。

## 怎么用（Usage）

**触发词**：成片封面、封面大字、hook、前三秒、viral_hook、时间线封面。

**剪映对照**：

- **时间线左侧「封面」**：平台缩略图 → 成片导出后由各发布 Skill 处理；成片内首屏仍由本链路生成。
- **文本轨道大字 / 标题预设** → `viral_hook`（第一优先）→ `hook_3sec` → `title`；`file_stem` 元数据与命名，规则见主工作流 §三。

## 执行步骤（Steps）

1. **在 Ⅰ-7 定稿四字段**：`file_stem`、`viral_hook`、`hook_3sec`、`title`，填表自检无重复字面。
2. **前 3 秒风格**：`Soul竖屏切片_SKILL.md` §2.1 二选一（嘉宾处境优先 / 房主猛句）。
3. **成片命令**：`soul_enhance` 带 `--title-only`（及竖屏 `--vertical` 等）；封面底约 10% 轻模糊见竖屏 Skill。  
4. **竖屏封面字号**：`soul_enhance.py` 内按**可见字数**自适应起始字号（字少更大、字多更小，约 28～100px），仍会在 padding 内自动换行/略降字号防溢出。
4. **若用自动化长标题模式**：以 `Soul竖屏切片_SKILL.md` §3.1 注（v2.14+ `title` 整句文件名）为准，与本 Skill 短字规则**取当前项目启用的一条**，勿混用两套命名而不自知。
