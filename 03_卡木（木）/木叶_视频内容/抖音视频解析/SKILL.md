# 抖音视频解析

> **输入**：抖音视频链接（短链 `v.douyin.com/xxx` 或完整 `www.douyin.com/video/xxx`，短链 path 可含 `-`）或整段「复制打开抖音…」粘贴  
> **输出**：解析出的 ID、**页面文案**（`{aweme_id}_文案.json` / `.txt`）；可选 **口播转写**；**内容分析**一律按下方 **「卡读模板」**。**默认不长期保存 mp4**（见下「存储策略」）

---

## 标准流水线（卡若AI · 强制）

用户给出 **抖音链接 + 复盘 / 读书笔记 / 卡读** 时，Agent **必须先跑通解析与文案验收，再写复盘**，**禁止**跳过脚本直接用网络搜索「代写整条视频理解」。

### 1）向用户说明「链接是什么」（极简即可）

| 形态 | 含义 |
|------|------|
| **短链** `https://v.douyin.com/xxxx/` | App 分享用跳转链接，302 到正式视频页；`xxxx` 可含字母、数字、`-`。 |
| **正式页** `https://www.douyin.com/video/<aweme_id>` | 单条作品页；**`aweme_id` 为一串数字**，是解析与落盘文件名的主键。 |
| **整段「复制打开抖音…」** | 从首条 `https://v.douyin.com/...` 或 `.../video/数字` 自动抽取即可。 |

### 2）卡若AI 提取文案（必跑）

- 默认：`python3 douyin_parse.py '<粘贴>' --no-download`（或 `./douyin_caption_only.sh "URL"`）  
- 页面抽失败时：**再试一次**；仍空则 `--missuo-fallback`（勿商用）；仍失败再 MCP 浏览器辅助，**不得**假装已提取。

### 3）文案验收闸门（通过后才允许卡读/复盘正文）

**须同时满足**才可进入「卡读模板」主分析：

1. 磁盘上存在 `{默认输出目录}/{aweme_id}_文案.json` 与 `_文案.txt`（或用户 `-o` 指定目录）。  
2. **`title` 或 `desc` 至少其一非空**（strip 后长度 > 0），或 JSON 内 `hashtags` 非空且能还原可见文案。  
3. 在回复里 **明示验收结果**：列出 **aweme_id**、**落盘路径**、**已提取标题/描述原文（可节选）或「页面文案为空」**。

若闸门未通过：**主输出只能是**「解析日志 + 已尝试命令 + 客观卡点一句」；卡读模板仅允许写 **§边界说明**，并写明「口播/全文未提取，不复盘实质内容」。

### 4）口播全文（可选、在闸门之后）

用户要 **全片逐字** 或金句必须来自口播时：再按「存储策略」走下载（`--keep-video`）+ `douyin_video_to_text.sh`；转写成功后把 **`导出/*_转写.txt` 路径** 写入回复，再套卡读。

### 5）复盘位置

对话级复盘仍遵守 `运营中枢/参考资料/卡若复盘格式_固定规则.md`；**本条 Skill 的「先提取再复盘」指业务交付顺序**，不替代复盘块。

## 存储策略（重要）

| 场景 | 视频文件 | 文案/转写 | 说明 |
|------|-----------|-----------|------|
| `douyin_parse.py` **默认**（带下载） | 下载校验后 **立即删除** mp4 | 保留 `-o` 目录下 `_文案.json` / `_文案.txt` | 需存档视频时加 **`--keep-video`** |
| `douyin_parse.py --no-download` | 不下载 | 仅文案 | 最轻量 |
| `douyin_video_to_text.sh` 口播转写 | 使用 `/tmp/douyin_extract_*`，脚本结束 **`trap` 整目录删除**（含临时 mp4、临时转写中间文件） | 仅 **`卡若Ai的文件夹/导出/*_转写.txt`** 保留（若不需要可手动删） | 解析阶段对 `douyin_parse` 传 **`--keep-video`**，避免转写前被删；转写后仍由 `trap` 清缓存 |

**原则**：分析/转写完成后 **不长期占用磁盘**；只保留你明确需要的 **文案 JSON/TXT** 或 **口播转写 txt**。

## 核心能力

1. **解析 ID**：从链接或页面提取 `aweme_id`、`video_id`、`file_id`
2. **提取文案**：从页面 metadata 提取标题、正文、话题标签
3. **下载视频（可选）**：拉取无水印 mp4 用于校验或口播链；**默认下载后删除文件**
4. **内容提取与分析**：在文案/口播可用后，按 **卡读模板** 结构化输出（与「卡读书童」对齐，适配短视频）

---

## 卡读 · 抖音文案/视频解析输出模板（强制）

用户要求「复盘 / 分析 / 提取内容」或 Agent 完成解析后需给结论时，**必须**按下列顺序输出（**不向用户追问**；材料不足则基于已有信息写，并一句说明边界）：

### 1）五行读书笔记（按口播/叙事块当「小节」）

| 行 | 含义 | 写什么 |
|----|------|--------|
| **金** | 定位、目标、人设/受众 | 标题钩子、账号定位、话题标签意图、这条想影响谁 |
| **水** | 经历、流程、事件 | 叙事顺序、案例、时间线、转折 |
| **木** | 落地、转化、变现 | 引流标签、CTA、产品/活动位（如「计划 202x」）、可执行建议 |
| **火** | 升级、优化、数据化 | 概念升级、对比、数字/比例、方法论命名 |
| **土** | 放大、合伙、圈层 | 社群/圈子/合作暗示、规模化叙事 |

每条下 **列出对应段落主题**（无章节则按「开场/论证/类比/收尾」拆块）。

### 2）一句话提炼

全书/全片 **一句**，不含空话。

### 3）三个与卡若及私域公司相关的问题

可落地到 **存客宝、IP、社群、合规** 等。

### 4）五个金句 + 五个关键字

金句优先来自口播；可据语义校正 ASR 错字并注明。关键字用于检索与二创。

### 5）以卡若角色与主讲人「讨论」式总结

**卡若**视角点评结构、可信度、风险；**合上**一句可复用或需警惕之处。

### 6）人物与性格（短视频适配）

- **多人叙事**：列出口播中出现的人物，**推测** MBTI、九型、PDP（**主+辅**，如「老虎+孔雀」），并注明「基于文本推测，非测评」。
- **仅单人/口播**：改为 **叙述者人设 + 隐含受众画像**，性格标签仍给主讲人即可。

### 7）执行纪律

- **禁止**因缺信息停住反问；能解析则解析，能转写则转写，再套模板。
- **禁止**在未通过上文「文案验收闸门」时，输出「仿佛已看完视频」的完整卡读；材料不足按 **标准流水线** 写边界。
- 仅有页面文案、无口播时：第 1、4、6 节以 **标题+话题+可见信息** 为主，第 6 节侧重 **人设与受众**。

## 执行步骤

### 用户提供抖音链接或整段分享文案时

1. **解析链接**：从文本中**自动提取**首条 `https://v.douyin.com/...`（支持「复制打开抖音…」整段粘贴）；再识别短链 / 完整链接，提取 `aweme_id`
2. **获取页面**：requests 获取页面（移动端 UA）；失败时可用 MCP 浏览器访问
3. **提取文案**：从页面 title、meta、`__vid`、`ROUTER_DATA` 等提取标题、正文、话题
4. **提取视频 URL**：从 `<source src="...">` 或 JSON 中获取 CDN/Play 链接
5. **下载视频（若未 `--no-download`）**：流式下载；**默认成功后删除 mp4**

### 一键命令（命令行粘贴链接即提取）

```bash
cd /Users/karuo/Documents/个人/卡若AI/03_卡木（木）/木叶_视频内容/抖音视频解析/脚本

# 推荐：仅页面文案，不下载视频
./douyin_caption_only.sh "https://v.douyin.com/xxx"

# 整段分享语（自动抽短链）
python3 douyin_parse.py '复制打开抖音… https://v.douyin.com/xxx-xxx/' --no-download

# 下载校验后删视频，只留文案（默认行为）

# 需要把 mp4 留在 -o 目录时
python3 douyin_parse.py "https://v.douyin.com/xxx" --keep-video

# missuo 兜底（勿商用）
python3 douyin_parse.py "https://v.douyin.com/xxx" --no-download --missuo-fallback
```

- **口播转文字**：`01_卡资（金）/金盾_数据安全/存客宝副本管理/douyin_video_to_text.sh`（MLX-Whisper；临时目录自动清理；产出见上表）

## AI 执行说明（Cursor）

当用户给出抖音链接或「分析/复盘这条抖音」时：

1. 读本 SKILL.md（含 **标准流水线** + **卡读模板**）
2. **提取**：`python3 douyin_parse.py`（整段粘贴可自动抽短链）；需要口播则 `douyin_video_to_text.sh`（见存储策略）
3. **验收**：读 `_文案.json` / `_文案.txt`，按「文案验收闸门」确认 **title/desc 已落盘且有内容**；未通过则不得写满血卡读
4. 若 requests 失败，可用 MCP 浏览器辅助页面信息；**至少两轮**解析尝试（含 `--missuo-fallback` 等）
5. **回复**：先 **解释链接形态 + aweme_id + 落盘路径 + 已提取文案原文（或明确为空）**，通过闸门后再按 **卡读模板 §1～§6**；用户要极简时仍保留模板标题，正文可缩
6. 对话结束按 `运营中枢/参考资料/卡若复盘格式_固定规则.md` 附复盘块
