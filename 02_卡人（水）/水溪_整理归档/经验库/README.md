## 自动捕捉规则

### 什么内容会被识别为"有价值"？

| 类型 | 特征 | 示例 |
|:---|:---|:---|
| **操作步骤** | 包含具体的操作流程 | "先执行A，再执行B，最后C" |
| **解决方案** | 解决了某个具体问题 | "这个bug是因为X，解决方法是Y" |
| **配置方法** | 涉及环境/工具配置 | "MongoDB连接需要设置这些参数" |
| **最佳实践** | 总结出的经验规律 | "做私域项目，先算账再动手" |
| **常用命令** | 重复使用的命令组合 | "部署时总是执行这三条命令" |

### 不捕捉的内容

- ❌ 一次性的简单问答
- ❌ 闲聊内容
- ❌ 敏感信息（密码、密钥）
- ❌ 临时性的数据

## 沉淀流程

### 1️⃣ 自动捕捉

当对话中出现有价值内容时，AI会提示：

```
💡 发现可沉淀经验：[主题]
是否存入经验库？(Y/n)
```

### 2️⃣ 存入待沉淀

用户确认后，自动创建经验文档：

```markdown
# [主题]

**来源**：对话时间 YYYY-MM-DD HH:MM
**分类**：开发经验 / 运维经验 / 商业经验 / 工具技巧
**标签**：#Python #FastAPI #部署

## 问题背景
[当时遇到了什么问题]

## 解决方案
[具体的操作步骤]

## 关键代码/命令
[如有]

## 经验总结
[一句话总结]
```

### 3️⃣ 定期整理

每周末，卡人（水）负责整理经验库：
- 合并相似经验
- 完善文档结构
- 移动到「已整理」目录

### 4️⃣ 转化为Skill

当某类经验积累到3条以上时，卡土（土）负责：
- 分析经验共性
- 提炼成标准化流程
- 生成新的SKILL.md
- 记录到「转化记录.md」

**可选路径：经能力元封装**
- 已整理经验 → **能力元草稿**（`运营中枢/能力元/草稿/`）→ 验证执行 → **能力元入库**（`运营中枢/能力元/已入库/`）→ 继承为 Skill  
- 触发词：`能力元`、`封装能力元`、`继承能力元`  
- 规范：`运营中枢/参考资料/能力元_概念与规范.md`
