# 目录结构最佳实践：平台配置与变更 SOP（2026-02-16）

## 背景

用户提出：将 Gitea、GitHub、模块等目录统一整理，优化目录结构，并形成可复用的目录变更方案。

## 结论：平台约定目录不可移动

`.gitea`、`.github`、`.cursor` 为**平台级约定**，必须位于仓库/工作区根目录：

| 目录 | 平台 | 移动后果 |
|:---|:---|:---|
| `.gitea/` | Gitea | 工单/PR 模板失效 |
| `.github/` | GitHub | Actions 不执行 |
| `.cursor/` | Cursor | 规则不加载 |

因此**不能**将它们移入 `运营中枢/` 等子目录。

## 解决方案

1. **`运营中枢/平台配置/README.md`**：说明各平台目录为何须在根、内容与维护归属
2. **`运营中枢/references/目录结构变更SOP.md`**：做目录重组时的必读清单（更新 SKILL、总索引、规则、脚本等）
3. **根 SKILL.md**：改为快捷入口，完整索引指向 `skill_router/SKILL.md`
4. **00_总索引、karuo-ai.mdc、skill_router/SKILL**：补充平台配置说明与目录变更 SOP 引用

## 后续目录变更时的操作顺序

1. 读 `目录结构变更SOP.md`
2. 确认不移动 `.gitea`、`.github`、`.cursor`
3. 变更后按 SOP 清单同步更新所有引用
