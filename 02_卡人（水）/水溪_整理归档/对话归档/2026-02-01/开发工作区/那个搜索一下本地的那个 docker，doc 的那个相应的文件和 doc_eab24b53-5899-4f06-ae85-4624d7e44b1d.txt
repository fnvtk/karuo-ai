## 一、本机 Docker 目录规范

### 1.1 标准布局（便于可视化）

本机 Docker 项目统一采用以下结构，目录一目了然：

```
项目根目录/
├── docker/                    # Docker 编排（必选）
│   ├── docker-compose.yml     # 主编排文件
│   └── README.md              # 本目录说明（可选）
├── scripts/                   # 运维脚本（可选）
│   ├── run_docker_import.sh   # 一键导入等
│   └── .env.example           # 环境变量模板
└── references/                # 文档（可选）
    └── 本机Docker说明.md
```

**原则**：
- `docker/` 下只放编排相关，不混入业务代码
- 数据卷用 Docker 命名卷，不散落本地目录
- 每个项目一个 `docker-compose.yml`，结构简单

## 一、标准目录模板（融合 GitHub 实践）

### 1.1 推荐结构（参考 MohKari/docker-compose-template、docker/awesome-compose）

```
项目根目录/
├── docker/                    # 编排目录（或根目录 compose）
│   └── docker-compose.yml     # 主编排
├── config/                    # 配置（可选，GitHub 常用）
│   └── *.conf
├── data/                      # 数据挂载（可选，命名卷更推荐）

│   ├── deploy.sh             # 或 run_docker_import.sh
│   └── .env.example          # 环境变量模板
├── .env.sample                # 根目录环境变量模板（GitHub 惯例）
├── .gitignore                 # 排除 .env、data/、logs/
├── README.md                  # 项目说明
└── references/                # 文档（可选）
```

### 1.2 核心原则（来自 awesome-compose、Docker 官方）

| 原则 | 说明 |
|------|------|
| **编排单一** | 每个项目一个 compose 文件，结构简单 |
| **配置分离** | `.env.sample` 模板，`.env` 不提交 |
| **数据卷** | 优先用 Docker 命名卷，不散落本地目录 |
| **脚本统一** | scripts/ 下放 deploy、import、health_check |
| **文档必读** | README.md 说明启动步骤 |

### 1.3 新项目创建检查清单

- [ ] `docker-compose.yml` 或 `docker/docker-compose.yml`
- [ ] `.env.example` 或 `.env.sample`
- [ ] `scripts/` 下至少一个启动/部署脚本
- [ ] `README.md` 含启动命令
- [ ] `.gitignore` 排除 `.env`、`data/`、`logs/`
