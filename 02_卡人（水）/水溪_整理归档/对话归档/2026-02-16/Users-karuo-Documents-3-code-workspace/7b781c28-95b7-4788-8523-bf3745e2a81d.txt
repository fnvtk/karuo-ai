## 五、统一最佳解决方案与推荐链路

### 5.1 目标

- 能**直接优化整个链路步骤**，减少踩坑、一步到位。  
- 区分：**企业微信**（推荐、有现成教程） vs **个人微信**（官方无、需第三方或桥接）。

### 5.2 方案对比（简要）

| 方案 | 适用场景 | 难度 | 推荐度 |
|:---|:---|:---|:---|
| **A. 企业微信直连** | 公司/团队用企微，要 7×24 AI 助手 | 中 | ⭐⭐⭐⭐⭐ |
| **B. 飞书 + 微信桥接** | 已有飞书桥，希望「飞书 ↔ 微信」互通 | 中高 | ⭐⭐⭐⭐（依赖现有飞书桥） |
| **C. 个人微信** | 个人号当机器人 | 高（合规与风控风险） | ⭐⭐（不推荐，仅作了解） |

### 5.3 推荐链路（企业微信 + OpenClaw）

1. **选型**：企业微信（有企业） or 飞书桥 + 微信桥接（已有飞书桥）。  
2. **环境**：云主机（如腾讯 Lighthouse 38 元/月）或本机；Node ≥22 或 Docker。  
3. **安装**：  
   - 云上：`curl -fsSL https://clawd.bot/install.sh | bash` 或 Docker 拉取 `ghcr.io/clawdbot/clawdbot:latest`；  
   - 或：`npm i -g openclaw@latest` → `openclaw onboard --install-daemon`。  
4. **模型**：Qwen（免费）或 Claude/OpenAI（按需）。  
5. **通道**：  
   - 企业微信：企微后台创建应用 → 获取 corpid、corpsecret、token、encodingAESKey → 在 OpenClaw 配置对应 Channel（或 simple-wecom 类插件）；  
   - 对外 URL：Nginx 反代 + SSL，暴露 https://你的域名/企微回调路径。  
6. **配对与测试**：企微里发消息到应用，完成验证与配对。  
7. **Skills**：按需在 onboard 或配置里加 Skills，与飞书桥类似可挂「卡若AI」规则/记忆。

### 5.4 与卡若现有「飞书桥」的衔接

- 飞书桥当前：飞书 → open.quwanzhi.com → frp → 本机 8888 桥接 → 卡若AI 规则/记忆/模型。  
- 手册中提到的 **「飞书与微信桥接操作步骤.md」**：建议补写或从本文档 + 腾讯云企微教程中摘步骤，形成「OpenClaw Gateway + 微信（企微）桥接」一篇，便于与飞书桥并列使用、统一优化链路。  
- 若要做「个人微信」：仅建议通过「飞书桥 + 某中间层转微信」的方式了解，不推荐直接违规协议。

## 一、用大白话讲清楚

**你要的是啥**：给微信（或企业微信）接一个 7×24 在线的 AI 助手，用的是 OpenClaw（以前叫 Clawdbot/Moltbot）。

**先说结论**：
- **个人微信**：官方不支持，别指望「个人号当机器人」一步到位。
- **企业微信**：可以，而且全网教程多，推荐做。
- 数据从哪来：我们帮你从**抖音对应的全网教程**（5 篇热门）、**GitHub 官方**、**学习站 Skills** 里筛过一遍，汇成下面这一套。

## 一、结论先说

- **个人微信**在这里 = 存客宝里绑定的**个人微信号**（你已在存客宝设备上登录的微信）。
- 存客宝提供：**主站 HTTP API**（登录、微信号列表、好友列表、场景/工作台等）+ **私域管理 s2 的 WebSocket**（发消息、后续可扩展收消息）。

- **收消息**：当前 SDK 未实现。要「个人微信 ↔ AI 对话」有两种路：
  - **方案 A**：存客宝 s2 若支持「新消息 WebSocket 推送」或「拉取新消息 HTTP 接口」→ 本机服务收消息 → 调卡若AI/飞书桥同款逻辑 → 用 SDK 发回。
  - **方案 B**：本机 Mac 登录同一微信，用**微信管理** Skill 读本地微信数据库（只读新消息），再调 AI 后用存客宝 SDK 或本地工具回发（适合本机为主场景）。

## 二、开发与对接流程（按顺序）

### 2.1 准备材料（本机已有）

| 材料 | 位置 | 用途 |
|:---|:---|:---|
| 存客宝账号 | 账号与API索引 / 你已知 | 主站登录 + 可选 s2 WebSocket |
| 存客宝私域SDK | `金盾/存客宝私域SDK/scripts/` | 登录、微信号/好友、发消息 |
| config | `存客宝私域SDK/scripts/config.py` | API 地址、环境变量 |
| 微信管理 | `金盾/微信管理/` | 本机微信数据分析（方案 B 收消息可选） |

### 2.2 对接流程概览

```
┌─────────────────────────────────────────────────────────────────────────┐
│  1. 配置凭证    2. 主站登录    3. 取微信号/好友    4. 发消息(WS)          │
│       │              │                │                    │            │
│       ▼              ▼                ▼                    ▼            │
│  CKB_USERNAME   POST /v1/auth/login  GET /v1/wechats    WS Token        │
│  CKB_PASSWORD   → JWT                GET /v1/wechats/{id}/friends      │
│  (可选 WS 验证码)                     解析 list/data       → CmdSendMessage │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.3 步骤 1：配置凭证

- **主站登录**（必做）：环境变量 `CKB_USERNAME`、`CKB_PASSWORD`（存客宝登录手机号 + 密码）。
- **发消息**（必做）：同上；若 s2 的 Token 接口需要验证码，需把 `verifysessionid`、`verifycode` 配到可改处（当前在 `ckb_sdk.py` 里写死，见下「配置方式」）。

### 2.4 步骤 2：主站登录

```python
from ckb_sdk import CunkebaoSDK
sdk = CunkebaoSDK()  # 从环境变量读 CKB_USERNAME / CKB_PASSWORD
if sdk.login():
    print(sdk.token_info.account_id)
```

- 接口：`POST https://ckbapi.quwanzhi.com/v1/auth/login`，body：`account`, `password`, `typeId=1`。
- 返回 JWT 写入 `token_info`，后续请求带 `Authorization: Bearer <token>`。

### 2.5 步骤 3：获取微信号与好友

- 微信号列表：`sdk.get_wechat_accounts()` → 列表每项含 `id`, `nickname`, `wechatId` 等。
- 好友列表：`sdk.get_wechat_friends(wechat_id)`（wechat_id 为上面返回的 `id`）→ 每项含 `id`, `nickname`, `remark` 等；**发消息时用这里的 `id` 作为 wechat_friend_id**。

### 2.6 步骤 4：发送消息（个人微信发出）

- 先取 **WebSocket Token**：`POST https://s2.siyuguanli.com:9991/token`（参数与 Headers 见 SKILL）。
- 再连 **WebSocket**：`wss://s2.siyuguanli.com:9993`，发 `CmdSignIn` 登录，再发 `CmdSendMessage`（wechatAccountId, wechatFriendId, wechatChatroomId=0, msgType=1, content）。
- SDK 已封装：`sdk.send_message(wechat_account_id=微信号id, wechat_friend_id=好友id, content="内容", msg_type=1)`。
- **风控**：发送间隔 3–8 秒、单号每日约 200 条、同一目标 60 秒内不重复发。

### 2.7 收消息（实现「个人微信 + AI」的关键）

- **当前**：SDK 未实现「拉取新消息」或「订阅新消息」。
- **若存客宝 s2 支持**：在同一 WebSocket 上会有新消息下行（如 `CmdNewMessage` 之类），则本机可：长连 WS → 收到新消息 → 调 AI → 用 `send_message` 回发。
- **若存客宝提供 HTTP 拉取**：本机轮询「未读/新消息」接口 → 同上 AI → 发回。
- **方案 B**：本机 Mac 登录该微信，用「微信管理」读本地库新消息（需解密与轮询），再调 AI，回发用存客宝 SDK 或本机自动化。

## 五、个人微信 + AI 的完整方案（能直接聊天实现功能）

目标：**用户给存客宝绑定的个人微信号发消息 → 自动得到 AI 回复**。

### 5.1 架构示意

```
用户发微信 → 存客宝设备收到 → [ 消息到达本机 ] → AI 处理 → 存客宝 SDK 发回微信
                                    ↑
                    方式1: s2 WebSocket 推送（若支持）
                    方式2: s2 HTTP 拉取新消息（若提供）
                    方式3: 本机微信数据库轮询（微信管理，仅本机登录时）
```

### 5.2 推荐实现顺序

1. **先打通「发」**：用现有 SDK + 上面配置，能稳定用 `send_message` 从指定微信号发给指定好友/群。
2. **再确认「收」**：  
   - 查存客宝/s2 文档或接口是否有「新消息推送」或「拉取未读消息」；  
   - 若有，在本机起一个**桥接服务**：收消息 → 调卡若AI（与飞书桥同款：规则+记忆+模型）→ 调 `sdk.send_message` 回发。  
   - 若无，用**方案 B**：本机登录该微信 + 微信管理读库 + 同上桥接逻辑。
3. **配置**：用上面 YAML 管理「用哪个微信号做 AI 回复、白名单、风控、AI 模型」。

### 5.3 桥接服务要做的事（伪代码）

- 启动：加载配置 → 存客宝登录 → 若用 WS 收消息则建立长连，若用轮询则定时拉取。
- 收到一条新消息：解析出发送人（wechat_friend_id 或 wxid）、微信号（wechat_account_id）、内容。
- 白名单校验：若配置了白名单，仅白名单内好友/群才进入 AI。
- 调 AI：与飞书桥一致（读规则+记忆，调 v0/ollama/openai），得到回复文本。
- 风控：间隔、日限、同目标间隔。
- 发回：`sdk.send_message(wechat_account_id, wechat_friend_id, reply_text, msg_type=1)`。

### 5.4 和你电脑上材料的对应关系

| 你本机材料 | 用途 |
|:---|:---|
| 存客宝私域SDK（ckb_sdk, config） | 登录、取号/好友、发消息 |
| 账号与API索引 | 存客宝账号、v0/Gemini 等 API |
| 飞书桥逻辑（规则+记忆+模型） | 复用到「个人微信 AI 回复」 |
| 微信管理（本地库解密、导出） | 方案 B 下「收消息」来源 |
| 上面 YAML 配置模板 | 桥接服务配置、白名单、风控 |

文档里按「准备材料 → 登录 → 取号/好友 → 发消息 → 收消息」整条链路都写清楚了，并说明了和你本机材料（SDK、账号索引、飞书桥逻辑、微信管理）的对应关系。

## 4. 配置方式

- **最少必配**：环境变量 `CKB_USERNAME`、`CKB_PASSWORD`。
- **可选**：`CKB_API_URL`、`CKB_WS_URL`、`CKB_TOKEN_URL`；若 s2 要验证码，可后续在 `config`/`ckb_sdk` 里用 `CKB_WS_VERIFY_SESSION_ID`、`CKB_WS_VERIFY_CODE` 等注入（文档里已说明）。
- **集中配置**：用 `个人微信_对接配置模板.yaml`，可放在 `~/.config/ckb_wechat_bridge/config.yaml` 或项目内，填风控、白名单、AI 模型等；账号密码建议仍用环境变量，不写进文件。

## 5. 个人微信 + AI 方案（能直接聊天实现功能）

- **目标**：用户给存客宝绑定的个人微信号发消息 → 自动得到 AI 回复。
- **实现思路**：  
  - **发**：已具备，用现有 SDK 的 `send_message`。  
  - **收**：若 s2 支持新消息推送或拉取 → 本机桥接收消息 → 用与飞书桥相同的规则+记忆+模型调 AI → 用 SDK 把回复发回该好友/群；若不支持，则用本机微信 + 微信管理读库作为「收」的来源（方案 B）。  
- **配置**：用上面的 YAML 指定「用哪个微信号做 AI 回复」、白名单、风控、AI 的 provider/model 等；文档里有桥接逻辑的伪代码和与飞书桥的复用说明。
