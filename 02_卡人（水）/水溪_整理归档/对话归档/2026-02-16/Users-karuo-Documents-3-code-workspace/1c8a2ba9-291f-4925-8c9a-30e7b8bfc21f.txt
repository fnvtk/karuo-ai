## 二、与服务端、中间层、设备端的交互规则

### 2.1 调用链（以业务为主）

```
存客宝业务（如「给某好友发微信消息」）
    → 中间层：WorkPhoneSDK::sendMessage(device_id, 'wechat', to_id, content)
    → 服务端：POST /api/v3/unified/message/send { device_id, platform, to_id, content }
    → 服务端：通道选择 → 组包 { script: 'wechat', action: 'send_message', params }
    → 设备端：WebSocket 收 execute → Skill wechat.send_message(params) → response 回传
    → 服务端 → 中间层 → 业务
```

### 2.2 本 Skill 规定的开发/对接顺序

1. **先定业务**：在交互矩阵里确认「业务动作 ↔ 服务端接口 ↔ 设备端 action」一致。
2. **服务端**：unified 已具备的接口直接对中间层暴露；新增业务时在 unified 增加路由，并在矩阵中登记。
3. **中间层**：存客宝 Server 使用 `WorkPhoneSDK` 调用服务端；新接口需在 PHP/TS SDK 中增加对应方法，并在矩阵中登记。
4. **设备端**：新平台或新动作需在 `agent/skills/{platform}/` 实现对应 action，与 unified 的 script/action 命名一致，并在矩阵中登记。
5. **联调**：按「中间层 → 服务端 → 设备端」顺序验证；本 Skill 的 references 提供矩阵与排错要点。

### 2.3 相关代码与配置位置

| 层级     | 位置 | 说明 |
|----------|------|------|
| 服务端   | `工作手机/sdk/app/routers/unified.py` | 统一接口；通道选择、_execute_skill、官方 API/AI 占位 |
| 服务端   | `工作手机/sdk/app/services/ws_hub.py` | WebSocket 设备连接、send_command |
| 服务端   | `工作手机/sdk/app/services/device_manager.py` | 设备/命令持久化 |
| 中间层   | `cunkebao_v3/Server/application/common/util/WorkPhoneSDK.php` | 存客宝 PHP SDK |
| 中间层   | `cunkebao_v3/Server/config/workphone.php` | SDK 地址、api_key |
| 设备端   | `工作手机/sdk/agent/agent.py` | 收 execute/agent_execute，分发到 Skill |
| 设备端   | `工作手机/sdk/agent/skills/{wechat,douyin,xhs,xianyu}/` | 各平台 Skill 实现 |
