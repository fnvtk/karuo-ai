## 一、合规边界（必守）

| 类型 | 说明 |

## 二、单账号用量与账单分析（方法论）

适用于：Cursor、Claude/Anthropic、OpenAI 等按用量计费的 AI 服务，**仅限本人账号**。

### 1. 用量通常在哪里看

- **Cursor**：Settings → Subscription / Usage 或官网账号页的用量/账单区。
- **Anthropic (Claude)**：账号中心 → Usage / Billing。
- **OpenAI**：Usage 页面、Billing 历史。

通用思路：登录 **本人** 账号 → 找到 Billing / Usage / Subscription → 看当前周期用量、历史账单。

### 2. 账单分析可看的维度（供小说或自用参考）

| 维度 | 说明 |
|:---|:---|
| 周期内用量 | 当前计费周期已用 Token 数或请求数。 |
| 费用分布 | 按模型（如 Claude Sonnet/Opus）、按时间段的费用占比。 |
| 高峰时段 | 哪几天、哪类任务用量集中。 |
| 单次对话成本 | 粗略用「总费用/请求数」或「总费用/会话数」看平均成本。 |

### 3. 超出用量时的合规解决方式

- **升级套餐**：改为更高档订阅或按量付费，在服务商规则内提高额度。
- **分流到本地模型**：轻量任务用本地 Ollama（如 qwen2.5）等，减少云端 Token 消耗。
- **优化使用方式**：缩短上下文、合并请求、用摘要代替长原文，降低单次调用 Token。
- **多模型分工**：简单任务用便宜/本地模型，复杂任务再用 Claude/Opus（符合各产品使用政策）。

不做：多账号轮换、共享账号、自动化多账号池化以规避单账号限制。

## 三、高低用量账号对比的叙事结构（小说大纲参考）

1. **现状呈现**  
   角色打开用量报表/账单页，看到多个订阅的列表；用 1–2 个具体数字点出「有个别账号明显偏高」。

2. **对比与疑问**  
   列出「高用量账号」与「普通账号」在用量、费用、主要用途上的差异；角色或同事提出「为什么 A 这么高？」。

3. **原因揭示**  
   通过对话或回忆交代：新项目、新需求、某类任务特别耗 Token 等（均不涉及多账号或违规使用）。

4. **合规应对**  
   角色提出或执行：升级该账号套餐、把部分任务转到本地模型、优化 prompt 等；可加入「老板要求控成本」等冲突。

5. **结果与余韵**  
   下个周期用量回落，或留下「以后每月都要看账单」的伏笔；可穿插技术细节（Token、模型名）增强真实感。
