> 用 **open.quwanzhi.com**（OPPO/公寓网域名）作为时间机器备份目标，直接备份到 NAS，并实现自动备份与读取。

## 一、前提说明

| 项目 | 说明 |
|------|------|
| **NAS** | 192.168.1.201，域名 **open.quwanzhi.com**（同一台群晖） |
| **共享** | NetBackup（/volume1/NetBackup），需在 DSM 启用 Time Machine |
| **内网** | Mac 与 NAS 需在同一局域网；时间机器走 SMB，仅内网可用 |
| **域名解析** | 本机需能解析 open.quwanzhi.com → 192.168.1.201（见下） |

---

## 二、本机域名解析（必做一次）

时间机器通过 **open.quwanzhi.com** 连接 NAS 时，Mac 必须能解析到 192.168.1.201。

**方式 A：内网 DNS 已配置**  
若路由器或公寓网已把 `open.quwanzhi.com` 指到 `192.168.1.201`，可跳过本步。

**方式 B：本机 hosts（推荐）**

```bash
# 编辑 hosts（需管理员权限）
sudo nano /etc/hosts

# 在文件末尾增加一行：
192.168.1.201   open.quwanzhi.com
```

保存后执行 `ping open.quwanzhi.com`，应能通且显示 192.168.1.201。

---

在终端执行（使用公寓网域名）：

**过程简述**：  
脚本会检查 open.quwanzhi.com 连通性 → 打开 DSM 配置页 → 挂载 `smb://fnvtk@open.quwanzhi.com/NetBackup` → 把凭据写入钥匙串 → 设置时间机器目标。首次会打开 Finder 挂载，按提示即可。

## 五、自动备份与读取

### 自动备份

- **系统行为**：配置完成后，时间机器会按系统设置自动备份（如「下次备份: 自动(每小时)」）。
- **条件**：Mac 与 NAS 在同一网络、NetBackup 已挂载或可连接；钥匙串中已有 SMB 凭据（脚本已写入）。
- **手动触发**：菜单栏时间机器图标 → **立即备份**。

### 读取 / 恢复

1. **进入时间机器**：菜单栏时间机器图标 → **进入时间机器**。
2. **选时间点**：右侧时间轴选择日期/时间。
3. **恢复**：选中文件或文件夹 → 点击 **恢复**。
4. 若备份目标是 **open.quwanzhi.com**，请先保证在同一局域网；若未挂载，可先 访达 → 连接服务器 → `smb://open.quwanzhi.com/NetBackup` 再进时间机器。

## 六、当前显示「正在等待完成第一次备份」时

若时间机器里已出现 **NetBackup - 192.168.1.201** 或 **NetBackup - open.quwanzhi.com**，但一直显示「正在等待完成第一次备份」：

1. **改用域名（可选）**：若希望统一用域名，可运行上面「一键配置（使用域名）」；或保留 IP 目标，两者都是同一 NAS，选其一即可。

## 一、按顺序执行（推荐）

### 步骤 1：停止当前备份

- 若有进度条在转：点击进度条旁的 **X** 停止。
- 或运行：`tmutil stopbackup`

### 步骤 2：补全钥匙串凭据（最常见原因）

backupd 按「目标地址」取密码。若目标是 **NetBackup - 192.168.1.201**，钥匙串里必须有 **192.168.1.201** 的 SMB 凭据；若用 **open.quwanzhi.com**，则要有 **open.quwanzhi.com** 的凭据。缺或错都会导致「未知错误」。

**在终端执行（一次性写入 IP 和域名两份，避免未知错误）：**

```bash
# NAS 账号（与 DSM 一致）
NAS_USER="fnvtk"
NAS_PASS="zhiqun1984"

# 为 192.168.1.201 写入凭据（供 NetBackup - 192.168.1.201 使用）
security delete-internet-password -s "192.168.1.201" -r "smb " 2>/dev/null || true
security add-internet-password -a "$NAS_USER" -s "192.168.1.201" -r "smb " -w "$NAS_PASS" \
  -T /usr/sbin/backupd \
  -T /System/Library/CoreServices/backupd.bundle/Contents/Resources/backupd

# 为 open.quwanzhi.com 也写一份（若你用域名或之后改用 open）
security delete-internet-password -s "open.quwanzhi.com" -r "smb " 2>/dev/null || true
security add-internet-password -a "$NAS_USER" -s "open.quwanzhi.com" -r "smb " -w "$NAS_PASS" \
  -T /usr/sbin/backupd \
  -T /System/Library/CoreServices/backupd.bundle/Contents/Resources/backupd

echo "✓ 凭据已写入（192.168.1.201 与 open.quwanzhi.com）"
```

执行时若提示输入 Mac 登录密码，输入即可。

### 步骤 3：清理 NAS 上未完成的备份目录（若曾中断过）

4. 断开连接。

或在 DSM 文件站中打开 `/volume1/NetBackup/Backups.backupdb/...` 做同样删除。

### 步骤 4：确认 NAS 端 Time Machine 已开启

## 〇、「备份磁盘不可用」与「至少保留 3 天备份」

### 备份磁盘不可用（192.168.1.201 不可用）

- **原因**：Mac 与 NAS 不在同一网络（如外出、Wi‑Fi 切换）、NAS 关机/睡眠，或 SMB 未连上。
- **处理**：Mac 与 NAS 在同一局域网后，时间机器会**自动重试**；无需操作。若经常出现，请确保：① NAS 和路由器常开；② 本机已运行过 `时间机器_修复NetBackup凭据.command`（钥匙串有 192.168.1.201 的 SMB 凭据）；③ 在时间机器里点「立即备份」触发一次。

### 本机时间机器保留至少 3 天备份

- 系统默认策略：**过去 24 小时的每小时备份 + 过去约 30 天的每日备份 + 更早的每周备份**，空间不足时从最早删除。因此**至少 3 天的备份**在首次完整备份完成后即会满足，**无需额外设置**。
- 若要保证有 3 天可恢复：确保 NetBackup 目标可用并完成过至少一次完整备份，之后保持自动备份开启即可。

## 一、当前生效的备份架构

| 层级 | 存储位置 | 恢复速度 | 保留时长 | 是否需要联网 |
|------|----------|----------|----------|-------------|
| **本地快照** | Mac 内置硬盘 | 秒级 | 3 天+（每小时一次，空间够就保留更久） | 不需要 |
| **NAS 备份** | 192.168.1.201 / NetBackup | 分钟级 | 数月至数年（15.5TB 可用） | 需要局域网 |

### 本地快照（已自动开启）

- macOS 在时间机器开启后**自动创建本地快照**（每小时一次）。
- 即使**不在局域网、外出、没有 NAS**，也能从本地快照恢复最近 3 天内的文件。
- 存储在 Mac 内置硬盘上，不占额外空间（空间不足时从最早删除）。
- 恢复方式：菜单栏时间机器图标 → **进入时间机器** → 选时间点 → 恢复。

### NAS 长期备份（已触发并在运行）

- 时间机器每小时自动备份到 NAS（AutoBackup = 1, 间隔 3600 秒）。
- NAS 上保留**完整历史**：24 小时内每小时一次，30 天内每天一次，更早的每周一次。
- 空间：15.5TB 可用，足够保留数年备份。
- 需要 Mac 与 NAS 在同一局域网（192.168.1.201 可达时自动备份）。

## 二、在外网（不在局域网）时的备份

### 已有保障

- **本地快照**自动在内置硬盘上创建，不依赖 NAS，保留 3 天+。
- 外出时若需恢复文件，直接「进入时间机器」从本地快照恢复即可。

### 不建议通过外网做时间机器备份

- 时间机器走 SMB 协议，对延迟和稳定性要求高，**外网（WAN）环境不稳定会导致备份中断、数据损坏**。
- 建议：**外出时靠本地快照**；**回到局域网后自动从 NAS 续备**。

### 如果确实需要外网备份（非必需，可选）

- 可通过 **VPN 回连到家庭局域网**后，等同于在局域网内，时间机器会自动识别并备份。
- 或使用 Synology 的 **Hyper Backup** 做补充（单独配置，不走时间机器）。

## 三、恢复文件

### 3 天内的文件（本地快照，最快）

1. 菜单栏时间机器图标 → **进入时间机器**
2. 右侧时间轴选 3 天内的时间点
3. 选中文件/文件夹 → 点 **恢复**
4. **不需要连 NAS**，本机即可完成

### 更久的文件（NAS 备份）

1. 确保 Mac 与 NAS 在同一局域网
2. 菜单栏时间机器图标 → **进入时间机器**
3. 右侧时间轴可选更早的日期（数月甚至数年前）
4. 选中文件 → **恢复**

## 四、检查备份状态

### 快速检查（终端）

```bash
# 查看时间机器状态（是否在运行、百分比）
tmutil status

# 查看备份目标
tmutil destinationinfo

# 查看本地快照列表（验证 3 天内有快照）
tmutil listlocalsnapshots /

# 手动创建一个本地快照
tmutil localsnapshot

# 手动触发一次 NAS 备份
tmutil startbackup
```

### 图形界面检查

- **系统设置** → **通用** → **时间机器**：查看各备份目标状态、最后备份时间。
- 菜单栏时间机器图标：快速查看「下次备份」时间或「立即备份」。
