## 一、Opus 4.6 深度能力拆解

### 1.1 核心升级概览

| 能力域 | 4.5 | 4.6 | 对卡若AI 的意义 |
|:---|:---|:---|:---|
| **规划与执行** | 基础 | 更细致规划、更长 agentic 任务、更大代码库 | 多助手协作、长任务执行 |
| **上下文** | 200K | **1M token**（Beta）+ **Compaction** | 长对话、多轮协作、记忆压缩 |
| **推理控制** | budget_tokens | **Adaptive thinking** + **Effort** | 按任务难度动态分配推理深度 |
| **Agent 协作** | 单 Agent | **Agent teams**（Claude Code） | 五行并行、跨组协调 |
| **自检能力** | 一般 | 更强 code review、catch 自己的错误 | 验证、回溯、经验沉淀 |

---

### 1.2 Adaptive Thinking（自适应思考）

**机制**：Claude 根据请求复杂度**动态决定**是否深度推理、推理多久，无需手动设 `budget_tokens`。

| 级别 | 行为 | 适用场景 |
|:---|:---|:---|
| **max** | 无限制深度推理（4.6 独有） | 技术攻关、架构决策 |
| **high**（默认） | 几乎总是深度思考 | 复杂拆解、多 Skill 协作 |
| **medium** | 适中推理，简单问题可跳过 | 平衡速度与质量 |
| **low** | 最小化推理，快速响应 | 子任务、摘要、分类 |

**与工具调用的结合**：支持 **interleaved thinking**（工具调用之间穿插思考），尤其适合 agentic 工作流。

**卡若映射**：
- **思考→拆解**：高难度任务用 `high/max`，简单任务用 `low`，与卡若的「难度评估」天然对齐。
- **多 Skill 协作**：火眼追问、火锤修复、卡人协调等涉及多步推理，适合 adaptive thinking。

---

### 1.3 Effort 参数（努力程度）

**机制**：控制模型在**全部 token**（文本、工具调用、思考）上的投入程度。

| Effort | 说明 | 卡若典型场景 |
|:---|:---|:---|
| **max** | 最高能力，无约束 | @卡火 技术攻关、架构决策 |
| **high** | 默认，高质量 | 复杂需求拆解、多助手协作 |
| **medium** | 平衡速度与成本 | 日常任务规划、周报生成 |
| **low** | 最省 token，偏速度 | 摘要、分类、查表、批量处理 |

**与工具调用的关系**：
- 低 effort：更少工具调用、更简洁、直接行动。
- 高 effort：更多工具调用、先解释再执行、总结更详细。

**卡若映射**：
- 卡若的「难度 ≤ 3 → 本地 qwen2.5；> 3 → 高级模型」可扩展为「高级模型 + effort 分级」。
- 例如：难度 4 → medium，难度 5 → high，技术攻关 → max。

---

### 1.4 Context Compaction（上下文压缩）

**机制**：当输入 token 接近阈值（默认 150K，最低 50K）时，自动将**旧对话** summarization 成 compact block，后续只保留摘要，实现「类无限长对话」。

**关键参数**：
- `trigger`：触发压缩的 token 数
- `instructions`：自定义摘要 prompt（可强调保留代码片段、技术决策等）
- `pause_after_compaction`：压缩后暂停，便于在继续前插入额外内容

**卡若映射**：
- **长任务会话**：神射手 + 上帝之眼 + 多助手协作可能产生大量上下文，Compaction 可避免 context rot。
- **经验沉淀**：压缩时可自定义 `instructions`，强调保留「definitions_of_done、lessons_learned、验证结果」。
- **飞书/企微长对话**：用户与卡若长期对话，Compaction 可维持连贯性。

---

### 1.5 1M Token 上下文（Beta）

**机制**：Opus 4.6 首次支持 1M token 上下文；超过 200K 有溢价（$10/$37.5 每百万 token）。

**卡若映射**：
- **全索引 + 多 Skill 一次性加载**：可考虑在「技术攻关」「全栈开发」等场景加载 00_卡若AI总索引 + 相关 Skill。
- **上帝之眼**：大量历史数据、策略回测结果可放入上下文。
- **成本权衡**：常规任务仍用 200K，仅高价值长任务用 1M。

---

### 1.6 Agent Teams（Claude Code）

**机制**：多个 agent 并行工作，自主协调，适合「可拆成独立子任务」的场景（如代码库审查）。

**卡若映射**：
- **五行并行**：金/水/木/火/土 各组可视为独立 agent，卡人（水）为协调枢纽。
- **跨组协调会**：Agent teams 的「协调自治」与卡人的跨组协调中心高度一致。
- **技术攻关会**：多专员并行排查不同模块，卡火汇总决策。

---

### 1.7 自检与回溯

**机制**：4.6 在 code review、debugging 上更强，能更好地发现自己的错误。

**卡若映射**：
- **验证不通过 → 回溯思考**：与 4.6 的自检能力叠加，可强化「验证→回溯→再执行」流程。
- **definitions_of_done**：每步完成标准检查，4.6 更易主动发现未达标项。
