# 群晖NAS管理

统一管理卡若私域数字底座的NAS矩阵，支持DSM、Docker、MongoDB、Qdrant、宝塔中控等核心服务。

## 核心环境

### NAS节点信息

| 节点 | IP地址 | 主机名 | 用途 |
|------|--------|--------|------|
| **NAS-2 (主节点)** | `192.168.110.101` | CKBNAS | 存储与容器管理主控 |
| **NAS-1 (备用)** | `192.168.2.200` | - | 管理节点 |

### 账号凭证

```
# NAS-2 主节点
主账户: fnvtk / Zhiqun1984
备用账户: admin / zhiqun1984

# NAS-1 备用节点  
账户: admin / zhiqun1984
```

### 关键端口

| 端口 | 服务 | 地址示例 |
|------|------|----------|
| 5000 | DSM管理界面 | http://192.168.110.101:5000 |
| 27017 | MongoDB | mongodb://admin:admin123@192.168.110.101:27017 |
| 6333 | Qdrant向量库 | http://192.168.110.101:6333 |
| 8890 | bt-hub中控 | http://192.168.110.101:8890 |
| 8888 | 本地宝塔 | http://192.168.110.101:8888 |

## 核心功能

### 1. SSH连接管理

群晖需要启用旧版SSH算法支持：

```bash
# 标准SSH连接命令
ssh -o KexAlgorithms=+diffie-hellman-group1-sha1 \
    -o Ciphers=+aes128-cbc,3des-cbc,aes192-cbc,aes256-cbc \
    -o StrictHostKeyChecking=no \
    fnvtk@192.168.110.101

# 使用sshpass自动登录（需安装：brew install sshpass）
sshpass -p 'Zhiqun1984' ssh -o KexAlgorithms=+diffie-hellman-group1-sha1 \
    -o Ciphers=+aes128-cbc,3des-cbc,aes192-cbc,aes256-cbc \
    fnvtk@192.168.110.101
```

### 2. Docker容器管理

群晖的Docker命令路径特殊，必须使用绝对路径：

```bash
# Docker命令路径
DOCKER_CMD="/volume1/@appstore/ContainerManager/usr/bin/docker"

# 查看所有容器
$DOCKER_CMD ps -a

# 查看容器日志
$DOCKER_CMD logs <container_name>

# 重启容器
$DOCKER_CMD restart <container_name>

# 进入容器
$DOCKER_CMD exec -it <container_name> /bin/bash
```

### 3. MongoDB管理

```bash
# 连接字符串
mongodb://admin:admin123@192.168.110.101:27017/

# 通过Docker进入MongoDB
$DOCKER_CMD exec -it mongodb mongosh -u admin -p admin123 --authenticationDatabase admin

# 常用命令
show dbs
use <database>
show collections
db.<collection>.find().limit(10)
```

### 4. 系统状态检查

```bash
# 内存使用
```

### synology-api 示例

```python
from synology_api import filestation, downloadstation

# 连接FileStation
fs = filestation.FileStation(
    ip_address='192.168.110.101',
    port='5000',
    username='fnvtk',
    password='Zhiqun1984',
    secure=False,
    cert_verify=False
)

# 列出共享文件夹
shares = fs.get_list_share()
print(shares)

# 上传文件
fs.upload_file('/volume1/homes/fnvtk/', '/local/path/file.txt')

# 下载文件
fs.get_file('/volume1/homes/fnvtk/file.txt', 'download')
```

### py-synologydsm-api 异步示例

```python
import asyncio
from synology_dsm import SynologyDSM

async def main():
    api = SynologyDSM(
        session=None,
        host='192.168.110.101',
        port=5000,
        username='fnvtk',
        password='Zhiqun1984',
        use_https=False
    )
    
    await api.login()
    
    # 获取系统信息
    info = api.information
    print(f"型号: {info.model}")
    print(f"序列号: {info.serial}")
    print(f"温度: {info.temperature}°C")
    
    await api.logout()

asyncio.run(main())
```

## 常见问题

### Q1: SSH连接失败

```bash
# 错误: no matching key exchange method found
# 解决: 添加旧算法支持
ssh -o KexAlgorithms=+diffie-hellman-group1-sha1 -o Ciphers=+aes128-cbc ...

# 或在 ~/.ssh/config 中配置
Host nas
    HostName 192.168.110.101
    User fnvtk
    KexAlgorithms +diffie-hellman-group1-sha1
    Ciphers +aes128-cbc,3des-cbc,aes192-cbc,aes256-cbc
```

### Q2: Docker命令找不到

```bash
# 群晖Docker路径不在PATH中，必须用绝对路径
/volume1/@appstore/ContainerManager/usr/bin/docker ps
```

### Q3: 权限不足

```bash
# 部分操作需要sudo
echo 'Zhiqun1984' | sudo -S <command>

# 或将用户加入docker组
sudo synogroup --add docker fnvtk
```

### Q4: 无法访问DSM

```
1. 检查NAS是否开机
2. 确认IP地址正确（可能DHCP变化）
3. 检查防火墙设置
4. 尝试用主机名访问: http://CKBNAS:5000
```

"""
MongoDB连接信息获取脚本
从群晖NAS的Docker容器中提取MongoDB配置
"""

import subprocess
import json

# NAS配置
NAS_IP = "192.168.110.101"
NAS_USER = "fnvtk"
NAS_PASSWORD = "Zhiqun1984"
DOCKER_CMD = "/volume1/@appstore/ContainerManager/usr/bin/docker"

# SSH选项
SSH_OPTIONS = [
    "-o", "StrictHostKeyChecking=no",
    "-o", "ConnectTimeout=10",
    "-o", "KexAlgorithms=+diffie-hellman-group1-sha1",
    "-o", "Ciphers=+aes128-cbc,3des-cbc,aes192-cbc,aes256-cbc"
]

def run_ssh(command):
    """通过SSH执行命令"""
    ssh_cmd = ["sshpass", "-p", NAS_PASSWORD, "ssh"] + SSH_OPTIONS + [f"{NAS_USER}@{NAS_IP}", command]
    try:
        result = subprocess.run(ssh_cmd, capture_output=True, text=True, timeout=30)
        return result.stdout, result.stderr, result.returncode
    except Exception as e:
        return "", str(e), -1

def main():
    print("=" * 60)
    print("MongoDB连接信息获取")
    print("=" * 60)
    print(f"NAS: {NAS_IP}")
    print()
    
    # 查找MongoDB容器
    print("【查找MongoDB容器】")
    stdout, stderr, code = run_ssh(f"{DOCKER_CMD} ps -a --format '{{{{.Names}}}}|{{{{.Image}}}}|{{{{.Status}}}}'")
    
    if code != 0:
        print(f"❌ 无法连接NAS: {stderr}")
        return
    
    containers = [c for c in stdout.strip().split('\n') if c and 'mongo' in c.lower()]
    
    if not containers:
        print("❌ 未找到MongoDB容器")
        return
    
    for container in containers:
        name, image, status = container.split('|')
        print(f"✓ 找到: {name} ({image})")
        
        # 获取容器详情
        stdout, _, code = run_ssh(f"{DOCKER_CMD} inspect {name}")
        if code != 0:
            continue
            
        info = json.loads(stdout)[0]
        
        # 提取环境变量
        env = {e.split('=')[0]: e.split('=', 1)[1] for e in info.get('Config', {}).get('Env', []) if '=' in e}
        username = env.get('MONGO_INITDB_ROOT_USERNAME', 'admin')
        password = env.get('MONGO_INITDB_ROOT_PASSWORD', 'admin123')
        
        # 提取端口
        ports = info.get('NetworkSettings', {}).get('Ports', {})
        port = '27017'
        for k, v in ports.items():
            if '27017' in k and v:
                port = v[0].get('HostPort', '27017')
        
        print()
        print("=" * 60)
        print("📌 MongoDB连接信息")
        print("=" * 60)
        print(f"主机: {NAS_IP}")
        print(f"端口: {port}")
        print(f"用户: {username}")
        print(f"密码: {password}")
        print()
        print("连接字符串:")
        print(f"  mongodb://{username}:{password}@{NAS_IP}:{port}/")
        print()
