# 自动执行规则

> 需求拆解完成后，自动执行计划直到完成

---

## 核心原则

### 1. 自动开始
- 拆解完计划后，**立即自动开始执行第一个任务**
- 不需要等待用户确认
- 不需要用户说"开始执行"

### 2. 逐步推进
- 完成一个任务后，**自动执行下一个可执行任务**
- 按照依赖关系和优先级顺序执行
- 持续执行直到所有任务完成

### 3. 上下文理解
- 每次执行前，理解对话上下文
- 结合之前的拆解结果
- 考虑前后任务的关联

### 4. 持续完成
- 持续执行直到完成用户描述的目标
- 除非用户明确停止，否则不中断
- 遇到阻塞自动寻找解决方案

---

## 执行流程

### 标准流程

```
1. 用户说："需求拆解"或"执行计划"
   ↓
2. 检查是否有计划文档
   ├─ 有 → 读取计划，找到下一个可执行任务
   └─ 没有 → 先深度思考，拆解需求，生成计划
   ↓
3. 自动开始执行第一个任务
   - 标记状态：⏳ → 🔄
   - 调用相应工具/能力
   ↓
4. 完成任务
   - 标记状态：🔄 → ✅
   - 更新执行日志
   ↓
5. 检查下一个可执行任务
   - 依赖已完成？
   - 优先级是否足够？
   ↓
6. 自动执行下一个任务
   - 重复步骤3-5
   ↓
7. 所有任务完成
   - 检查成功标准
   - 生成完成报告
```

## 任务执行规则

### 可执行任务判断

一个任务可以执行，当且仅当：

1. **状态为"⏳待开始"**
2. **所有依赖任务已完成**（依赖列表为空或所有依赖任务状态为✅）
3. **优先级为P0或P1**（P2任务可以延后）
4. **没有阻塞**（没有被标记为🚫阻塞中）

### 任务执行顺序

1. **优先级优先**：P0 > P1 > P2
2. **依赖优先**：先执行依赖少的任务
3. **Epic顺序**：按Epic顺序执行
4. **任务ID顺序**：同一Epic内按ID顺序

## 上下文理解规则

### 1. 读取对话历史

**必须理解**：
- 用户之前说了什么？
- 当前对话的目标是什么？
- 是否有未完成的任务？
- 是否有需求变更？

### 2. 理解当前任务

**必须理解**：
- 当前要执行的任务是什么？
- 这个任务在整体计划中的位置？
- 需要什么资源/信息？
- 预期输出是什么？

### 3. 结合上下文执行

**执行原则**：
- 不要孤立地执行任务
- 考虑前后任务的关联
- 保持执行的一致性
- 如果上下文变化，重新评估计划

---

## 阻塞处理规则

### 阻塞识别

任务可能阻塞的原因：
1. **技术问题**：工具不可用、API失败等
2. **依赖未完成**：前置任务未完成
3. **需求不明确**：需要用户澄清
4. **资源不足**：文件不存在、权限不足等

### 阻塞处理流程

```
1. 识别阻塞原因
   ↓
2. 尝试自动解决
   ├─ 寻找替代方案
   ├─ 调整依赖关系
   └─ 跳过非关键任务
   ↓
3. 如果无法自动解决
   ├─ 标记为🚫阻塞中
   ├─ 记录阻塞原因
   └─ 继续执行其他可执行任务
   ↓
4. 等待用户输入或资源到位
   ↓
5. 解决阻塞后，继续执行
```

## 执行日志更新规则

### 实时更新

**每完成一个任务，立即更新**：
1. 任务状态：🔄 → ✅
2. 执行日志：记录完成时间、结果
3. 计划文档：更新任务表格状态

**格式**：
```markdown
### YYYY-MM-DD HH:MM

**✅ 完成**:
- [x] T1.1: 分析现有章节质量（耗时30分钟）
  - 结果：发现3个章节需要优化
  - 输出：章节质量分析报告.md

**🔄 进行中**:
- [ ] T1.2: 识别未完成章节（进度50%）

**📋 下一步**:
- 完成T1.2后，自动执行T1.3
```

## 用户交互规则

### 用户可以在任何时候

1. **查看进度**："当前进度如何？"
2. **暂停执行**："暂停一下"
3. **继续执行**："继续执行"
4. **调整计划**："这个任务优先级改为P0"
5. **跳过任务**："跳过这个任务"
6. **停止执行**："停止执行"

### AI响应规则

- **查看进度**：显示当前执行状态和进度
- **暂停执行**：停止执行，等待用户指令
- **继续执行**：从当前任务继续执行
- **调整计划**：更新计划，重新评估执行顺序
- **跳过任务**：标记为❌已取消，执行下一个
- **停止执行**：停止执行，保存当前状态

---

## 完成确认规则

### 所有任务完成后的检查

1. **检查成功标准**
   - 所有成功标准是否达成？
   - 是否有遗漏？

2. **生成完成报告**
   - 总结完成的任务
   - 记录关键成果
   - 列出未完成的任务（如有）

3. **确认目标达成**
   - 用户描述的目标是否达成？
   - 是否需要补充工作？

## 示例场景

### 场景1：新需求拆解

**用户**："帮我拆解这本书的需求"

**AI执行**：
1. 🧠 深度思考（3-5分钟）
2. 📋 需求拆解（生成计划）
3. 📄 生成计划文档
4. ✅ **自动开始执行第一个任务**（无需用户说"开始"）

### 场景2：继续执行

**用户**："继续执行"

**AI执行**：
1. 🧠 理解上下文：已有计划，正在执行
2. ✅ 继续执行下一个可执行任务

### 场景3：调整计划

**用户**："这个任务优先级改为P0"

**AI执行**：
1. 更新计划：调整任务优先级
2. 重新评估执行顺序
3. ✅ 继续执行（按新优先级）

---

## 注意事项

1. **不要等待用户确认**：拆解完成后立即开始执行
2. **不要孤立执行**：理解上下文，考虑关联
3. **不要死守计划**：遇到问题自动调整
4. **不要中断执行**：持续执行直到完成或用户停止
5. **及时更新状态**：每完成一个任务立即更新

---

> **核心原则**：拆解完计划后，自动开始执行，逐步推进，持续完成，直到达成目标。
