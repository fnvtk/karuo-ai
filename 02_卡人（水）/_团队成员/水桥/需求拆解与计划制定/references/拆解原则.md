# 需求拆解原则

> 将复杂需求拆解成可执行的原子任务

---

## 拆解原则

### 1. 原子化原则

**标准**：每个任务30分钟~4小时可完成

| ✅ 好的拆解 | ❌ 差的拆解 |
|:---|:---|
| T1.1: 调用微信API获取code（30m） | T1: 实现微信登录（2天） |
| T1.2: 后端验证code并生成token（1h） | |
| T1.3: 前端存储token并跳转（30m） | |

### 2. 可测试原则

每个任务必须有明确的验收标准：

```
任务：T1.1 调用微信API获取code
验收标准：
- [ ] 能成功调用微信API
- [ ] 正确获取code参数
- [ ] 处理API异常情况
```

### 3. 依赖清晰原则

明确写出前置任务：

```
T1.2 依赖 T1.1
T1.3 依赖 T1.1, T1.2
```

### 4. 优先级明确原则

| 优先级 | 含义 | 示例 |
|:---|:---|:---|
| **P0** | 核心功能，必须完成 | 用户登录、支付 |
| **P1** | 重要功能，应该完成 | 数据统计、导出 |
| **P2** | 锦上添花，可以延后 | 主题切换、动画效果 |

## 拆解步骤

### Step 1: 提取功能点

```
需求文档
  ↓
功能模块（Epic）
  ↓
功能点（Task）
  ↓
子任务（Subtask）
```

**示例**：

```
需求：用户登录系统

Epic 1: 微信登录
  ├─ Task 1.1: 前端获取微信授权
  │   ├─ Subtask 1.1.1: 调用微信JS-SDK
  │   └─ Subtask 1.1.2: 处理授权回调
  ├─ Task 1.2: 后端验证并生成token
  │   ├─ Subtask 1.2.1: 调用微信API验证code
  │   └─ Subtask 1.2.2: 生成JWT token
  └─ Task 1.3: 前端存储token
      └─ Subtask 1.3.1: 存储到localStorage
```

### Step 2: 识别依赖关系

**依赖类型**：

| 类型 | 说明 | 示例 |
|:---|:---|:---|
| **技术依赖** | 需要先完成技术基础 | 数据库设计 → API开发 |
| **功能依赖** | 功能之间的依赖 | 登录 → 个人中心 |
| **数据依赖** | 需要先有数据 | 用户数据 → 数据统计 |

**依赖图示例**：

```
数据库设计
    ↓
API开发
    ↓
前端对接
    ↓
测试
```

### Step 3: 设置优先级

**优先级判断标准**：

1. **P0核心功能**：
   - 没有这个功能，系统无法运行
   - 用户核心路径必须的功能

2. **P1重要功能**：
   - 提升用户体验
   - 提高系统效率

3. **P2可选功能**：
   - 锦上添花
   - 可以延后实现

### Step 4: 估算时间

**时间估算原则**：

| 任务复杂度 | 预估时间 |
|:---|:---|
| 简单（复制粘贴） | 30分钟 |
| 中等（需要思考） | 1-2小时 |
| 复杂（需要设计） | 4小时 |
| 非常复杂（需要研究） | 1天 |

**缓冲时间**：
- 总时间 = 任务时间累加 × 1.2（20%缓冲）

### Step 5: 生成任务清单

**格式**：

```markdown
### Epic 1: [功能模块名]

| ID | 任务 | 依赖 | 优先级 | 状态 | 预估 | 负责人 |
|----|------|------|--------|------|------|--------|
| T1.1 | 任务描述 | - | P0 | ⏳ | 30m | |
| T1.2 | 任务描述 | T1.1 | P0 | ⏳ | 1h | |
```

---

## 任务粒度标准

| 粒度 | 时间范围 | 示例 | 验收标准 |
|:---|:---|:---|:---|
| **Epic** | 1-2周 | "用户登录模块" | 整个模块可用 |
| **Task** | 4-8小时 | "实现微信登录" | 功能可用 |
| **Subtask** | 30分钟-2小时 | "调用微信API获取code" | 代码可运行 |

---

## 拆解检查清单

拆解完成后，检查：

- [ ] 每个任务都有明确的验收标准
- [ ] 任务粒度合适（30分钟~4小时）
- [ ] 依赖关系清晰
- [ ] 优先级设置合理
- [ ] 时间估算合理
- [ ] 没有遗漏关键任务
