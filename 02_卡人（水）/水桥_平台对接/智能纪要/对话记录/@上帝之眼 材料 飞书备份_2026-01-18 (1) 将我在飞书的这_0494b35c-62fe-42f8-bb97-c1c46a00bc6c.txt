## 策略库架构

本策略库整合了卡若多年的实战经验，包含11大核心策略，支持自动交易、情绪监控和AI分析。

## 核心理念

基于视频会议记录的核心思想：
- **7根K线理论**：人类情绪周期是7天，从兴奋到平静需要7次经历
- **强中强选股**：在强势板块中选择强势个股（如同在广东、浙江选首富）
- **轮询纠正**：每天从4000只股票中选择当前最强的5只，持续轮询
- **三模块架构**：采集模块 + 交易模块 + 评估模块
- **高手跟踪**：实时监控前10名高手的买卖点，AI自动跟随

## 使用方法

### 1. 策略调用
每个策略都是独立模块，可通过策略ID调用：
```python
from strategy_manager import StrategyManager

# 初始化策略管理器
sm = StrategyManager()

# 执行7根K线策略
result = sm.execute("7K_LINE")

# 执行节假日策略
result = sm.execute("HOLIDAY")
```

### 2. 自动交易
```python
from auto_trader import AutoTrader

# 初始化自动交易
trader = AutoTrader()

# 设置策略
trader.add_strategy("7K_LINE", weight=0.4)
trader.add_strategy("EXPERT_FOLLOW", weight=0.3)
trader.add_strategy("HOLIDAY", weight=0.3)

# 开始自动交易
trader.start()
```

### 3. 情绪监控
```python
from emotion_monitor import EmotionMonitor

# 监控市场情绪
em = EmotionMonitor()

user_emotion = em.get_user_emotion(user_id)
```

### 4. AI分析
```python
from ai_analyzer import AIAnalyzer

# AI分析当前市场
analyzer = AIAnalyzer()

```

## 关键参数

### 7根K线策略
- 持有周期：7个交易日
- 止损：-5%无条件止损
- 目标收益：3-5%
- 资金管理：10万元，每天5万进5万出（T+1循环）

### 节假日策略
- 买入时机：节前2个交易日
- 卖出时机：节后第7个交易日
- 目标板块：医药生物、非银金融、电子信息、新能源、消费电子

### 高手跟踪
- 监控对象：排名前10的高手
- 跟随方式：实时AI采集 + 自动交易
- 延迟容忍：<1秒（需骨干网）

## 风控规则

1. **坚决止损**：-5%无条件止损
2. **坦然接受损失**：不做情绪化交易
3. **保证交易次数**：小步快跑，每天复利
4. **顺势操作**：跟随市场趋势
5. **把握时机**：根据大盘情况调整买卖点
6. **良好心态**：不贪婪不恐惧
7. **记录日志**：每笔交易必记录

```python
class BaseStrategy:
    """策略基类"""
    
    def __init__(self, strategy_id, config):
        self.strategy_id = strategy_id
        self.config = config
        self.name = config['name']
        self.type = config['type']
        self.cycle = config['cycle']
        self.stop_loss = config['stop_loss']
        self.target_profit = config['target_profit']
        self.positions = []  # 当前持仓
        self.history = []    # 历史记录
    
    def select_stocks(self):
        """选股逻辑（子类实现）"""
        raise NotImplementedError
    
    def entry_signal(self, stock):
        """买入信号（子类实现）"""
        raise NotImplementedError
    
    def exit_signal(self, position):
        """卖出信号（子类实现）"""
        raise NotImplementedError
    
    def risk_control(self, position):
        """风控检查"""
        # 止损检查
        if position['profit_rate'] <= self.stop_loss:
            return {'action': 'SELL', 'reason': '触发止损'}
        
        # 止盈检查
        if position['profit_rate'] >= self.target_profit:
            return {'action': 'SELL', 'reason': '达到目标收益'}
        
        # 时间检查
        if position['hold_days'] >= self.cycle:
            return {'action': 'SELL', 'reason': '达到持有周期'}
        
        return {'action': 'HOLD', 'reason': '继续持有'}
    
    def execute(self, market_data):
        """执行策略"""
        results = {
            'buy_signals': [],
            'sell_signals': [],
            'hold_positions': []
        }
        
        # 检查持仓
        for position in self.positions:
            decision = self.risk_control(position)
            
            if decision['action'] == 'SELL':
                results['sell_signals'].append({
                    'stock': position['stock'],
                    'reason': decision['reason'],
                    'position': position
                })
            else:
                # 检查策略信号
                if self.exit_signal(position):
                    results['sell_signals'].append({
                        'stock': position['stock'],
                        'reason': '策略信号',
                        'position': position
                    })
                else:
                    results['hold_positions'].append(position)
        
        # 选股
        candidates = self.select_stocks()
        
        # 检查买入信号
        for stock in candidates:
            if self.entry_signal(stock):
                results['buy_signals'].append({
                    'stock': stock,
                    'strategy': self.strategy_id,
                    'reason': self.get_entry_reason(stock)
                })
        
        return results
    
    def get_entry_reason(self, stock):
        """获取买入理由（子类可重写）"""
        return f"{self.name}选股"
```

```python
class StrategyManager:
    """策略管理器"""
    
    def __init__(self):
        self.strategies = {}
        self.load_strategies()
    
    def load_strategies(self):
        """加载所有启用的策略"""
        for strategy_id, config in STRATEGY_REGISTRY.items():
            if config['enabled']:
                # 动态导入策略类
                module = __import__(f"strategies.{config['file'][:-3]}", 
                                   fromlist=[config['class']])
                strategy_class = getattr(module, config['class'])
                
                # 实例化策略
                self.strategies[strategy_id] = strategy_class(strategy_id, config)
    
    def execute(self, strategy_id, market_data):
        """执行指定策略"""
        if strategy_id not in self.strategies:
            raise ValueError(f"策略 {strategy_id} 未启用或不存在")
        
        return self.strategies[strategy_id].execute(market_data)
    
    def execute_all(self, market_data):
        """执行所有策略"""
        all_results = {}
        
        for strategy_id, strategy in self.strategies.items():
            try:
                results = strategy.execute(market_data)
                all_results[strategy_id] = results
            except Exception as e:
                print(f"策略 {strategy_id} 执行失败: {e}")
                all_results[strategy_id] = {'error': str(e)}
        
        return all_results
    
    def get_combined_signals(self, market_data):
        """获取综合信号（多策略投票）"""
        all_results = self.execute_all(market_data)
        
        # 统计买入信号
        buy_votes = {}
        for strategy_id, results in all_results.items():
            if 'buy_signals' in results:
                weight = STRATEGY_REGISTRY[strategy_id]['weight']
                
                for signal in results['buy_signals']:
                    stock_code = signal['stock']['code']
                    
                    if stock_code not in buy_votes:
                        buy_votes[stock_code] = {
                            'stock': signal['stock'],
                            'votes': 0,
                            'strategies': []
                        }
                    
                    buy_votes[stock_code]['votes'] += weight
                    buy_votes[stock_code]['strategies'].append({
                        'id': strategy_id,
                        'name': STRATEGY_REGISTRY[strategy_id]['name'],
                        'reason': signal['reason']
                    })
        
        # 排序（按投票权重）
        ranked_signals = sorted(buy_votes.values(), 
                               key=lambda x: x['votes'], 
                               reverse=True)
        
        return ranked_signals
    
    def evaluate_strategy(self, strategy_id, period_days=30):
        """评估策略表现"""
        strategy = self.strategies[strategy_id]
        
        # 统计历史记录
        history = strategy.history[-period_days:]
        
        if not history:
            return None
        
        total_trades = len(history)
        win_trades = sum(1 for t in history if t['profit'] > 0)
        total_profit = sum(t['profit'] for t in history)
        avg_profit = total_profit / total_trades if total_trades > 0 else 0
        win_rate = win_trades / total_trades if total_trades > 0 else 0
        
        return {
            'strategy_id': strategy_id,
            'strategy_name': strategy.name,
            'total_trades': total_trades,
            'win_trades': win_trades,
            'win_rate': win_rate,
            'total_profit': total_profit,
            'avg_profit': avg_profit,
            'max_drawdown': self._calculate_max_drawdown(history)
        }
    
    def _calculate_max_drawdown(self, history):
        """计算最大回撤"""
        if not history:
            return 0
        
        cumulative = 0
        peak = 0
        max_dd = 0
        
        for trade in history:
            cumulative += trade['profit']
            if cumulative > peak:
                peak = cumulative
            dd = (peak - cumulative) / peak if peak > 0 else 0
            if dd > max_dd:
                max_dd = dd
        
        return max_dd
    
    def switch_strategy(self, current_market_state):
        """根据市场状态切换策略"""
        # 市场状态：牛市/熊市/震荡/转折
        recommendations = {
            '牛市': ['7K_LINE', 'WHALE_SWALLOW', 'CELEBRITY_TRAFFIC'],
            '熊市': ['SLEEPING_TURTLE', 'EXPERT_FOLLOW'],
            '震荡': ['TWIN_STOCK', 'MA_SYSTEM'],
            '转折': ['MARKET_ROTATION', 'HOLIDAY']
        }
        
        return recommendations.get(current_market_state, ['7K_LINE'])
```

## 理论基础

### 核心理念
基于人类情绪周期的7天理论：

> "人类从动荡到平稳的时间就是7天，经历过7次后情绪就会平复。突然中了彩票，7天后就没啥感觉了；员工受到鼓励，7天后也没感觉了。这是大部分人的平均值。"  
> —— 卡若，2026年1月18日视频会议

### 策略逻辑

1. **强中强选股**  
   - 从强势板块中选择强势个股
   - 类比：在广东、浙江（最富裕的省）中选首富，明年出首富的概率最高

2. **节假日效应**  
   - 五一、国庆、春节等长假期间更容易采集到强中强股票
   - 有更多时间进行分析和筛选

3. **7天卖出**  
   - 买入后持有7个交易日卖出
   - 因为人类情绪在7天后会平复，股价动能减弱

4. **轮询机制**  
   - 每天从4000只股票中选择当天最强的5只
   - 7天后卖出，继续选择下一批最强的5只
   - 不断轮询和纠正，保持持有的都是当前最强的股票

```python
def select_strong_stocks_in_sector(sector, date):
    """
    在板块内选择强势个股
    
    条件：
    1. 最近3天涨幅>5%
    2. 今日涨幅>2%
    3. 量比>1.5
    4. 流通盘<10亿
    5. 不在涨停（避免买不到）
    6. 不在跌停
    """
    
    stocks = get_stocks_in_sector(sector, date)
    
    candidates = []
    for stock in stocks:
        # 基本条件
        if (stock['capital'] > 10  # 流通盘>10亿，跳过
            or stock['is_st']  # ST股，跳过
            or stock['is_limit_up']  # 涨停，跳过
            or stock['is_limit_down']):  # 跌停，跳过
            continue
        
        # 最近3天涨幅
        rise_3d = calculate_rise_rate(stock, days=3)
        if rise_3d < 0.05:  # 3天涨幅<5%，跳过
            continue
        
        # 今日涨幅和量比
        if (stock['change_rate'] > 0.02  # 今日涨幅>2%
            and stock['volume_ratio'] > 1.5):  # 量比>1.5
            
            # 计算综合评分
            score = calculate_stock_score(stock)
            candidates.append({
                'stock': stock,
                'score': score
            })
    
    # 按评分排序
    candidates.sort(key=lambda x: x['score'], reverse=True)
    
    # 返回前3名
    return [c['stock'] for c in candidates[:3]]
```

## 评分系统

```python
def calculate_comprehensive_score(stock, date):
    """
    综合评分系统
    
    权重分配：
    - 涨幅趋势：30%
    - 成交量：25%
    - 板块效应：20%
    - 资金流向：15%
    - 技术形态：10%
    """
    
    score = 0
    
    # 1. 涨幅趋势（30%）
    rise_3d = calculate_rise_rate(stock, days=3)
    rise_5d = calculate_rise_rate(stock, days=5)
    trend_score = (rise_3d * 0.6 + rise_5d * 0.4) * 100
    score += trend_score * 0.3
    
    # 2. 成交量（25%）
    volume_ratio = stock['volume_ratio']
    volume_score = min(volume_ratio * 20, 100)  # 量比越大越好，最高100分
    score += volume_score * 0.25
    
    # 3. 板块效应（20%）
    sector_score = get_sector_strength(stock['sector'], date)
    score += sector_score * 0.2
    
    # 4. 资金流向（15%）
    money_flow = get_money_flow(stock, date)
    money_score = (money_flow + 50) * 2  # 转换为0-100分
    score += money_score * 0.15
    
    # 5. 技术形态（10%）
    tech_score = analyze_technical_pattern(stock)
    score += tech_score * 0.1
    
    return score
```

### 卖出逻辑

```python
def generate_sell_signals(date, current_positions):
    """
    生成卖出信号
    """
    signals = []
    
    # 获取大盘状态
    market_state = get_market_state(date)
    
    for position in current_positions:
        # 只处理7根K线策略的持仓
        if position['strategy'] != '7K_LINE':
            continue
        
        # 计算持有天数
        hold_days = calculate_hold_days(position['buy_date'], date)
        
        # 计算当前盈亏
        current_price = get_current_price(position['stock_code'])
        profit_rate = (current_price - position['buy_price']) / position['buy_price']
        
        # 判断卖出条件
        sell_reason = None
        
        # 1. 持有满7天
        if hold_days >= 7:
            sell_reason = "持有满7天"
        
        # 2. 触发止损
        elif profit_rate <= -0.05:
            sell_reason = "触发止损-5%"
        
        # 3. 达到目标收益
        elif profit_rate >= 0.05:
            sell_reason = "达到目标收益5%"
        elif profit_rate >= 0.03 and hold_days >= 5:
            sell_reason = "达到目标收益3%且持有5天"
        
        # 4. 大盘暴跌
        elif market_state['change_rate'] < -0.03:
            sell_reason = "大盘暴跌>3%"
        
        # 5. 技术破位
        elif check_technical_breakdown(position['stock_code']):
            sell_reason = "技术破位"
        
        # 生成卖出信号
        if sell_reason:
            sell_time = get_sell_timing(market_state)
            
            signals.append({
                'action': 'SELL',
                'stock_code': position['stock_code'],
                'stock_name': position['stock_name'],
                'price': current_price,
                'amount': position['amount'],
                'reason': sell_reason,
                'timing': sell_time,
                'profit_rate': profit_rate,
                'hold_days': hold_days,
                'strategy': '7K_LINE'
            })
    
    return signals
```

## 理论基础

### 核心理念

> "历年来，每逢春节、五一、国庆等节假日，大盘往往会迎来小幅度的上涨。这种上涨的规律性为投资者提供了短线交易的机会。"

### 策略逻辑

1. **节假日效应**  
   - 节假日前市场情绪乐观
   - 资金回流推动股价上涨
   - 历史规律性强

2. **冠军逻辑**  
   - 选出每年节假日期间表现最强的板块
   - 从这些板块中挑选优质个股
   - 优先选择连续几年表现出色的板块和个股

3. **稳定性优先**  
   - 选择连续几年都在涨幅榜单上的股票
   - 这些股票的稳定性和市场认可度更高

```python
# 大智慧公式
SECTOR_FORMULA = """
input:DateTime1(开始日期),DateTime2(结束日期);
startDay:=barslast(date=DateTime1);{开始时间}
stopDay:=barslast(date=DateTime2);{结束时间}
startC:=ref(c,startDay);{开始价格}
stopC:=ref(c,stopDay);{结束价格}
区间涨幅:(StopC-startC)/startC*100,precis2;{区间涨幅}
"""

def select_strong_sectors(year, holiday_name, history_years=5):
    """
    筛选节假日期间表现最强的板块
    
    步骤：
    1. 统计过去N年该节假日期间各板块涨幅
    2. 选出涨幅前5的板块
    3. 按出现次数排序（连续几年都强的板块）
    """
    
    sector_stats = {}
    
    # 统计历史数据
    for y in range(year - history_years, year):
        dates = get_holiday_dates(y, holiday_name)
        
        # 获取各板块涨幅
        sectors = get_sector_rise_rate(
            dates['holiday_start'], 
            dates['sell_date']
        )
        
        # 记录前5名
        top5 = sorted(sectors, key=lambda x: x['rise_rate'], reverse=True)[:5]
        
        for sector in top5:
            name = sector['name']
            if name not in sector_stats:
                sector_stats[name] = {
                    'appear_times': 0,
                    'total_rise': 0,
                    'years': []
                }
            
            sector_stats[name]['appear_times'] += 1
            sector_stats[name]['total_rise'] += sector['rise_rate']
            sector_stats[name]['years'].append(y)
    
    # 按出现次数和平均涨幅排序
    ranked_sectors = []
    for name, stats in sector_stats.items():
        if stats['appear_times'] >= 3:  # 至少出现3次
            ranked_sectors.append({
                'name': name,
                'appear_times': stats['appear_times'],
                'avg_rise': stats['total_rise'] / stats['appear_times'],
                'years': stats['years']
            })
    
    # 排序：优先看出现次数，其次看平均涨幅
    ranked_sectors.sort(
        key=lambda x: (x['appear_times'], x['avg_rise']),
        reverse=True
    )
    
    return ranked_sectors[:5]
```

```python
def filter_experts(candidates):
    """
    筛选合格的高手
    
    条件：
    1. 收益率稳定且较高
    2. 操作频繁（超短线特征）
    3. 一致性强（不是靠运气）
    4. 选股逻辑清晰可理解
    """
    
    qualified_experts = []
    
    for expert in candidates:
        # 收益率要求
        if expert['total_return'] < 0.5:  # 总收益<50%，排除
            continue
        
        # 操作频率要求
        if expert['total_trades'] < 50:  # 交易次数<50，排除
            continue
        
        # 胜率要求
        if expert['win_rate'] < 0.6:  # 胜率<60%，排除
            continue
        
        # 一致性检查（波动性）
        if expert['return_std'] > 0.3:  # 收益波动太大，排除
            continue
        
        # 选股逻辑可理解性
        logic_score = analyze_stock_selection_logic(expert)
        if logic_score < 60:  # 逻辑不清晰，排除
            continue
        
        qualified_experts.append(expert)
    
    return qualified_experts
```

### 2. 评分系统

```python
def score_expert(expert, history_days=30):
    """
    给高手评分
    
    权重分配：
    - 收益率：30%
    - 胜率：25%
    - 稳定性：20%
    - 交易频率：15%
    - 逻辑清晰度：10%
    """
    
    score = 0
    
    # 1. 收益率（30%）
    return_score = min(expert['return_rate'] * 50, 100)
    score += return_score * 0.3
    
    # 2. 胜率（25%）
    win_rate_score = expert['win_rate'] * 100
    score += win_rate_score * 0.25
    
    # 3. 稳定性（20%）
    # 收益标准差越小越好
    stability_score = max(100 - expert['return_std'] * 200, 0)
    score += stability_score * 0.2
    
    # 4. 交易频率（15%）
    # 超短线高手应该交易频繁
    trade_freq = expert['total_trades'] / history_days
    freq_score = min(trade_freq * 20, 100)
    score += freq_score * 0.15
    
    # 5. 逻辑清晰度（10%）
    logic_score = analyze_stock_selection_logic(expert)
    score += logic_score * 0.1
    
    return score
```

```python
class ExpertPositionMonitor:
    """高手持仓实时监控"""
    
    def __init__(self, experts, frequency=30):
        self.experts = experts
        self.frequency = frequency  # 监控频率（秒）
        self.last_positions = {}
    
    def start_monitor(self):
        """开始监控"""
        while True:
            for expert in self.experts:
                # 获取最新持仓
                current_positions = get_expert_positions(expert['id'])
                
                # 与上次持仓对比
                changes = self.detect_changes(
                    expert['id'],
                    current_positions
                )
                
                # 发现变化，生成信号
                if changes:
                    self.handle_position_changes(expert, changes)
                
                # 更新缓存
                self.last_positions[expert['id']] = current_positions
            
            # 休眠
            time.sleep(self.frequency)
    
    def detect_changes(self, expert_id, current_positions):
        """检测持仓变化"""
        last_pos = self.last_positions.get(expert_id, {})
        
        changes = {
            'new_buy': [],   # 新买入
            'sold': [],      # 卖出
            'increased': [], # 加仓
            'decreased': []  # 减仓
        }
        
        # 检测新买入
        for code, amount in current_positions.items():
            if code not in last_pos:
                changes['new_buy'].append({
                    'code': code,
                    'amount': amount
                })
            elif amount > last_pos[code]:
                changes['increased'].append({
                    'code': code,
                    'amount': amount - last_pos[code]
                })
            elif amount < last_pos[code]:
                changes['decreased'].append({
                    'code': code,
                    'amount': last_pos[code] - amount
                })
        
        # 检测卖出
        for code, amount in last_pos.items():
            if code not in current_positions:
                changes['sold'].append({
                    'code': code,
                    'amount': amount
                })
        
        return changes if any(changes.values()) else None
```

```python
def generate_follow_buy_signals(expert_changes):
    """
    根据高手买入生成跟随信号
    
    规则：
    1. 至少2个高手同时买入
    2. 高手的业绩权重投票
    3. 延迟<1秒跟随
    """
    
    # 统计买入票数
    buy_votes = {}
    
    for expert_id, changes in expert_changes.items():
        expert = get_expert_info(expert_id)
        weight = expert['score'] / 100  # 根据评分计算权重
        
        for stock in changes.get('new_buy', []):
            code = stock['code']
            
            if code not in buy_votes:
                buy_votes[code] = {
                    'votes': 0,
                    'experts': [],
                    'stock_info': get_stock_info(code)
                }
            
            buy_votes[code]['votes'] += weight
            buy_votes[code]['experts'].append({
                'id': expert_id,
                'name': expert['name'],
                'score': expert['score']
            })
    
    # 筛选票数>=2的股票
    signals = []
    for code, data in buy_votes.items():
        if data['votes'] >= 2 and len(data['experts']) >= 2:
            signals.append({
                'action': 'BUY',
                'stock_code': code,
                'stock_name': data['stock_info']['name'],
                'reason': f"{len(data['experts'])}位高手买入（投票权重{data['votes']:.2f}）",
                'experts': data['experts'],
                'confidence': min(data['votes'] / 5, 1.0),  # 置信度
                'strategy': 'EXPERT_FOLLOW'
            })
    
    # 按置信度排序
    signals.sort(key=lambda x: x['confidence'], reverse=True)
    
    return signals
```

```python
def find_common_patterns(experts):
    """
    寻找多个高手的共同点
    
    共同点：
    1. 是否集中在某些板块或题材
    2. 是否有一致的买卖节奏
    3. 是否遵循相似的资金管理原则
    """
    
    patterns = {
        'common_sectors': [],     # 共同关注的板块
        'common_rhythms': [],     # 共同的交易节奏
        'common_indicators': [],  # 共同使用的指标
        'common_stocks': []       # 共同持有的股票
    }
    
    # 1. 分析共同板块
    sector_votes = {}
    for expert in experts:
        for sector in expert['favorite_sectors']:
            sector_votes[sector] = sector_votes.get(sector, 0) + 1
    
    # 出现次数>=5的板块
    patterns['common_sectors'] = [
        s for s, count in sector_votes.items() if count >= 5
    ]
    
    # 2. 分析交易节奏
    # 是否都偏好低吸、高抛还是追涨打板
    rhythm_stats = analyze_trade_rhythms(experts)
    patterns['common_rhythms'] = rhythm_stats
    
    # 3. 分析技术指标
    indicator_stats = analyze_indicators(experts)
    patterns['common_indicators'] = indicator_stats
    
    # 4. 分析共同持股
    stock_votes = {}
    for expert in experts:
        for stock in expert['holdings']:
            code = stock['code']
            stock_votes[code] = stock_votes.get(code, 0) + 1
    
    # 至少3个高手都持有
    patterns['common_stocks'] = [
        {'code': code, 'count': count}
        for code, count in stock_votes.items() if count >= 3
    ]
    
    return patterns
```

## 买点与卖点优化

### 一、买点设计（分档 + 盘中时段）

#### 1. 推荐买入日（按优先级）

| 档位 | 含义 | 建议仓位 | 说明 |
|-----|------|----------|------|
| **T-2** | 休市前第 2 个交易日 | 50%～60% | 首选买点，避免 T-1 情绪过热、价格冲高 |
| **T-1** | 休市前最后 1 个交易日 | 40%～50% | 补仓或观望后确认，若 T-2 已满仓可不再加 |
| **不推荐** | 节前 T-3 及更早 | — | 持有时间过长，易受节前波动影响，仅作观察 |

**原则**：以 T-2 为主、T-1 为辅；单日不追涨，宁可少买不买错。

#### 2. 盘中买点时段（按大盘状态）

| 大盘状态 | 早盘 | 午盘 | 说明 |
|----------|------|------|------|
| **大涨(>0.5%)** | 9:35～9:50 观察，10:00 后分批 | 13:25～13:40 | 不追高开，等第一次回调或横盘再买 |
| **平衡(-0.5%～0.5%)** | 9:45～10:15 分批 | 13:00～13:30 | 主买时段，可分批挂单 |
| **大跌(<-0.5%)** | 9:50～10:30 或 14:00～14:45 | 14:30～14:50 | 恐慌盘后分批抄底，尾盘控制仓位 |

**单笔建议**：单只个股不超过总仓位的 20%；T-2 与 T-1 合计不超过计划总仓位的 100%。

#### 3. 买点弹性规则

- **T-2 已大涨(>2%)**：当日可只建 30%～40% 仓，留一部分到 T-1 看情绪。
- **T-1 大幅低开(>-1.5%)**：可把预留资金在 9:35～10:00 或 14:00 后分批买入。
- **节前两日连续大跌**：可维持原计划仓位，不因恐慌加仓；若大盘破位则降低仓位或观望。

### 二、卖点设计（分档 + 盘中时段）

#### 1. 推荐卖出日（按优先级）

| 档位 | 含义 | 建议动作 | 说明 |
|-----|------|----------|------|
| **T+7** | 开市后第 7 个交易日 | **默认卖出日**，可一次性或分两日卖 | 历史统计上情绪与涨幅多在此附近收敛 |
| **T+5** | 开市后第 5 个交易日 | 若已达标可提前止盈 | 收益已达 5%～8% 时可先卖一半 |
| **T+10** | 开市后第 10 个交易日 | 最晚清仓线 | 未在 T+7 卖完的，原则上 T+10 前清仓 |

**原则**：以 T+7 为核心卖出日；提前达标可分批止盈，不贪最后一段。

#### 2. 盘中卖点时段（按大盘状态）

| 大盘状态 | 早盘 | 午盘 | 说明 |
|----------|------|------|------|
| **大涨(>0.5%)** | 9:30～9:45 高开冲高先卖一部分 | 13:30～14:00 | 优先兑现，避免午后回落 |
| **平衡(-0.5%～0.5%)** | 9:35～10:00 或 10:30 后 | 13:05～13:35 | 分批挂单，不挤在开盘一刻 |
| **大跌(<-0.5%)** | 9:45～10:15 等反弹 | 14:45～14:55 | 反弹无力则尾盘止损/止盈出清 |

**单日操作**：若计划在 T+7 全部卖出，可 50% 在早盘、50% 在午盘，避免单一时点冲击。

#### 3. 卖点弹性规则（优先于固定日期）

| 条件 | 动作 |
|------|------|
| **浮盈 ≥ 8%** | 可在 T+5～T+7 任意一天分批止盈，不必死等 T+7 |
| **浮盈 3%～8%** | 以 T+7 为主，T+7 早盘或午盘分批卖 |
| **浮亏 ≥ -3%（止损）** | 当日或次日择机止损，不拖到 T+7 |
| **T+7 当日大跌(>-2%)** | 可延后 1～2 个交易日，在反弹时减仓，但不超过 T+10 |
| **T+3 前已涨超 10%** | 可先卖 50% 锁定利润，余下按 T+7 或 T+10 处理 |

### 三、买点与卖点汇总（执行清单）

**买入阶段**

1. 对照当年交易所公告，确认 **T-2、T-1** 的准确日期。
2. T-2：计划 50%～60% 仓位，按大盘状态选择 9:45～10:15 或 13:00～13:30 分批。
3. T-1：补足至 100% 计划仓位或观望；大跌日可 9:35～10:00 或 14:00 后分批。
4. 单只个股 ≤ 20% 仓位，总仓位不超过计划。

**卖出阶段**

1. 确认 **T+7** 的准确日期（节后第 7 个交易日）。
2. T+7：早盘卖一部分、午盘卖一部分，或按 3%～8% 达标情况分批。
3. 浮盈 ≥ 8% 可提前在 T+5～T+7 止盈；浮亏 ≥ -3% 按止损规则执行。
4. 未在 T+7 卖完的，原则上在 **T+10** 前清仓。
