# Pipeline 执行清单

> 可标准化的长流程，按步骤读对应 SKILL 执行。用户说「跑新项目上线」「跑内容发布」「跑日常运维」时用本清单。
> 版本：1.0 | 更新：2026-02-16

---

## 使用方式

1. 用户触发（如「跑新项目上线」）→ 确定用哪条 Pipeline
2. 按步骤顺序执行：每步读该步的 SKILL.md，执行后交付输出，再进入下一步
3. 任一步失败 → 回退上一步，最多重试 2 次 → 仍失败则通知用户并停止

## 失败与重试

- 任一步报错或超时 → 本步重试 1 次
- 仍失败 → 回退到上一步输出，检查输入是否正确，再重跑当前步 1 次
- 再次失败 → 停止 Pipeline，向用户汇报：卡在第几步、错误信息、建议（如手动处理或拆单）

## 扩展

新增 Pipeline 时：在本文件按相同格式追加「触发词 + 步骤表（执行者、动作、读的 SKILL、输入、输出）」；并在 `SKILL_REGISTRY.md` 或 `BOOTSTRAP.md` 的「标准命令」中增加触发词说明（可选）。
