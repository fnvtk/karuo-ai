# 卡若读书笔记格式（固定规则）

> 卡若AI 内所有「读书笔记」类产出，均按本格式书写；与复盘格式并列，为统一输出规范。  
> 版本：1.0 | 更新：2026-03-02

---

## 一、适用场景与两种形态

| 形态 | 适用 | 核心结构 |
|:---|:---|:---|
| **完整版（五行拆书）** | 书/课/长内容、要落地行动、可复用方法 | 一句话总结 + 金水木火土五块 + 问答 + 人物 + 金句 + 流程图 + 使用规则 |
| **精简版（按书本章节）** | 单本书按章记、偏知识梳理、不需五行 | 来源说明 + 按章节/主题分块 + 要点小结 + 整理时间 |

同一本书可先写精简版，后续再补完整版五行拆书；也可只写其中一种。

---

## 二、文件与存放

- **文件名**：`作者或主题_书名或简称_读书笔记.md`（例：`曾仕强_易经_读书笔记.md`、`纳瓦尔访谈_读书笔记.md`）。
- **存放路径（默认）**：`/Users/karuo/Documents/个人/2、我写的日记/读书笔记/`。**以后读书笔记一律放此目录，固定不发他处。**
- **图片与流程图**：笔记内引用的图片、流程图建议与笔记**同目录**存放，用 `文件名.png` 或 `./文件名.png` 引用，避免用 `../../图片/` 等跨目录路径导致在不同预览/工作区下**不显示**。若用 Mermaid，可在同节内再插入一张导出好的流程图 PNG，保证不依赖渲染也能看到图。
- **图裂了怎么办**：若在 Cursor/VS Code 里图片不显示，可（1）用 Typora 打开该 md（以文件所在目录为基准）；（2）在 Finder 中打开笔记所在文件夹，双击 md 用系统默认预览打开；（3）直接打开同目录下对应的 PNG 文件。建议在每张图下方加一句备用说明：「若图不显示，请打开本笔记所在文件夹中的 `xxx.png` 查看。」
- **记忆写入**：在 `个人/1、卡若：本人/记忆.md` 当日日期下追加 1～2 行摘要 + 本笔记文件路径；便于后续从记忆调出。

## 三、完整版格式（五行拆书）

### 3.1 文档头部（必填）

```markdown
# 卡若的读书笔记 · 《书名》/ 主题

> 来源：（阅读场景、时间、媒介，如「2026-03-02 图书馆阅读」「视频/访谈」）。
> 关键词：（3～5 个，便于检索）。
```

### 3.2 一句话总结（必填）

用一两句话概括：这本书/这场内容在解决什么问题、达成什么结果。  
示例：*这是一场把「幸福」当作首要目标、把「财富」当作可训练技能、并用长期系统消化焦虑与欲望的访谈。*

### 3.3 五行五块（必填，顺序固定）

每块内用 **1、2、3、4…** 编号，语言简洁、大白话；可加小标题（如「A. 选择与决策」）。

| 块 | 标题 | 关注点 |
|:---|:---|:---|
| **金** | 金（定位与角色） | 是谁、给谁、站在什么位置；目标人群与对立面；核心价值主张；时代/环境位置。 |
| **水** | 水（经历与路径） | 关键经历时间线；重大转折；失败/试错/代价；路径选择逻辑。 |
| **木** | 木（方法与产出） | 核心方法论；工具/模型/技巧；可复用动作清单；实际产出形态。（干货最密） |
| **火** | 火（认知与判断） | 核心认知差异；关键判断标准；反共识观点；高阶洞察。（拉开层级） |
| **土** | 土（系统与沉淀） | 系统结构；风险与边界；长期复利机制；可持续性。（老板级视角） |

块标题可带五行色标与副标题，例如：  
`## 金（定位与角色）｜这场内容站在哪个问题上`

### 3.4 问题与解答（建议有）

以「问 + 答」形式，每题给**可照抄的参考答案**，便于日后直接使用。  
格式：`问：……` / `答：……`

### 3.5 人物分析（可选，适合有明确作者/主讲人）

- MBTI、DISC、PDP 推断 + 简短理由  
- 优势、盲区  
- 合作/学习建议  

### 3.6 金句与关键词（建议有）

- **5 个金句**：可改写，不必逐字引用。  
- **5 个关键词**：用「｜」分隔，便于标签与检索。

### 3.7 流程图示（可选）

- 文字版：Step 1 → Step 2 → Step 3。  
- 或 Mermaid 的 flowchart，概括从「起点」到「结果」的路径。

### 3.8 使用规则（建议有）

- 适合谁用、不适合谁用。  
- 使用顺序建议（如：先选一个最痛的欲望合约 → 设 7 天游标 → 每周复利盘点）。  
- 搭配场景（如：和卡若AI 五行框架、私域落地一起用）。

### 3.9 文末（必填）

```markdown
---
*笔记整理时间：YYYY-MM-DD。*
```

## 四、精简版格式（按书本章节）

### 4.1 文档头部（必填）

```markdown
# 卡若的读书笔记 · 作者《书名》

> 来源：（阅读场景、时间；若有版权/正版说明可在此写）。
> 说明：（可选）仅个人学习整理、不提供侵权下载等。
```

### 4.2 按章节或主题分块

- 用 `## 一、……`、`## 二、……` 分大块（可对应书的篇章或自拟主题）。  
- 块内用 **短条** 或 **编号列表**，每条 1～2 行，突出**概念、结构、结论**。  
- 可少量用加粗、箭头（→）、符号（·）增强可读性。

### 4.3 要点小结（建议有）

单独一节「要点小结」或「卡若自用」：  
用 3～5 条概括全书/全内容的**可记忆、可迁移**要点，便于和现有框架（如五行、私域）打通。

### 4.4 文末（必填）

```markdown
---
*笔记整理时间：YYYY-MM-DD。*
```

## 五、与记忆、Skill 的衔接

- **记忆**：读书笔记写完后，在 `个人/1、卡若：本人/记忆.md` 当日下追加一条，格式：  
  `[HH:MM] 读书笔记·《书名》/主题：一句话摘要；全文见 个人/2、我写的日记/读书笔记/xxx.md。`
- **读书笔记 Skill**：`04_卡火（火）/火炬_全栈消息/读书笔记/SKILL.md` 定义五行拆书流程与 XMind 写入；本格式与之对齐，完整版即该 Skill 的 Markdown 版输出。
- **分类**：个人提升 / 人际关系 / 创业 / 商业思维 / 投资；若写 XMind 需选其一。

---

## 六、书写原则（通用）

1. **结论先行**：每块先给结论或框架，再展开。  
2. **可执行**：方法、动作、问答都要能「照着做」。  
3. **可迁移**：尽量和卡若的五行、私域、项目框架挂钩。  
4. **不遗漏模块**：完整版中金水木火土 + 一句话总结必填；顺序不打乱。  
5. **精简版**：不强求五行，但必须有「来源 + 分块 + 小结 + 整理时间」。
