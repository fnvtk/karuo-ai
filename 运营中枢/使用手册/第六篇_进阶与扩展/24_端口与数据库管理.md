## 24.1 唯一 MongoDB 约定

所有新生成的数据一律放在**唯一 MongoDB 实例**：

| 项目 | 值 |
|:---|:---|
| 容器名 | `datacenter_mongodb` |
| 宿主机端口 | **27017** |
| 分组 | datacenter |
| 数据目录 | `/Users/karuo/数据库/mongodb/data` |

### 库用途

| 库名 | 用途 |
|:---|:---|
| `karuo_site` | 卡若AI网站（官网 + 控制台） |
| `wanzhi_esports` | 玩值电竞 |
| `KR` / `KR_*` | 神射手 |

**禁止**新建 MongoDB 实例或使用非 27017 端口。

## 24.2 端口注册表

所有项目端口统一登记在 `运营中枢/工作台/项目与端口注册表.md`，部署前必查防重复。

### 主要端口

| 项目 | 部署端口 | 开发端口 |
|:---|:--:|:--:|
| 卡若AI网站 | 3102 | 3102 |
| 玩值电竞App | 3001 | 3001 |
| 神射手 | 3117 | 3117 |
| 存客宝 Web | 3100 | 3100 |
| 触客宝 Web | 3101 | 3101 |
| Soul 创业实验 | 3000 | 3002 |
| OpenClaw 网关 | 18789/18790 | — |
| n8n | 5678 | — |
| 万推 | 8000 | 8001 |

### Docker 分组

| 分组 | 包含 |
|:---|:---|
| datacenter | MongoDB、Qdrant 等数据库 |
| website | 卡若AI网站、玩值电竞等应用 |

website 组通过 `datacenter_network` 连接数据库。

## 24.3 端口冲突处理

端口冲突时：先结束占用端口的进程，再部署，不改用其他端口。

## 24.4 更新规则

项目、端口、启动命令或部署流程有变更时，必须同步更新端口注册表。
