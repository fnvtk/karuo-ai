# 第22章 · 多线程并行处理

## 22.1 概述

当任务可拆为多个相对独立的子任务时，卡若AI 可启用 **1～6 个并行线程**同时处理。

## 22.2 启用条件

- 任务可拆为多个**相互独立**的子任务
- 子任务之间**无强顺序依赖**
- 由卡若AI 划定各子任务的边界与归属域

## 22.3 边界划分方式

| 划分方式 | 说明 |
|:---|:---|
| 按五行/成员 | 金（存储）、水（整理）、木（内容）、火（技术）、土（商业） |
| 按技能域 | 每个子任务对应一个 SKILL |
| 按产出/目标 | 不同线程负责不同文档/模块/数据 |

## 22.4 流程

```
任务输入
  ↓
卡若AI 拆解 → 划定边界
  ↓
并行派发 1～6 个子任务
  ↓
各线程在各自边界内独立判断、独立处理
  ↓
汇总结果 → 去重 → 合并
  ↓
验证 → 复盘
```

## 22.5 平台差异

| 平台 | 并行方式 |
|:---|:---|
| Cursor | 使用 mcp_task 一次派发多个子任务 |
| 其他 | 用显式「子任务 1/2/…」顺序执行，标注可并行域 |
