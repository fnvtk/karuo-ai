## 4.1 启动顺序（Boot Sequence）

每次新对话开始，卡若AI 按以下顺序加载：

| 步骤 | 读什么 | 为什么 |
|:--:|:---|:---|
| 1 | `BOOTSTRAP.md` | 身份、团队、流程、规则 |
| 2 | `SKILL_REGISTRY.md`（热技能优先） | 查找用户需求对应的技能 |
| 3 | 对应 SKILL.md | 拿到具体执行指令 |
| 4 | 在对话中展示思考与拆解 | 让用户看到完整思考过程 |

**启动瘦身**：步骤 2 优先只加载 🔴 热技能（≤8 个）的触发词+路径，未命中再懒加载其余。

## 4.2 各平台接入方式

### Cursor（自动）

`.cursor/rules/karuo-ai.mdc` 自动加载，指向 BOOTSTRAP.md。正常对话即可，无需额外配置。

### 其他 AI（粘贴）

对话开头粘贴 BOOTSTRAP.md 全文，再附上 SKILL_REGISTRY.md 作为技能库。AI 即可按卡若团队分工和流程执行。

### API 调用（HTTP）

```bash
curl -s -X POST "https://kr-ai.quwanzhi.com/v1/chat" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"你的问题"}' | jq -r '.reply'
```
