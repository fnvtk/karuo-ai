## 17.1 协同规范

### 架构层级

大总管（卡若AI）→ 组长（五行负责人）→ 成员（14 人）

### 四种协作模式

| 模式 | 说明 | 适用场景 |
|:---|:---|:---|
| 串行流水线 | A → B → C 逐步传递 | 有强依赖的任务 |
| 并行委派 | 同时分发多个子任务 | 相互独立的子任务 |
| 计划者-执行者 | 水泉拆解，成员执行 | 需求拆解后分工 |
| Pipeline | 长流程自动化 | 标准化流程 |

### 文件写入权

各技能目录仅归属成员可写。跨组通过任务交接单协作，不直接改对方文件。

### 跨组收敛

2 组以上参与时：各组出「观点 + 依据 + 置信度」，轮次讨论，大总管裁决。

## 17.2 三大 Pipeline

### Pipeline A · 新项目上线

```
土基（商业分析）→ 水泉（需求拆解）→ 木果（项目生成）
→ 火炬（全栈开发）→ 火锤（代码修复）→ 金仓（部署）
→ 水溪（归档）→ 土砖（基因胶囊，可选）
```

### Pipeline B · 内容发布

```
木叶（视频切片）→ 火炬/水溪（文档清洗）
→ 土渠（流量投放）→ 水溪（归档）
```

### Pipeline C · 日常运维

```
金仓（系统监控）→ 金仓（磁盘清理）
→ 金仓（容灾备份）→ 水溪（归档日志）
```

### 失败处理

1. 某步失败 → 本步重试 1 次
2. 仍失败 → 回退上一步，检查输入后再试 1 次
3. 再失败 → 停止 Pipeline，向用户汇报
