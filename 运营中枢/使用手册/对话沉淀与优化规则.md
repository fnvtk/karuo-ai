# 卡若AI 使用手册 · 对话沉淀与优化规则

> **强制**：每次对话结束后，除完成「复盘」外，须按本规则做**沉淀**与**使用手册优化**（若有适用项）。
> **版本**：2.0 | **更新**：2026-03-12

---

## 一、适用范围

- **所有**在卡若AI 工作台内完成的对话（Cursor、API 等）。
- **每次对话收尾**：先完成「强制复盘」（见 `运营中枢/参考资料/卡若复盘格式_固定规则.md`），再执行本规则中的沉淀与手册优化。

---

## 二、对话后沉淀（必做项）

### 2.1 飞书复盘总结发群（可选 · 默认零推送）

**默认不发群。** 写完卡若复盘后**不**自动发飞书（飞书群内容可能出现在关联微信侧）。**仅当**用户**当轮明确口述**要发时才发（建议 ≤500 字）。与 Cursor 规则 `.cursor/rules/karuo-ai.mdc`「飞书复盘发群（强制关闭·默认零推送）」一致；**不因**某 Skill 文档里写过发群步骤而自动代发。

- **脚本**：`02_卡人（水）/水桥_平台对接/飞书管理/脚本/send_review_to_feishu_webhook.py`
- **用法**：`python3 send_review_to_feishu_webhook.py "【卡若AI复盘】YYYY-MM-DD HH:mm\n🎯 目标·结果·达成率\n📌 完成要点\n▶ 下一步"`
- **说明**：见 `运营中枢/工作台/飞书复盘总结发群说明.md`。

### 2.2 经验沉淀

若本次对话中有**可复用经验**（新踩坑、新解法、配置要点、与某 Skill 相关的实操结论等）：

1. **写入**：`02_卡人（水）/水溪_整理归档/经验库/待沉淀/`
2. **命名**：`YYYY-MM-DD_主题或技能名_简短描述.md`
3. **内容建议**：问题背景、做法、结果、适用场景、相关 Skill 或文档路径。

### 2.3 技能与注册表更新

若本次对话中**新增了技能**或**变更了某技能**（触发词、路径、能力描述）：

1. **SKILL_REGISTRY.md**：新增或修改对应技能行（触发词、路径、一句话）。
2. **对应 SKILL.md**：新增则新建并写全；变更则修改步骤、用法、注意点。
3. **技能控制台**（若有）：按 `运营中枢/工作台/01_技能控制台.md` 做配置/登记。

## 三、使用手册优化（有条件执行）

在**同一轮对话收尾**或**后续一次专门维护**中，按下列情况更新使用手册：

> **手册 v2.0 为多文件书籍结构**，总目录在 `README.md`，每篇一个子文件夹，每章一个 .md。

| 情况 | 动作 |
|:---|:---|
| 新增了技能 | 在**第三篇对应组章节**（第7～11章）的技能表中新增一行；在**附录 A 速查表**中新增；**附录 D 更新记录**追加一行 |
| 某技能触发词/职责变更 | 修改**第三篇对应章节**的表格行；**附录 A** 同步修改；**附录 D** 追加更新记录 |
| 新增了板块或负责人 | 在**第3章（团队与角色）**和**对应新章**中补充；**附录 D** 追加 |
| 运营中枢新增/变更规范 | 在**第四篇对应章节**（第12～17章）中补充或修改；**附录 D** 追加 |
| 网站功能新增/变更 | 在**第五篇对应章节**（第18～20章）中补充或修改；**附录 D** 追加 |
| 本次对话产出了更准确描述 | 直接替换对应章节段落；**附录 D** 追加 |

### 3.1 手册文件位置

- **总目录**：`运营中枢/使用手册/README.md`
- **导航页**：`运营中枢/使用手册/卡若AI使用手册.md`
- **本规则**：`运营中枢/使用手册/对话沉淀与优化规则.md`
- **各章节**：`运营中枢/使用手册/第X篇_XXX/XX_章节名.md`
- **附录**：`运营中枢/使用手册/附录/`

### 3.2 更新记录格式

在 `附录/D_更新记录.md` 表格中追加：

```text
| YYYY-MM-DD | vX.X | 变更说明（如：新增 xx 技能到第8章；更新第19章控制台说明）|
```

## 四、流程小结（每次对话结尾检查）

1. **复盘**：已按 `运营中枢/参考资料/卡若复盘格式_固定规则.md` 输出完整复盘块。
2. **沉淀**：若有可复用经验 → 已写入 `经验库/待沉淀/`；若涉及技能新增/变更 → 已更新 SKILL_REGISTRY 与对应 SKILL.md。
3. **手册**：若有技能/板块/规范/网站功能新增或变更 → 已更新对应章节并追加附录 D 更新记录。
4. **Gitea**：若有文件变更 → 执行 `bash 01_卡资（金）/金仓_存储备份/Gitea管理/脚本/自动同步.sh`（单文件 >20MB 不提交）。
