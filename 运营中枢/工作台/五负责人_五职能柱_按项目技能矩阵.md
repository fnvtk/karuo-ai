# 五负责人 · 五职能柱 · 按项目技能矩阵

> **真源不重复**：人设摘要见 `运营中枢/技能路由/SKILL.md`；**技能编号、路径、触发词**以根目录 **`SKILL_REGISTRY.md`** 与 **`01_技能控制台.md`** 为准。本文件只做 **视图重构**：每人 **5 根职能柱**（任务域）+ **项目三级分类** + **技能编号锚点**（不写步骤、不复制 SKILL 正文）。
>
> **红线**：不改动 `01_～05_卡X` 物理目录结构；不迁移 SKILL 文件；仅增加/维护本「矩阵视图」。

**版本**：1.0 | **更新**：2026-03-29

## 一、全局风格与使用方式（统一口径）

| 维度 | 约定 |
|:---|:---|
| **路由优先级** | 多技能命中时 **金 → 水 → 木 → 火 → 土**；可用 `@负责人` / `@成员` 指定 |
| **命名** | 对外产品名保留原文（Cursor、MongoDB、Gitea、飞书/Lark 等）；体系内用 **卡若 / 卡若AI** |
| **宫殿名** | 与五行并行：间名、路径串见 `01_技能控制台.md` **〇、卡若记忆宫殿命名体系** |
| **五席人设** | 每组 5 席（MBTI/PDP/子五行）与 **@执行映射** 见 `五行编制_五席人设与执行映射.md`（**不替代**本矩阵的职能柱与项目级） |
| **开发专项** | 写代码/搭项目/部署/小程序与获客深链路 → **`开发域_Skill模块化索引.md`**（M1～M11）与本矩阵 **§三** 交叉查 |
| **Soul 套件** | K01～K03、Stream 规约真源在 `02_卡人（水）/水岸_项目管理/Soul技能归口/`；**水岸** 为套件掌管人 |

**本矩阵读法**：先定 **项目（§三 L1→L2→L3）** → 看 **主责职能柱与技能编号** → 回 **`SKILL_REGISTRY.md`** 取路径执行。
