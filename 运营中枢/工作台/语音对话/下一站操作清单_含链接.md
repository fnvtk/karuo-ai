## 零、自检前请确认（易踩坑）

1. **`pnpm test:voice` 的 BASE**：默认 `http://127.0.0.1:3000`。若 3000 不是本仓库 Next、或 **Docker 3102 为旧镜像**（会出现 `/api/chat/voice/status` 只有半截 JSON、`/api/chat/tts` **404**），请改用最新代码：`VOICE_TEST_BASE=http://127.0.0.1:3102`（仅当已 `--build` 最新）或本地 `pnpm dev -p 3005` 再测。  
2. **TTS 已开通仍 502**：若接口报错 **`resource pack allowance has been exhausted`**，须在 [语音合成控制台](https://console.cloud.tencent.com/tts) **购买资源包**或确认 **按量计费/余额** 可用，与「是否开通产品」是不同一步。

## 一、须你配合（密钥与控制台）

### 1. 腾讯云（识别 + 朗读 · 同一密钥）

| 步骤 | 做什么 | 链接 |
|:---|:---|:---|
| 1.1 | 登录后创建或查看 **访问密钥**（SecretId / SecretKey），子账号需勾选 ASR、TTS 相关策略 | [API 密钥管理](https://console.cloud.tencent.com/cam/capi) |
| 1.2 | 开通并确认 **一句话识别** 等产品可用、计费正常 | [语音识别控制台](https://console.cloud.tencent.com/asr) |
| 1.3 | 开通 **语音合成**；音色 ID 与站内预设一致（如智瑜 101001 等） | [语音合成控制台](https://console.cloud.tencent.com/tts) |

**写入位置（二选一或并存）：**

- 卡若 ai 网站：`site/.env.local` 中 `TENCENTCLOUD_SECRET_ID`、`TENCENTCLOUD_SECRET_KEY`、`TENCENTCLOUD_REGION`（常用 `ap-guangzhou`）、`TENCENT_ASR_ENG_TYPE`（常用 `16k_zh`）。  
- 或在控制台 **设置 → 语音** 里保存到 MongoDB（与 env 逻辑见 `TencentVoiceSettings` / `voice-settings.ts`）。

**可选开关：** 仅环境变量强制开识别 `TENCENT_ASR_ENABLED=true`；关腾讯朗读 `TENCENT_TTS_DISABLED=true`（见 `site/.env.example`）。

### 1.4 写入官网 Mongo（有密钥后一条命令）

在 **`卡若ai网站/site`** 的 `.env.local` 中填好 `TENCENTCLOUD_SECRET_ID` / `TENCENTCLOUD_SECRET_KEY`（及可选 region、eng），**Next 已启动**时执行：

```bash
cd "/Users/karuo/Documents/开发/3、自营项目/卡若ai网站/site"
pnpm run push:voice-from-env
# 非 3000：VOICE_PUSH_BASE=http://127.0.0.1:3102 pnpm run push:voice-from-env
```

不向聊天窗口粘贴密钥；脚本不会打印 Secret。

### 2. 火山引擎（已弃用 · 官网朗读）

官网 **`/api/chat/tts` 已改为仅腾讯云**，不再读取 `VOLC_*`。若历史文档仍提豆包，以代码与 `voice-settings.ts` 为准。

### 3. MongoDB（控制台设置持久化）

| 步骤 | 做什么 | 链接 |
|:---|:---|:---|
| 3.1 | 本机或服务器确保 Mongo 已起，URI 与 `MONGO_URI`、`MONGO_DB=karuo_site` 一致 | 无统一外链；本地常用 Docker：`卡若AI/01_卡资（金）/金仓_存储备份/datacenter` 内 `docker compose` |
| 3.2 | 网站进程 `USE_MONGO=true` | 见 `site/.env.example` |

### 4. 浏览器权限（麦克风）

- 打开官网控制台对话页后，地址栏允许 **麦克风**；若用 **HTTPS 以外** 的局域网 IP，部分浏览器会限制媒体设备，需 `localhost`、HTTPS 或合规证书站点。

### 5. OpenClaw 网关（TryVoice 7860）

| 步骤 | 做什么 | 链接 / 说明 |
|:---|:---|:---|
| 5.1 | 网关进程可达（本机 `127.0.0.1:18789` 或 Tailscale/域名端口） | 配置真源：`运营中枢/工作台/阿猫Mac_OpenClaw配置情况分析.md` |
| 5.2 | `OPENCLAW_GATEWAY_TOKEN` 与网关 `gateway.auth.token` **完全一致** | 复制自阿猫 `~/.openclaw/openclaw.json`（**勿提交 Git**） |

**写入位置：** `运营中枢/工作台/语音对话/.env`（由同目录 `env.example` 复制）。

### 6. Cursor API 费用与 Token 用量（每轮对话自查）

- 助手无法在对话内读取你的 Cursor 账单；请在客户端查看用量与订阅。  
- [Cursor 设置 / 账户相关入口](https://cursor.com/settings)（登录后查看 Usage / Billing 以实际 UI 为准）。

## 三、验收顺序（建议按序打勾）

1. **自动化**：在 `卡若ai网站/site` 执行 `pnpm run test:voice`（可先起 `pnpm dev`，见 `语音对话/README.md`）。  
2. **Mongo + env**：`pnpm dev` 无报错，控制台可打开。  
3. **腾讯 ASR**：按住说话 / 说完即发，转写中文正常。  
4. **朗读**：选腾讯云音色 → 云端有声音；失败时 ChatPanel 会回退浏览器朗读。  
5. **唤醒**：说「乖乖」或「乖乖来」→ 应触发录音逻辑（见 `WakeWordListener`）。  
6. **TryVoice**：7860 页可连网关、对话有往返。  

详细矩阵：`[语音能力验收矩阵.md](./语音能力验收矩阵.md)`。
