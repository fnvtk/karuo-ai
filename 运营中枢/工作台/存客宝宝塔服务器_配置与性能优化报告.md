## 三、性能情况

### 3.1 历史监控摘要（来自《存客宝_公网数据问题与服务器深度分析报告》）

| 指标 | 典型/峰值 | 说明 |
|------|-----------|------|
| **CPU 使用率** | 平均约 11%，峰值约 26% | 无过载，有余量 |
| **公网出带宽** | 峰值约 **41.59 Mbps** | 接近 50M 上限，存在突发与计费风险 |
| **公网入带宽** | 短时尖峰 **33.07 Mbps**（如 14:44） | 入向突发明显，需结合日志区分业务/异常 |
| **外网出流量** | 某时段均值约 1.45 MB，峰值 2.45 MB | 出向持续活跃，直接关系公网费用 |
| **入包量** | 尖峰约 **2864 个/s** | 与入带宽尖峰同步，需排查是否正常请求 |

### 3.2 已识别问题（摘录）

| # | 问题 | 严重程度 |
|---|------|----------|
| 1 | 入向流量与入包量短时尖峰（如 14:44） | 高：需确认业务预期或排查异常/攻击 |
| 2 | 出向流量长时间维持较高 | 中：推高公网费用，需区分正常业务与可优化项 |
| 3 | 公网带宽利用率接近 50M 上限 | 中：活动或异常时易打满，影响体验与计费 |
| 4 | 缺少该实例的流量/费用告警 | 中：异常或费用突增难以及时发现 |
| 5 | 入向尖峰根因未完全定位 | 高：需结合 Nginx/应用/防火墙日志做一次完整排查 |

### 3.3 当前无法直接获取的数据（因 SSH/API 限制）

- 实时：内存使用率、磁盘使用率、各站点 QPS、数据库连接数。
- 获取方式建议：
  - 登录 **宝塔面板** 首页与「监控」查看实时与历史曲线。
  - 将执行机 IP 加入 **API 白名单** 后，运行 `存客宝_CPU与网络用量查询.py`、`存客宝_腾讯云API全量分析.py`。
  - **腾讯云控制台** → 该实例 → 监控：CPU、公网出/入带宽、流量。

## 四、常规性优化建议

### 4.1 安全与可运维性

| # | 建议 | 操作 |
|---|------|------|
| 1 | **开放/恢复 SSH 或 API 可访问** | 若需远程运维：在安全组已放行 22 的前提下，在机器上检查 `sshd` 监听、防火墙、`PermitRootLogin` 等；或保证至少一种方式（VNC + 宝塔终端）可用。 |
| 2 | **宝塔 API 白名单** | 将常用运维 IP（如本机、跳板机）加入面板「设置 → API 接口」白名单，便于脚本拉取状态与自动化。 |
| 3 | **安全组最小化** | 仅开放业务与运维必需端口（如 80、443、9988、22 或指定 SSH 端口），避免 0.0.0.0/0 大范围放行。 |
| 4 | **定期更新与漏洞修复** | 系统与宝塔、Nginx/PHP/MySQL 等定期更新，关注安全公告。 |

### 4.2 监控与告警

| # | 建议 | 操作 |
|---|------|------|
| 5 | **为 ins-ciyv2mxa 配置告警** | 腾讯云云监控：公网出带宽 > 阈值（如 35 Mbps）、出网流量日/小时用量、费用日预算（若支持）。 |
| 6 | **定期复跑全量分析** | 执行 `存客宝_腾讯云API全量分析.py`，对比 CVM 状态、CPU、公网带宽与账单趋势。 |
| 7 | **保留基线** | 保存本次及后续报告与监控截图，便于费用争议或故障时对比时间线。 |

### 4.3 成本与计费

| # | 建议 | 操作 |
|---|------|------|
| 8 | **核对账单与流量** | 腾讯云「费用中心 → 账单明细」按产品/按资源查看「云服务器」「公网网络」「流量」，确认与监控曲线是否一致。 |
| 9 | **评估带宽计费方式** | 若为按量且出流持续较高，可评估改为包月带宽或「按带宽峰值」是否更省。 |
| 10 | **费用/流量告警** | 设置月度预算或流量阈值告警，避免「感觉乱扣费」时无据可查。 |

## 五、性能优化建议清单

### 5.1 网络与带宽

| # | 建议 | 说明 |
|---|------|------|
| 1 | **静态资源与下载走 CDN** | 将静态资源、大文件下载等迁到 CDN，降低 CVM 出网流量与带宽峰值。 |
| 2 | **应用层限流** | Nginx `limit_req`、应用层限流，防止单次活动或爬虫把 50M 打满。 |
| 3 | **异常 IP/地域限制** | 结合 Nginx/宝塔/防火墙对异常 IP 或地域做限制，减轻入向尖峰与攻击面。 |
| 4 | **排查入向尖峰** | 对 14:44 类时段：查 Nginx/应用访问日志、防火墙/安全日志、`ss -tnp`/连接数，区分正常业务与异常。 |

### 5.2 应用与数据库

| # | 建议 | 说明 |
|---|------|------|
| 5 | **缓存策略** | 对可缓存接口/页面设置合理缓存（Nginx 或应用层），降低重复请求与出流。 |
| 6 | **数据库与连接池** | 优化慢 SQL、索引；应用侧使用连接池，避免连接数过高占满内存或连接。 |
| 7 | **异步与队列** | 非实时任务（如同步、推送、报表）走队列异步处理，平滑 CPU 与带宽。 |

### 5.3 系统与资源

| # | 建议 | 说明 |
|---|------|------|
| 8 | **日志轮转与清理** | 配置 Nginx/应用日志轮转、定期清理旧日志，避免占满磁盘。 |
| 9 | **进程与守护** | 关键服务用 systemd 或宝塔守护，避免裸 nohup 导致进程退出难发现。 |
| 10 | **资源基线** | 在可登录或 API 可用时，记录内存/磁盘/连接数基线，便于后续对比扩容或优化。 |

### 5.4 高可用与架构（可选）

| # | 建议 | 说明 |
|---|------|------|
| 11 | **与 kr宝塔 分工明确** | 明确主备或读写分离、域名与 Nginx 反代关系，避免重复部署与流量浪费。 |
| 12 | **关键数据与配置备份** | 数据库与关键配置定期备份到对象存储或另一地域，便于灾备与恢复。 |

## 七、小结

- **配置**：存客宝宝塔服务器为 **2 核 16G、50M 公网带宽**，腾讯云 CVM（ins-ciyv2mxa，广州），三台宝塔中规格最高，适合作为私域主业务与数据库主机。
- **网络**：50M 带宽为费用与流量重点；历史监控显示出/入峰值接近 50M，存在突发与计费风险，建议 CDN、限流、告警与账单核对。
- **项目**：承担存客宝主站/API/管理及与卡若AI 网关（反代到 kr宝塔）等；具体站点与端口以宝塔面板为准。
- **性能**：CPU 有余量；公网带宽与出流是主要瓶颈与成本点，建议按「常规性优化」与「性能优化清单」逐项落实，并优先完成入向尖峰根因排查与告警配置。
