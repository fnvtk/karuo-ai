## 日志记录规范

### 单条记录格式

```markdown
### [HH:MM] 任务标题
- **执行人**：金剑
- **状态**：✅完成 / ⏳进行中 / ❌失败
- **耗时**：5分钟
- **内容**：具体做了什么
- **结果**：产出了什么
```

### 日报汇总格式

```markdown
# 2026-01-29 执行日报

## 📊 今日统计
| 团队 | 执行任务数 | 完成数 | 进行中 |
|:---|:---|:---|:---|
| 金组 | 5 | 4 | 1 |
| 水组 | 3 | 3 | 0 |
| ... | ... | ... | ... |

## 🔥 重点完成
1. [金剑] 服务器部署完成
2. [火炬] 新功能开发上线

## ⏳ 进行中
1. [木叶] 视频切片处理中

## 📝 明日计划
1. ...
```

## 自动记录触发

当执行以下操作时，自动记录到对应成员日志：

| 触发场景 | 记录到 | 示例 |
|:---|:---|:---|
| 服务器操作 | 金剑.md | "部署了XX项目" |
| 数据库操作 | 金盾.md | "备份了XX数据库" |
| 文件整理 | 水溪.md | "整理了XX目录" |
| 需求拆解 | 水泉.md | "拆解了XX需求" |
| 视频切片 | 木叶.md | "切片了XX视频" |
| 代码开发 | 火炬.md | "开发了XX功能" |
| Bug修复 | 火锤.md | "修复了XX问题" |
| 商业分析 | 土基.md | "分析了XX项目ROI" |
