+### 双服务器标准自动化流程（A/B 同步）
+
+1. **预检**：连通性、磁盘、CPU、Docker/PM2进程、端口占用、证书有效期。

+4. **验收A**：健康接口、关键业务接口、首页可用性、错误日志扫描。
+5. **发布B**：与A同流程执行，保证版本一致。
+6. **验收B**：同A；通过后执行双机一致性校验（版本号/配置哈希）。
+7. **收尾**：写入部署记录（时间、版本、结果、回滚点、异常）。
+
+### 失败回滚规则
+
+- A失败：立即回滚A并停止B发布。
+- B失败：回滚B，并根据风险级别决定是否回滚A到同版本。
+- 任一失败必须输出「失败原因 + 已执行回滚动作 + 下一步建议」。

## 常见问题速查

### Q1: 外网无法访问（ERR_EMPTY_RESPONSE）

**原因**: 腾讯云安全组只开放443端口  
**解决**: 必须配置SSL证书；Nginx配置添加443监听。

### Q2: Node项目启动失败（Could not find production build）

**原因**: 使用 `npm run start` 但未执行 `npm run build`  
**解决**: 先 `pnpm run build` 再重启。

### Q3: 端口冲突（EADDRINUSE）

```bash
ss -tlnp | grep :端口号
# 修改 package.json 中的端口: "start": "next start -p 新端口"
```

### Q4: DNS被代理劫持

关闭代理或用手机4G测试。

### Q5: 宝塔与PM2冲突

停止所有独立PM2: `pm2 kill`；只使用宝塔界面管理。

## 三、标准部署流程

1. 读取部署配置（JSON）
2. 修复API白名单（环境修复）
3. 查询服务器状态（宝塔API）
4. 探测空闲端口（SSH）
5. 创建项目目录（宝塔API）
6. 上传代码压缩包（SSH/scp）
7. 解压 → 安装依赖 → 构建 → 修正启动端口
8. 宝塔添加项目 / 绑定域名 / 配置HTTPS
9. 健康检查（端口监听、HTTP 200、HTTPS 200）
10. 输出部署报告

## 四、问题诊断流程

1. 能连接服务器？→ 否：检查 SSH、宝塔面板、API 白名单
2. 项目端口监听？→ 否：检查 PM2、宝塔项目状态、端口占用
3. Nginx 配置正确？→ 否：nginx -t、server_name、proxy_pass
4. SSL 证书有效？→ 否：证书过期、路径、通配符
5. DNS 解析正确？→ 否：dig、阿里云DNS、本地缓存、代理劫持

## 二、在 NAS 上部署 Gitea（一次性）

### 2.1 创建目录并写入 docker-compose

在**能 SSH 到 NAS 的电脑**上执行（或复制到 NAS 上执行）：

```bash
# 创建目录
ssh nas "mkdir -p /volume1/docker/gitea/data"

# 写入 docker-compose（Gitea 官方镜像，数据持久化到 /volume1/docker/gitea/data）
ssh nas "cat > /volume1/docker/gitea/docker-compose.yml << 'ENDOFFILE'
version: \"3.8\"
services:
  server:
    image: docker.gitea.com/gitea/gitea:latest
    container_name: gitea
    environment:
      - USER_UID=1026
      - USER_GID=100
      - GITEA__server__DOMAIN=open.quwanzhi.com
      - GITEA__server__ROOT_URL=http://open.quwanzhi.com:3000/
      - GITEA__server__HTTP_PORT=3000
    restart: unless-stopped
    ports:
      - \"3000:3000\"
    volumes:
      - /volume1/docker/gitea/data:/data
      - /etc/timezone:/etc/timezone:ro
      - /etc/localtime:/etc/localtime:ro
ENDOFFILE"
```

### 2.2 启动容器

```bash
ssh nas "cd /volume1/docker/gitea && echo 'zhiqun1984' | sudo -S /volume1/@appstore/ContainerManager/usr/bin/docker compose up -d"
```

### 2.3 确认运行

```bash
ssh nas "/volume1/@appstore/ContainerManager/usr/bin/docker ps | grep gitea"
```

浏览器访问 **http://192.168.1.201:3000**，应看到 Gitea 首次安装向导。
