## 一、问题现象

| 现象 | 说明 |
|------|------|
| 接口/页面返回 **404** | 浏览器或小程序请求 API/页面得不到正确响应 |
| 响应内容为 **"No input file specified."** | PHP 未收到正确脚本路径，多为路径含中文导致 |
| 小程序/H5 空白页 | 前端依赖的接口 404，数据拿不到 |

**根因**：站点根目录或上级路径包含**中文**（如 `自营`、`客户`、`扩展`、`存客宝备用服务器`），Nginx 传给 PHP-FPM 的 `SCRIPT_FILENAME` 或 PHP 的 `open_basedir` 在解析时出问题。

---

## 二、处理原则（不改中文名）

1. **物理目录保持中文**：不重命名 `/www/wwwroot/自营` 等目录。
2. **用英文符号链接暴露给 Nginx/PHP**：在 `/www/wwwroot` 下建 `self` → `自营` 等，Nginx 的 `root` 只写英文路径。
3. **open_basedir 同时兼容英文与中文路径**：`.user.ini` 里把「英文路径」和「中文路径」都写上，避免 PHP 越权或报错。

## 四、标准处理步骤（遇到 404/No input file 时按此执行）

### 步骤 1：确认根因

- 看 Nginx 站点配置里 `root` 是否含中文。
- 看响应内容是否含 `No input file specified.`。
- 若为 PHP 站点，看站点 `public`（或根目录）下是否有 `.user.ini` 及 `open_basedir`。

### 步骤 2：保证英文符号链接存在

- 若站点在 `自营` 下 → 使用 `self`（如 `/www/wwwroot/self/...`）。
- 若在 `客户`/`扩展`/`测试`/`归档` 下 → 使用 `client`/`ext`/`test`/`archive`。
- 若在 `扩展/ckb-backup/存客宝备用服务器` 下 → 使用 `ext/ckb-backup/ckb-backup-server`。
- 如上节命令，缺哪个链接建哪个。

### 步骤 3：改 Nginx 配置

- 编辑站点对应的 `*.conf`（如 `/www/server/panel/vhost/nginx/站点.conf`）。
- 将 `root` 从中文路径改为**英文符号链接路径**（不改目录原名，只改配置里的路径）。

示例：

```nginx
# 修改前
root /www/wwwroot/自营/quwan/api/public;

# 修改后
root /www/wwwroot/self/quwan/api/public;
```

改完后：`nginx -t && nginx -s reload`。

### 步骤 4：修正 PHP open_basedir（仅 PHP 站点且有 .user.ini 时）

- 进入站点运行目录（如 `public`）。
- 若存在 `.user.ini`，先解除防篡改：`chattr -i .user.ini`。
- 将 `open_basedir` 改为同时包含「英文路径」和「中文路径」（以站点实际目录为准），例如：

```ini
open_basedir=/www/wwwroot/self/quwan/api/:/www/wwwroot/自营/quwan/api/:/tmp/
```

- 写回后：`chattr +i .user.ini`，并重启对应 PHP-FPM（如 `service php-fpm-72 restart`）。

### 步骤 5：验证

- 浏览器或 curl 请求该站点接口/页面，应返回 200 或业务正常状态，不再出现 404 或 "No input file specified."。
- 若有小程序/H5，再测一遍前端是否还空白。
