### 4.1 阶梯式合伙机制

### 4.3 云阿米巴分润公式

```
【月度可分配利润】= 总营收 - 流量成本 - 系统成本 - 人员底薪
【团队分润池】= 可分配利润 × 30%

【个人分润】= 
  基础提成（按岗位固定比例）
  + 模块奖金（按完成度）
  + 贡献分润（个人贡献占团队比例 × 团队分润池）
  + 徒弟奖金（徒弟业绩 × 5%，仅合伙人）
```

## 一、C2B模式总览

### 1.1 什么是C2B用户共创

```
┌─────────────────────────────────────────────────────────────────┐
│                    C2B用户共创模式                               │
│              Consumer to Business - 用户驱动商业                  │
└─────────────────────────────────────────────────────────────────┘

传统模式 (B2C)：
企业生产 → 渠道分发 → 用户购买

共创模式 (C2B)：
用户需求 → 用户参与 → 用户传播 → 用户受益
    │           │           │           │
    ▼           ▼           ▼           ▼
 需求共创    产品共创    内容共创    利益共创
```

### 1.2 玩值电竞C2B四层共创

| 共创层级 | 用户角色 | 参与方式 | 用户收益 | 平台收益 |
|:---|:---|:---|:---|:---|
| **内容共创** | 内容贡献者 | 分享体验、案例、心得 | 积分+曝光 | 降低内容成本40% |
| **产品共创** | 产品测试员 | 参与产品改进、内测 | 优先体验+现金 | 迭代周期缩短30% |
| **服务共创** | 服务优化者 | 反馈问题、优化建议 | 专属权益 | 满意度提升40% |
| **品牌共创** | 品牌大使 | 传播推广、口碑裂变 | 分润+荣誉 | 获客成本降低50% |

## 一、云阿米巴模式说明

### 1.1 三大核心原则

| 原则 | 解释 | 玩值应用 |
|:---|:---|:---|
| **分不属于对方的钱** | 只分增量，不动存量 | 只分新增利润，不分老客户收益 |
| **按创造价值分钱** | 谁创造，谁受益 | 按贡献GMV/进粉数计算 |
| **分现金不分所有权** | 降低风险，灵活退出 | 不占股份，按月结算分润 |

### 1.2 股权 vs 分润对比

| 维度 | 传统股权模式 | 云阿米巴分润模式 |
|:---|:---|:---|
| **绑定程度** | 强绑定，退出难 | 弱绑定，随时可退 |
| **风险承担** | 共担亏损 | 只分利润，不分亏损 |
| **激励效果** | 长期激励 | 即时激励 |
| **操作复杂度** | 需工商变更 | 合同约定即可 |
| **税务成本** | 分红需缴税 | 按劳务/服务处理 |

## 三、分润机制设计

### 3.1 利润分配公式

```
【月度可分配利润】= 月度总营收 - 运营成本 - 预留资金(20%)

【运营成本】= 人员底薪 + 流量成本 + 系统成本 + 其他费用

【分润池】= 可分配利润 × 分润总比例(60%)

【个人分润】= 分润池 × 个人分润比例
```

### 3.2 分润比例明细

| 分润对象 | 基础分润 | 超额分润 | 计算方式 |
|:---|:---|:---|:---|
| **创始人** | 净利润×40% | 超额部分×50% | 月结 |
| **联创合伙人** | 净利润×10% | - | 月结 |
| **增长负责人** | 底薪+GMV×3% | 超额部分GMV×5% | 月结 |
| **内容运营** | 底薪+进粉×3元 | 爆款奖200元/条 | 月结 |
| **用户运营** | 底薪+成交×10% | C2B订单×15% | 月结 |
| **期权池** | 预留 | - | 年度分配 |

### 3.3 分润计算示例

**假设月度数据**：
- 月度总营收：50万
- 运营成本：20万
- 净利润：30万
- 预留资金：6万（20%）
- 可分配利润：24万

| 分润对象 | 计算方式 | 金额 |
|:---|:---|:---|
| 创始人 | 24万×40% | 9.6万 |
| 联创合伙人 | 24万×10% | 2.4万 |
| 增长负责人 | 底薪0.8万+GMV×3%=1.5万 | 2.3万 |
| 内容运营 | 底薪0.5万+进粉奖+爆款奖 | 1.2万 |

## 四、期权激励设计

### 4.1 期权池分配规则

| 期权类型 | 分配对象 | 分配比例 | 授予条件 |

### 4.2 期权成熟机制

```

           │               │               │               │
           └───────────────┴───────────────┴───────────────┘
                              需持续在职
```

### 4.3 期权行权条件

| 条件 | 要求 |
|:---|:---|
| **在职要求** | 行权时需仍在职（离职后90天内可行权已成熟部分） |
| **业绩要求** | 累计KPI达标率≥80% |
| **行权价格** | 授予时估值的20% |
| **锁定期** | 行权后锁定1年 |

## 六、考核流程

### 6.1 月度考核流程

```
每月1日：下发考核表，确认目标
    ↓
每周五：周数据同步，过程指标检查
    ↓
月末最后一天：数据截止，自评
    ↓
次月3日前：上级评分，反馈面谈
    ↓
```

| **问题分析** | 没做好的地方，原因分析 | 5分钟 |

## 三、核心提示词 v4.0

```markdown
# Role: 卡若 - 玩值电竞操盘手

## 1. 角色设定
你是**卡若**，玩值电竞项目的操盘手。
- **项目定位**：电竞行业的"私域银行"，收编电竞明星，构建标准化获客渠道
- **核心模式**：云阿米巴（不占股、分现钱、稳流量）
- **口头禅**："别整那些没用的，直接说怎么分钱"

## 2. 防重复机制
在生成任何内容前，先检查：
1. 是否已有相关文档？→ 引用而非重写
2. 数据是否已定义？→ 使用统一数据源
3. 话术是否已存在？→ 指向已有话术库

**统一数据源**：
- 产品价格 → 产品矩阵报价表
- 用户分类 → 用户画像分析表
- 分润比例 → 渠道分销政策表
- KPI指标 → 团队KPI考核表

## 3. 按需生成逻辑

### 场景A：需要新文档
1. 确认现有文档不能满足需求
2. 按五行框架定位归属
3. 引用已有数据，不重复定义
4. 输出Markdown表格格式

### 场景B：需要补充内容
1. 找到对应文档
2. 增量补充，不覆盖原有
3. 保持格式一致

### 场景C：需要执行方案
1. 引用现有SOP/话术
2. 根据具体场景微调
3. 输出可执行的行动清单

## 4. 输出格式要求
- 必须使用Markdown表格
- 数据要具体、可执行
- 引用已有文档时标注来源
- 新增内容标注【新增】

## 5. 快捷指令

| 指令 | 动作 |
|:---|:---|
| "查产品" | 返回产品矩阵报价表核心内容 |
| "查用户" | 返回A/B/C/D用户画像摘要 |
| "查话术" | 返回对应场景的成交话术 |
| "查分润" | 返回渠道分销政策 |
| "查KPI" | 返回对应岗位的KPI指标 |
| "生成朋友圈" | 基于已有模板生成今日朋友圈 |
| "生成日报" | 基于模板生成今日运营日报 |

现在，请告诉我你需要什么？
```
