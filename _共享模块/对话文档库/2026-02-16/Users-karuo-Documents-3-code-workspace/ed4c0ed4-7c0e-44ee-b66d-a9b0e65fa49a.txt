### 2. 在 NAS 上用 Docker 部署 Gitea

> 下面假设群晖 NAS 是 `192.168.2.201`，SSH 用户是 `fnvtk`，数据放在 `volume1/docker/gitea`。

#### 2.1 在 NAS 上创建目录

在你 Mac 上执行（通过 SSH 连接 NAS）：

```bash
ssh fnvtk@192.168.2.201

# 创建 Gitea 相关目录
sudo mkdir -p /volume1/docker/gitea/{gitea,db}
sudo chown -R 1000:1000 /volume1/docker/gitea/gitea
sudo chown -R 1000:1000 /volume1/docker/gitea/db

# 在目录里准备 docker-compose
cd /volume1/docker/gitea
```

#### 2.2 创建 `docker-compose.yml`

在 `/volume1/docker/gitea/docker-compose.yml` 写入（可用 vi 或群晖 File Station 编辑）：

```yaml
version: "3"

services:
  db:
    image: postgres:15
    container_name: gitea-db
    restart: always
    environment:
      POSTGRES_USER: gitea
      POSTGRES_PASSWORD: gitea_password   # 可改复杂一点
      POSTGRES_DB: gitea
    volumes:
      - /volume1/docker/gitea/db:/var/lib/postgresql/data
    networks:
      - gitea-net

  gitea:
    image: gitea/gitea:1.22
    container_name: gitea
    restart: always
    environment:
      USER_UID: 1000
      USER_GID: 1000
      DB_TYPE: postgres
      DB_HOST: db:5432
      DB_NAME: gitea
      DB_USER: gitea
      DB_PASSWD: gitea_password
      # 设置根 URL，后面可改为 Web Station 反代路径
      ROOT_URL: http://192.168.2.201:3000/
      # 镜像默认同步间隔（例：10 分钟，可改 1m/30m/1h）
      MIRROR_DEFAULT_INTERVAL: 10m
    volumes:
      - /volume1/docker/gitea/gitea:/data
      - /etc/timezone:/etc/timezone:ro
      - /etc/localtime:/etc/localtime:ro
    ports:
      - "3000:3000"   # Web UI
      - "2222:22"     # Gitea 内置 SSH（可选）
    depends_on:
      - db
    networks:
      - gitea-net

networks:
  gitea-net:
    driver: bridge
```

然后在 NAS 上执行：

```bash
cd /volume1/docker/gitea
docker compose up -d
# 或 docker-compose up -d 取决于 NAS 上命令名字
```

验证容器：

```bash
docker ps | grep gitea
```

### 4. 配置 GitHub → Gitea 仓库镜像（同步）

目标：以 **GitHub 为源头**，Gitea 作为「镜像 + 备份 + 浏览」端，自动定时拉取更新。

#### 4.1 在 GitHub 上创建一个 PAT（只读）

1. 登录 GitHub → 右上角头像 → `Settings` → `Developer settings` → `Personal access tokens`。
2. 创建 `Fine-grained` 或 `Classic` token：
   - 权限：至少勾选 **repo: read** 权限（只读足够）。
3. 复制 token（只显示一次）。

#### 4.2 在 Gitea 中配置镜像仓库

方式一：单仓库配置（推荐，简单直接）：

1. 登录你刚建的 Gitea 管理员账号。
2. 顶部点击「**新建迁移 / New Migration**」（不同版本菜单名字略有差异）：
   - 源仓库类型：`GitHub`
   - 仓库克隆地址：填写 GitHub 的 HTTPS 地址，例如  
     `https://github.com/fnvtk/your-repo.git`
   - 认证部分：
     - 用户名：你的 GitHub 用户名
     - 密码/Token：刚才生成的 PAT
   - 选项：
     - 勾选「**镜像此仓库 / This is a mirror**」
3. 保存后，Gitea 会：
   - 立即拉取 GitHub 仓库完整内容；
   - 按 `MIRROR_DEFAULT_INTERVAL` 设定的时间周期自动拉取新提交。

你可以为每个需要备份的 GitHub 仓库重复这个操作。

> **实时程度说明**：  
> - 默认镜像间隔可以设置到 **1 分钟级别**，已经接近「准实时」。  
> - 如果你以后想做到「GitHub 一有 push 就立刻同步」，可以再加一层 **GitHub Webhook → Gitea 同步接口**，但这会略微复杂，此处先用「定时镜像」方案即可。

### 5. （可选）通过 `open.quwanzhi.com` 把 Gitea 暴露到外网

如果你希望在外网访问这个 Git 服务，可以基于你现有的 FRP 架构加一条 HTTP 隧道。

整体思路：

1. 在 NAS 上 Gitea 仍监听 `3000` 端口。
2. 在 `frpc_nas.toml` 中新增一个 HTTP 类型的 `proxies`，例如路径 `/gitea/` 反代到 NAS Gitea。
3. 在公网服务器 Nginx（open.quwanzhi.com）上对应增加 location，把 `/gitea/` 转发给 frps。

示意（仅给你概念，不强制马上动）：

```toml
# frpc_nas.toml 里
[[proxies]]
name = "gitea-http"
type = "http"
localPort = 3000
customDomains = ["open.quwanzhi.com"]
locations = ["/gitea/"]
```

然后外网就能用：

- `https://open.quwanzhi.com/gitea/` 访问你的 Git 服务。
