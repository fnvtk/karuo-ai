## 二、工作流（只看不说，不操作）

文档里定义的工作流可以概括成三条线：**业务/系统层**、**指令执行**、**设备连接**。下面都是「他们文档里写的工作流是什么」，没有执行任何一步。

### 1. 核心业务流程（业务视角）

- **存客宝后端**（ThinkPHP）→ 调 **WorkPhone SDK 服务器**（API Gateway + 各服务）→ **WebSocket Hub** 连多台设备。
- 设备端：每台手机/模拟器上是 **工作手机 Agent**（Frida / u2 / scrcpy），再操作微信、抖音、小红书等 APP。
- 即：**存客宝 → SDK 服务器 → 多设备** 的三角关系。

### 2. 指令执行流程（发一条「发微信」之类）

- 存客宝：`POST /execute`（例如 wechat, send_message）。
- SDK 服务器：查设备连接 → 通过 **WebSocket** 把 execute 命令推到对应设备。
- 设备：用 **uiautomator2 / ADB** 在目标 APP 上操作。
- 设备把执行结果经 WebSocket 回传 → SDK 再 HTTP 返回给存客宝。

（你看到的「开发工作手机」的「工作流」里，**执行一条指令**就是这条链。）

### 3. 设备连接流程（设备怎么上线）

- 设备主动 **WebSocket** 连上 SDK 服务器。
- 发 **register**（设备信息）→ 服务器落库 **MongoDB**、在线状态写 **Redis**。
- 之后按固定间隔发 **heartbeat**，服务器刷新 TTL、回 **pong**，保持在线。
