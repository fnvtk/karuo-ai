### 1. React 组件模板

```tsx
import React, { useState, useEffect } from 'react';
import { message, Spin } from 'antd';

interface Props {
  id: number;
}

/**
 * 用户详情组件
 * 展示用户的详细信息
 */
const UserDetail: React.FC<Props> = ({ id }) => {
  const [loading, setLoading] = useState(false);
  const [data, setData] = useState<UserInfo | null>(null);

  // 加载用户数据
  useEffect(() => {
    const fetchData = async () => {
      setLoading(true);
      try {
        const res = await getUserInfo(id);
        setData(res);
      } catch (error) {
        message.error('加载失败');
      } finally {
        setLoading(false);
      }
    };
    fetchData();
  }, [id]);

  if (loading) return <Spin />;
  if (!data) return null;

  return (
    <div className="user-detail">
      <h2>{data.name}</h2>
      {/* 更多内容 */}
    </div>
  );
};

export default UserDetail;
```

### 2. FastAPI 路由模板

```python
from fastapi import APIRouter, Depends, HTTPException
from typing import List

from app.schemas.user import UserCreate, UserResponse
from app.services.user_service import UserService

router = APIRouter(prefix="/users", tags=["用户管理"])


@router.get("/", response_model=List[UserResponse], summary="获取用户列表")
async def get_users(
    skip: int = 0,
    limit: int = 10,
    service: UserService = Depends()
):
    """
    获取用户列表
    
    - **skip**: 跳过数量
    - **limit**: 返回数量
    """
    return await service.get_users(skip=skip, limit=limit)


@router.post("/", response_model=UserResponse, summary="创建用户")
async def create_user(
    user: UserCreate,
    service: UserService = Depends()
):
    """创建新用户"""
    return await service.create_user(user)
```

```bash
# 查看运行中的容器
docker ps

# 查看所有容器（包括停止的）
docker ps -a

# 启动/停止/重启容器
docker start container_name
docker stop container_name
docker restart container_name

# 进入容器
docker exec -it container_name /bin/bash

# 查看容器日志
docker logs -f container_name
docker logs --tail 100 container_name
```

### 2. 镜像操作

```bash
# 查看本地镜像
docker images

# 拉取镜像
docker pull image_name:tag

# 构建镜像
docker build -t image_name:tag .

# 删除镜像
docker rmi image_name:tag
```

### 3. Docker Compose

```bash
# 启动所有服务
docker-compose up -d

# 停止所有服务
docker-compose down

# 重启某个服务
docker-compose restart service_name

# 查看服务日志
docker-compose logs -f service_name

# 查看服务状态
docker-compose ps
```

## Dockerfile 模板

### Python FastAPI

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制代码
COPY . .

# 启动命令
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

EXPOSE 8000
```

### Node.js

```dockerfile
FROM node:18-alpine

WORKDIR /app

# 安装依赖
COPY package*.json ./
RUN npm ci --only=production

# 复制代码
COPY . .

# 构建
RUN npm run build

# 启动命令
CMD ["npm", "start"]

EXPOSE 3000
```

## 常见问题处理

### 1. 容器无法启动

```bash
# 查看容器日志
docker logs container_name

# 检查容器状态
docker inspect container_name

# 重新创建容器
docker-compose up -d --force-recreate service_name
```

### 2. 磁盘空间不足

```bash
# 清理未使用的资源
docker system prune -a

# 清理未使用的卷
docker volume prune

# 查看磁盘使用
docker system df
```

### 3. 网络问题

```bash
# 查看网络
docker network ls

# 检查容器网络
docker network inspect network_name

# 重建网络
docker-compose down
docker-compose up -d
```

## 最佳实践

1. **使用 .dockerignore**：排除不需要的文件
2. **多阶段构建**：减小镜像体积
3. **使用 alpine 基础镜像**：更小更安全
4. **定期清理**：清理未使用的镜像和容器
5. **数据卷持久化**：重要数据使用 volumes

## ⚠️ 必守规则

**每次执行文件插入/整理/归档/上传操作前**，必须完成以下流程，**不得跳过**直接执行：

```
观察目标 → 思考对标 → 针对提问 → 回溯扩展 → 解答澄清 → 执行操作
```

### 一致性、标准性原则（不可跳过）

**先读后写**：必须先读取**目标位置周围/附近的原有文档**，分析其格式、结构、风格，再生成或修改待插入内容。

| 维度 | 要求 | 完成标准 |
|:---|:---|:---|
| **标题** | 与周围文档的命名规范一致（如「YYYY-MM-DD 主题」「会议纪要：XXX」） | 能复述周围文档的标题格式 |
| **内容** | 段落结构、层级、列表/表格风格与原有文档一致 | 能描述原有文档的内容结构 |
| **风格** | 语气、长度、重点呈现方式与原有文档协调 | 插入后整体风格统一 |
| **附件** | 若有附件，格式与原有文档的附件规范一致 | 附件命名、类型符合规范 |

**结论**：以**原有文档为范本**，按同一标准生成；**不得**在未读取、未分析周围文档的情况下直接插入。

## 一、流程详解

### 1. 观察目标（Read Target & Context）

| 动作 | 说明 | 完成标准 |
|:---|:---|:---|
| **读取周围文档** | **优先**读取目标位置同级/父级/子级的**原有文档**（标题、内容、附件） | 能复述周围文档的格式、结构、风格 |
| 分析原有格式 | 提取标题规范、段落结构、列表/表格风格、附件命名 | 形成「格式范本」清单 |
| 读取目标文件 | 理解待插入/整理的文件内容、结构、格式 | 能复述核心内容与结构 |
| 读取目标位置 | 目标目录/知识库/飞书节点及其**周围文档** | 了解同级、父级、子级结构 |
| 观察周围文档 | 同级/父级/子级文档的**美观度、整洁度、易读度** | 能描述现有风格与可优化点 |
| 识别约束 | 格式限制、命名规范、容量限制 | 明确可执行边界 |

### 2. 思考对标（Think & Benchmark）

| 动作 | 说明 | 完成标准 |
|:---|:---|:---|
| 理解目标文件 | 内容主题、受众、用途、与现有文档的关系 | 能说明「放这里为什么合适」 |
| 格式适配 | 目标平台（飞书/本地）的格式要求、块类型、样式 | 明确输出形态（纯文本/表格/标题层级） |
| **美观度/整洁度/易读度** | 参考周围文档风格，对待插入内容做优化 | 插入后与周围文档风格协调、易读 |

### 3. 针对提问 → 4. 回溯扩展 → 5. 解答澄清

基于观察与对标，列出 **1-3 个关键问题**，必须回答后再执行：位置合理性、格式与呈现、命名与检索。解答清晰、方案明确、**内容已按周围文档格式做一致性修改**后，再执行。

### 6. 执行操作（Execute）

在解答清晰、方案明确、**内容已按周围文档格式做一致性/标准性修改**、美观度优化完成后，执行文件生成/插入/整理/上传。

## Cursor shell 环境超时

- **现象**：Cursor 终端执行命令时报 shell 解析超时  
- **解决**：重启 Cursor / 检查 `.zshrc` 是否有阻塞命令 / 清理 PATH 中过多路径  

---

## Cursor 图像生成 Connection Stalled

- **现象**：内置 generate_image 卡住不返回  
- **解决**：禁用 Cursor 内置图像生成，改用本地脚本或外部服务（Pollinations/Gemini/Ollama 等）  
- **说明**：团队开发场景下如需图像生成，可统一走自建或第三方 API  

---

## 开发环境 PATH / 环境变量

- **现象**：终端里能跑的命令在 Cursor 里找不到  
- **解决**：确保 Cursor 启动时加载了与终端一致的环境（如 `.zshrc` / `.bash_profile`），必要时在项目里用 `env` 或 `.env` 显式指定

## 1. 问题与目标

- 从飞书云文档**指定文件夹**拉取多份会议纪要（如存客宝产研每日会议）。
- 生成**周期复盘**（目标-结果-过程-反思-下一步），并**写入团队日志·飞书云文档**；可选在对应飞书群发短通知+链接。

## 2. 成功做法

### 2.1 访问云文档必须用「用户 Token」

- 应用 Token（tenant_access_token）**无法**访问未授权给应用的云文档文件夹，会 403。
- 使用**用户 Token**（user_access_token）：一次授权后保存 `access_token` + `refresh_token`，脚本内自动刷新并写回同一文件。
- **统一 Token 路径**：团队内约定单一路径（如 `config/user_token.json` 或飞书管理 scripts 下的 `.feishu_tokens.json`），避免多处授权、重复提取。

### 2.2 团队日志：云文档 vs 群（必区分，防发错）

- **团队日志·飞书云文档**：日报/今日事情/复盘**正文**写在此处，不要发到群。
- **团队日志·飞书群**：仅发**短通知+云文档链接**（如「今日日志已更新：链接」）；Webhook 需单独配置。
- 错误做法：把整篇日志/复盘正文发到飞书群（群是通知用，不是正文落地点）。

### 2.3 文件夹内 .txt 用「下载接口」拉内容

- 云文档里 type 为 `file` 的 .txt（如产研会议纪要），不能用 docx 的 raw_content 接口。
- 使用：`GET /open-apis/drive/v1/files/{file_token}/download`，用用户 Token 请求，响应 body 即为文本内容。

### 2.4 最短路径一键命令

- 存客宝复盘：可封装为 `python3 scripts/run_cunkebao_review.py`（拉取默认文件夹 → 复盘 → 正文写入团队日志·云文档，可选群发短通知+链接）。
- Token、云文档 URL、Webhook、文件夹 token 全部从配置/凭证文档读取。

### 2.5 协同分工

- **单场会议** → 纪要正文 → 团队日志·云文档，可选群发短通知+链接。
- **多场会议周期复盘** → 目标/结果/过程/反思/下一步 → 正文写入**团队日志·云文档**，可选群发短通知+链接。

## 常见问题速查

| 问题 | 原因 | 解决 |
|:---|:---|:---|
| 迁移后站点502 | PHP版本不匹配/FPM未启动 | 检查PHP版本→重启FPM→检查.user.ini |
| 中文目录导致404 | PHP-FPM无法解析中文路径 | 站点根目录避免中文，或创建英文软链接 |
| No input file specified | open_basedir限制 | 修改.user.ini的open_basedir |
| 全量迁移丢站点 | 未对比新旧面板站点列表 | 先导出旧面板站点列表→逐个核实 |

---

## 迁移标准流程

1. 旧面板导出站点/数据库列表 → 新面板对比  
2. rsync 同步网站文件 → mysqldump 导出数据库  
3. 新面板创建站点+数据库 → 导入数据 → 检查 PHP 版本  
4. 修改域名解析 → 验证每个站点  
5. 遗留问题排查：.user.ini、伪静态、SSL 证书
