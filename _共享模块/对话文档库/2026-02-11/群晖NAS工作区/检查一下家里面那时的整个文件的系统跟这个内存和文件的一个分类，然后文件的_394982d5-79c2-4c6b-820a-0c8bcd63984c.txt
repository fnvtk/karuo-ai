## 五、重复文件查找

### 5.1 在 NAS 上执行的命令（SSH 登录后）

**按文件名找「同名且多个」的抽样（仅文件名）：**

```bash
find /volume1 -type f 2>/dev/null | xargs -I {} basename "{}" | sort | uniq -d | head -100
```

**按文件大小+名前几 KB 找可能重复（需较长时间）：**

```bash
# 大文件按大小排序，便于人工比对
find /volume1 -type f -size +10M 2>/dev/null -exec ls -la {} \; | sort -k5 -rn | head -80
```

**用 checksum 找内容完全一致（示例：仅 homes 下）：**

```bash
find /volume1/homes -type f -size +1M 2>/dev/null -exec md5sum {} \; | sort | uniq -w32 -d --all-repeated=separate
```

### 5.2 重复文件结果表（待采集后填入）

| 类型 | 说明 | 结果示例 |
|------|------|----------|
| 同名文件 | 上述 uniq -d 输出 | 将 NAS 上命令输出粘贴至此 |
| 大文件列表 | 按大小排序的前 N 个 | 便于人工判断是否重复或可删 |
| 同 MD5（可选） | 内容完全一致 | 去重后可只保留一份 |

## 七、CPU 与整体性能优化（把负载降下来）

### 7.1 高 CPU 来源（典型）

| 进程/套件 | 说明 | 建议 |
|-----------|------|------|
| synoelasticd / SynoFinder | 索引服务 | 缩小索引范围、降低强度、错峰 |
| fileindexd | 文件索引 | 同上 |
| smbd | SMB 文件共享 | 正常；若持续高可查是否有大量读写客户端 |
| syslog-ng | 系统日志 | 限制日志级别与大小 |
| defunct (SYNO.*) | 僵尸进程 | 重启对应套件或重启 NAS 清理 |

### 7.2 可执行降负方案

| 优先级 | 操作 | 做法 |
|--------|------|------|
| 1 | 降低索引负载 | 控制面板 → 索引服务：只勾选必要共享文件夹；Photo/Video 只选必要相册 |
| 2 | 错峰计划任务 | 备份、同步、归档放到凌晨或空闲时段；避免多任务同时全量 |
| 3 | 限制容器资源 | Container Manager：每个容器设 CPU/内存上限；不用的停止 |
| 4 | 减少套件 | 不用的套件停用或卸载 |
| 5 | 用资源监控定位 | 资源监控 → 进程，按 CPU 排序，对高占用套件/容器限流或错峰 |
| 6 | 先腾空间再减负 | /volume1 满时索引与日志会加重 I/O，先清理到 95% 以下 |
| 7 | 重启释放 | 若僵尸进程多、负载持续高，可安排一次重启 NAS |

### 7.3 本机（Mac）与 NAS 的区分

- **本报告与 [本机Mac性能与降负方案](./本机Mac性能与降负方案.md)** 分开：本报告只针对**家里 DiskStation（29）**的磁盘与 CPU；本机 Mac 的 CPU/内存降负见该文档。
- 若本机通过 Synology Drive 与 29 同步，可在本机设「按需下载」或限速，减轻双方 CPU 与 I/O。
