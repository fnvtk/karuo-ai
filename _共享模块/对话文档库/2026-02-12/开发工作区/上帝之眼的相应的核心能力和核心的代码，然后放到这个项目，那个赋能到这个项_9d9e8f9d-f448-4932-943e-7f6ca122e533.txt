## 二、能力 → 代码 → 接口 详细映射

### 2.1 数据采集

- **能力**：雪球热门帖子采集、解析、提取股票代码与作者信息。
- **文档**：`开发文档/6、后端/01-数据采集服务.md`
- **核心代码**：
  - `backend/app/services/collect/xueqiu_crawler.py`  
    - `XueqiuCrawler`：`crawl_hot_posts()`、`_parse_post()`、`_extract_stocks()`
  - `backend/app/services/collect/crawl_scheduler.py`  
    - 调度爬虫、可写库（依赖 `PostRepository`、`get_db`）
- **赋能说明**：爬虫已按文档实现；扩展东财/淘股吧时，可复用同一调度与存储模型。

### 2.2 NLP 分析

- **能力**：情感分析（利好/利空/中性）、实体识别（股票代码、公司名）。
- **文档**：`开发文档/6、后端/02-NLP分析服务.md`、`开发文档/核心代码提取/05-利好利空关键字提取.md`
- **核心代码**：
  - `backend/app/services/nlp/keywords.py`  
    - `POSITIVE_KEYWORDS` / `NEGATIVE_KEYWORDS`、`KeywordMatcher.analyze()`
  - `backend/app/services/nlp/sentiment_analyzer.py`  
    - `SentimentAnalyzer.analyze()` / `batch_analyze()`  
    - `EntityExtractor.extract()`
- **接口**：当前为服务层，供采集管道或后续「股票舆情」API 调用；`GET /api/v1/stocks/{code}/sentiment` 可对接此处结果。
- **赋能说明**：关键字库与规则已全量在代码中；后续可加 BERT 分支（文档中有说明）。

### 2.3 策略引擎（7 根 K 线 + 节假日）

- **能力**：节假日前 2 天买入、第 7 天卖出、止损 -5%；强中强选股（强势板块前 5 等）。
- **文档**：
  - `开发文档/2、架构/策略库/01-核心策略/7根K线策略.md`（参数、选股流程、回测预期）
  - `开发文档/2、架构/策略库/01-核心策略/节假日策略.md`（春节/五一/国庆 T-2、T+7 表与买点卖点）
- **核心代码**：
  - `backend/app/services/strategy/seven_kline.py`  
    - `SevenKLineStrategy`：`get_next_holiday()`、`should_buy()`、`generate_buy_signals()`  
    - 节假日列表、强势股目前为示例数据，可与行情/板块接口对接后替换。
- **API**：`GET /api/v1/signals`（已接 `SevenKLineStrategy.generate_buy_signals()`）
- **赋能说明**：文档中节假日策略的完整参数与时间表已全量写入；代码侧可逐步把「强势板块/强势股」改为从行情服务读取，实现文档中的选股流程。

### 2.4 高手跟踪与排行榜

- **能力**：高手综合评分（年化收益×0.5 + 胜率×0.3 + 夏普×0.2）、排行榜、跟踪持仓。
- **文档**：`开发文档/2、架构/策略库/01-核心策略/高手跟踪策略.md`、`开发文档/核心代码提取/04-高手排行榜算法.md`
- **核心代码**：
  - `backend/app/services/ranking/expert_ranking.py`  
    - `ExpertRankingService.calculate_expert_score()`、`generate_ranking()`
  - `backend/app/services/strategy/seven_kline.py`  
    - `ExpertTrackingStrategy.monitor_experts()`（骨架，待实现）
- **API**：`GET /api/v1/experts`、`GET /api/v1/experts/{user_id}` 当前为 mock；可改为调用 `ExpertRankingService` + 持久化数据。
- **赋能说明**：算法与公式已全量在代码与文档中；全量赋能 = 将 experts API 接到 `ExpertRankingService` 并接入真实数据源。

### 2.5 回测

- **能力**：策略历史回测、收益/胜率/夏普/回撤等指标。
- **文档**：`开发文档/6、后端/03-策略引擎服务.md`
- **核心代码**：`backend/app/services/backtest/backtest_engine.py`  
  - `BacktestEngine.run_backtest(strategy_name, stock_codes, start_date, end_date)`
- **API**：`POST /api/v1/backtest`（body: strategy_name, stock_codes, start_date, end_date, initial_capital）
- **赋能说明**：当前为模拟回测；全量赋能可接入真实 K 线（如 Tushare/AKShare）与 Backtrader 完整引擎（文档有技术选型）。

AI 字段向量化服务

将情感、实体等 AI 分析结果编码为固定维度的向量，便于检索与展示。
优先使用 sentence-transformers，不可用时回退为基于哈希的确定性向量。
"""

from typing import List, Dict, Any
import hashlib
import struct


# 默认向量维度（与简单回退一致，便于后续切换模型）
DEFAULT_DIM = 64

_embedding_model = None


def _get_model():
    """延迟加载 embedding 模型（可选）"""
    global _embedding_model
    if _embedding_model is not None:
        return _embedding_model
    try:
        from sentence_transformers import SentenceTransformer
        # 使用轻量多语言模型，支持中文
        _embedding_model = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2")
        return _embedding_model
    except Exception:
        return None


def _hash_to_vector(text: str, dim: int = DEFAULT_DIM) -> List[float]:
    """将文本转为确定性向量（无模型依赖）"""
    h = hashlib.sha256(text.encode("utf-8")).digest()
    vec = []
    for i in range(dim):
        start = (i * 4) % len(h)
        chunk = h[start : start + 4].ljust(4, b"\x00")
        val = struct.unpack("<f", chunk)[0]
        vec.append(val)
    return vec


def embed_text(text: str, dim: int = DEFAULT_DIM) -> List[float]:
    """
    对单段文本做向量化。
    若 sentence_transformers 可用则返回模型向量，否则返回哈希向量。
    """
    model = _get_model()
    if model is not None:
        emb = model.encode(text, convert_to_numpy=True)
        # 若模型维度大于 dim 则截断，小于则补零
        if len(emb) >= dim:
            return emb[:dim].tolist()
        return list(emb) + [0.0] * (dim - len(emb))
    return _hash_to_vector(text, dim)


def embed_ai_fields(analysis: Dict[str, Any], body_snippet: str = "", dim: int = DEFAULT_DIM) -> List[float]:
    """
    将 AI 分析结果（情感、实体等）编码为一段文本再向量化，便于后台统一展示与检索。
    """
    parts = []
    # 情感
    if "sentiment" in analysis:
        s = analysis.get("sentiment", {}) if isinstance(analysis.get("sentiment"), dict) else {}
        if isinstance(analysis["sentiment"], str):
            parts.append(f"sentiment:{analysis['sentiment']}")
        else:
            parts.append(f"sentiment:{s.get('label', s.get('sentiment', ''))}")
        parts.append(f"confidence:{analysis.get('confidence', 0)}")
    # 实体
    entities = analysis.get("entities", {})
    if entities.get("stocks"):
        parts.append("stocks:" + ",".join(entities["stocks"]))
    if entities.get("companies"):
        parts.append("companies:" + ",".join(entities["companies"]))
    if body_snippet:
        parts.append(body_snippet[:300])
    text = " ".join(parts)
    return embed_text(text, dim)

import asyncio
from typing import List, Dict, Any
from datetime import datetime


def _run_sync(fn, *args, **kwargs):
    """在线程池中运行同步函数，避免阻塞事件循环"""
    return asyncio.get_event_loop().run_in_executor(None, lambda: fn(*args, **kwargs))


async def run_ai_on_posts(posts: List[Dict]) -> List[Dict]:
    """
    对一批帖子执行：情感分析 + 实体识别 + 向量化。
    返回与 posts 同序的 [ { "analysis", "embedding" }, ... ]。
    """
    from services.nlp.sentiment_analyzer import SentimentAnalyzer, EntityExtractor
    from services.nlp.embedding_service import embed_ai_fields

    analyzer = SentimentAnalyzer()
    extractor = EntityExtractor()
    results = []

    for post in posts:
        body = ""
        if post.get("content"):
            body = (post["content"].get("title") or "") + " " + (post["content"].get("body") or "")
        body = (body or "")[:2000]

        # 情感
        sentiment_result = analyzer.analyze(body)
        # 实体
        entities = extractor.extract(body)
        analysis = {
            "sentiment": {
                "label": sentiment_result.get("label", "中性"),
                "sentiment": sentiment_result.get("sentiment", "neutral"),
                "confidence": sentiment_result.get("confidence", 0),
                "method": sentiment_result.get("method", "keyword"),
                "positive_count": sentiment_result.get("positive_count", 0),
                "negative_count": sentiment_result.get("negative_count", 0),
            },
            "entities": entities,
            "analyzed_at": datetime.utcnow().isoformat() + "Z",
        }

        # 向量化（在线程中执行，避免阻塞）
        embedding = await _run_sync(
            embed_ai_fields,
            analysis,
            body[:500],
        )
        results.append({"analysis": analysis, "embedding": embedding})

    return results


async def save_ai_results_to_posts(db_collection, posts: List[Dict], ai_results: List[Dict]) -> int:
    """
    将 AI 结果写回帖子文档。
    posts 与 ai_results 一一对应；通过 post_id 或 _id 更新。
    返回更新条数。
    """
    updated = 0
    for post, ai in zip(posts, ai_results):
        post_id = post.get("post_id") or str(post.get("_id", ""))
        if not post_id:
            continue
        from bson import ObjectId
        try:
            if hasattr(post.get("_id"), "binary"):
                q = {"_id": post["_id"]}
            else:
                q = {"post_id": post_id}
            await db_collection.update_one(
                q,
                {
                    "$set": {
                        "analysis": ai["analysis"],
                        "embedding": ai["embedding"],
                        "ai_updated_at": datetime.utcnow(),
                    }
                },
            )
            updated += 1
        except Exception:
            continue
    return updated

在你这个项目里，`AI 向量化` 本质上是把“文本和 AI 字段”变成机器可计算的数字坐标（向量），这样系统才能做“语义级”处理，不只是关键词匹配。

## 为什么要做向量化（核心原因）

- **把主观内容变成可计算资产**：帖子正文、情感、实体这些本来是非结构化信息，向量化后可比较、可检索、可排序。
- **从“字面匹配”升级到“语义匹配”**：不再只靠“有没有这个词”，而是看“意思是否相近”。
- **给策略和推荐提供统一输入**：后续做高手跟踪、信号筛选、相似事件召回，都需要统一数值特征。
- **支持规模化自动化**：帖子量大时，向量检索和聚类比人工看内容快很多。
- **为后续 AI 能力预留接口**：RAG、语义搜索、相似案例回放、异常检测都依赖向量。

## 向量化在上帝之眼里具体起到什么作用

你现在这条链路已经是：

`采集帖子 -> AI分析(情感/实体) -> 向量化 -> 回写数据库 -> 后台API展示`

它在项目里的作用就是 4 件事：

- **语义检索能力**：可以按“内容相似度”找帖子，不只是按股票代码/关键词。
- **信号增强能力**：策略可用向量相似度过滤噪音帖子，提升信号质量。
- **内容聚类能力**：把同类观点自动归组，方便看市场叙事和情绪簇。
- **后台可观测能力**：你在后台能直接看到 AI 字段 + embedding，便于调试和复盘。

## 一句话总结

向量化不是“可有可无的技术点”，它是把你项目里的“采集内容”升级成“可被策略和 AI 反复利用的数据底座”的关键步骤。  
没有向量化，你只有数据；有了向量化，你才有可复用的智能能力。
