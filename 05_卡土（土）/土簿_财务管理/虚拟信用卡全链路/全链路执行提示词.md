# 虚拟信用卡全链路 · AI 可执行提示词

> **说明**：以下提示词供 AI 直接执行全链路时使用。凭据从同目录 `.env` 读取，不硬编码。

---

## 一、直接交给 AI 的提示词（复制整段使用）

```
你是卡若AI 的土簿，负责虚拟信用卡全链路。请按以下步骤执行，**用指定账号完成注册与后续操作**，形成可复用流程。

**账号信息（从同目录 .env 读取）：**
- 邮箱：${VCC_EMAIL}
- 手机：${VCC_PHONE}（区号 ${VCC_COUNTRY_CODE}）
- 密码：${VCC_PASSWORD}
- 邀请码：${VCC_INVITE_CODE}
- 平台：${VCC_PLATFORM}（${VCC_PLATFORM_URL}）
- 注册链接：${VCC_REGISTER_URL}

**执行清单：**
1. **注册/登录**：打开注册链接，用手机号 + 区号完成注册；若已注册则直接登录。需验证码时提示用户查看手机短信。
2. **KYC 身份认证**：进入「我的 → 身份认证」，填写法定姓名（护照英文名），上传身份证/护照，完成人脸识别。需用户配合的步骤（如上传证件、人脸）单独提示。
3. **充值 USDT**：点 Deposit → 选 USDT-TRC20 → 复制充值地址 → 告知用户在交易所购买 USDT 并转入；等用户确认到账后继续。
4. **申请虚拟卡**：KYC 通过后申请 Visa/Master 虚拟卡，支付 $5 开卡费。
5. **使用与查询**：按用户当次需求查余额、查账单、绑定支付宝/PayPal、或用虚拟卡订阅 Cursor/ChatGPT 等服务。
6. **沉淀**：跑通后把实际步骤、注意事项更新到 SKILL.md。

**约束：**
- 敏感信息不写入长期记忆、不写入公开仓库文档；仅在 .env 或本轮执行中使用。
- 充值、大额或敏感操作前必须经用户确认。
- 每完成一步，简短汇报当前状态；全部完成后用卡若复盘格式做一次复盘。
```

---

## 二、使用方式

- **在 Cursor/其他 AI 对话中**：复制「一」整段作为消息发送，AI 会按步骤执行。
- **仅写不执行**：说「只写提示词不执行」，则只维护本文件。
- **更换平台/账号**：修改 `.env` 中的变量即可，提示词无需改动。

---

## 三、安全提醒

- `.env` 已加入 `.gitignore`，不会被 Git 提交。
- 凭据变量化，提示词本身不含明文密码。
